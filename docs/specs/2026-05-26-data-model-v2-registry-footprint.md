# V2 Data Model — Open Registry + Object Footprint (ADR)

> **Branch:** `mmo-rpg/zone-map-amaw` (continuing — V1 viewer + V2 data model in one PR)
> **Builds on:** V1.2 viewer (`a9c516d7`) — viewer exposed the architectural gaps this ADR resolves.
> **Status:** PROPOSED 2026-05-26 (PO direct approval for scope + branch strategy)
> **Size:** XL (50+ files, 15+ logic blocks, 5 side-effect surfaces)

## 1. Why this ADR exists

V1.2 viewer surfaced two cross-system architectural gaps that closed-enum
contracts cannot solve:

### Gap A — TerrainKind / TilemapObjectKind extensibility

Both `TerrainKind` (10 closed variants) and `TilemapObjectKind` (9 closed
variants) are Rust enum + TS mirror. Adding any new variant requires:
- Rust enum change
- 200+ `match` statement updates
- TS mirror sync
- Re-baseline golden tests
- Re-deploy both services

LoreWeave is a multi-book platform — each ingested book becomes its own
world with its own thematic types ("qi-meadow" / "demon-blight-soil" /
"holo-portal" / "wish-well" etc.). Closed enum **cannot scale** to per-book
content; it forces every new book to land enum migrations.

### Gap B — Object footprint contract

`TilemapObjectPlacement` declares only an `anchor: TileCoord` per placement.
Placer treats every kind identically as a 1×1-tile occupancy. But each
kind has dramatically different real spatial footprint:
- Tree = 1 tile
- Mountain = 3×3 tiles
- Mine = 2×2 (entrance + approach)
- Town = 4–6 tiles

Frontend renders sprites at visually-true size (Tier-XL = 6 tiles wide,
Tier-M = 3 tiles), but anchor is single-tile. Result: sprites visually
overlap because backend treats nearby anchors as non-conflicting.

These two gaps are **architecturally entangled**: the registry refactor
(Gap A) naturally carries footprint as a per-kind property (Gap B). Fixing
them in one pass is cleaner than two sequential refactors.

## 2. Design — Hybrid Primitive + Open Registry

### 2.1 Closed-primitive layer (engine semantics) — world-sim framing

**Decision criterion:** primitive = class **engine code path** that
cannot be derived from a property. Test: if adding it requires the
engine to write a new handler (tick loop, persistent state, lookup
table), it's a primitive. If it's configurable behavior on an
existing primitive, it's a property.

LoreWeave is a **world simulator** broader than RPG/puzzle — primitives
expanded after survey of 6 reference games (Dwarf Fortress, RimWorld,
Minecraft, Stellaris, Project Zomboid, Caves of Qud).

Universal across every book — physics + game logic only. Closed enums,
fast match, stable.

```rust
/// Engine-level terrain semantics. Every tile resolves to exactly one
/// primitive. Adding a primitive is a major version bump.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum TerrainPrimitive {
    /// Walkable substrate. Default for any non-special tile.
    /// Configurable via properties (movement_speed, growth_capacity,
    /// resource_yield, climate_zone, z_level).
    Land,
    /// Impassable surface; bridge/ford crossings allowed.
    /// `depth` property reserved for future swim-mechanic split.
    Water,
    /// Impassable rock face; no crossings.
    Wall,
    /// Walkable + road render + future movement bonus. Generated by
    /// RoadPlacer; bridges are Path tiles over Water.
    Path,
    /// Walkable + damage tick handler (lava, acid, poison fog).
    /// `damage_per_tick` property pins the rate; engine ticks it.
    Hazard,
    /// Unrendered; collision-as-wall. Off-map / unknown region.
    Void,
}

/// Engine-level object semantics. Every object kind resolves to one
/// primitive. Adding a primitive is a major version bump.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum ObjectPrimitive {
    /// Walkable, no effect, no state. Lantern, sign-only, debris.
    /// `readable: true` + `text: "..."` for narrative signs.
    Decoration,
    /// Walkable; auto-collect on enter; one-shot consume.
    /// Treasure pile, dropped coin, herb on ground.
    Pickup,
    /// Walkable; manual open; persistent loot state.
    /// Chest, barrel, crafting station (`interaction_kind: crafting`),
    /// bookshelf (`interaction_kind: reading`).
    Container,
    /// Walkable; drill-down to interior scene (CSC_001). Town,
    /// settlement, dungeon entrance.
    Habitable,
    /// Impassable; stateless. Mountain, tree, rock pillar.
    /// `transparent: true` for future window/LOS engine.
    Blocker,
    /// Toggleable Blocker; persistent open/closed state.
    /// `locked: true + key_id: "..."` for locked doors. Gate, hatch,
    /// portcullis.
    Door,
    /// Walkable; fires event on enter (or other condition via
    /// `condition: "proximity_3 | item_held=key | time=midnight"`).
    /// Portal, pressure plate, narrative anchor. Subsumes Sensor
    /// (no-walk trigger) via condition property.
    Trigger,
    /// Walkable; transport mechanic; routes player to destination.
    /// Ferry, mount, cart, teleporter (distinct from Trigger:
    /// Trigger fires event + done; Vehicle transports + may have
    /// persistent state like fuel/durability).
    Vehicle,
    /// Periodic entity generation. Monster nest, spawn point, gate
    /// to invasion wave. `spawn_pool` + `spawn_interval` properties.
    Spawner,
    /// Periodic INVENTORY resource generation (distinct from Spawner
    /// which yields ENTITIES). Farm, mine, fishing spot, herb patch.
    /// `resource_yield: HashMap<String, u32>` + `tick_interval`.
    Producer,
    /// Growing entity — harvestable; may spread; may have age states.
    /// Tree, crop, mushroom patch, herb. Distinct from Producer
    /// (which is a static building); Plant is the entity itself.
    /// `growth_capacity` + `harvest_yield` + `spread_chance` properties.
    Plant,
}
```

### 2.1.1 Property conventions

Engine code reads these properties WHEN the corresponding handler is
implemented. Books define values; engine ignores unknown properties.

| Property | Engine reads when |
|---|---|
| `movement_speed: f32` | movement / pathfinding (already exists) |
| `damage_per_tick: u32` | Hazard primitive tick handler |
| `growth_capacity: f32` | Plant growth tick |
| `harvest_yield: HashMap<String, u32>` | Plant harvest event |
| `spread_chance: f32` | Plant spread tick (future) |
| `resource_yield: HashMap<String, u32>` | Producer tick |
| `tick_interval: u32` | Producer / Spawner / Plant tick scheduling |
| `spawn_pool: Vec<String>` | Spawner |
| `loot_table_id: String` | Container open |
| `interaction_kind: String` | Container UI dispatch (loot / crafting / reading) |
| `readable: bool, text: String` | Decoration UI overlay (Sign) |
| `condition: String` | Trigger evaluator (on_walk default) |
| `key_id: Option<String>` | Door lock |
| `destination_event: String` | Vehicle transport target |
| `climate_zone: String` | weather engine (future) |
| `z_level: String` | render layer (future) |
| `transparent: bool` | LOS engine (future) |

### 2.1.2 Promotion roadmap

These ARE valid primitives, but their engine code paths haven't been
designed yet. Promote when game mechanic ships:

| Future primitive | Promotion trigger |
|---|---|
| `Slippery` (terrain) | ice/slip physics ships |
| `DeepWater` (terrain) | swim mechanic ships |
| `Subterranean` (terrain) | z-level / underground rendering ships |
| `Atmosphere` (terrain) | breathable-air mechanic ships |
| `NPC` (object) | entity system branch ships |
| `Animal` (object) | wildlife branch ships |
| `Workplace` (object) | job-assignment branch ships |
| `LightSource` (object) | lighting / LOS engine ships |
| `SoundSource` (object) | audio sim ships |
| `Conduit` (object) | sci-fi book requires power / pipe networks |

Forward compat: adding a new primitive variant is ADDITIVE to Rust
enum; all existing registry entries continue to work. Properties on
existing primitives may also be promoted to primitives later (e.g.
`Land + slip: true` → `Slippery` primitive) — registry migrator
handles this.

### 2.2 Open registry layer (per-book content)

Each book ships a registry file (TOML) describing its terrain + object
kinds. Loaded at world-gen time; engine resolves tags to definitions.

```rust
/// Per-tag terrain definition loaded from registry.
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct TerrainKindDef {
    /// Stable tag string — used in `TerrainCell.tag` and on the wire.
    /// Convention: lowercase + dashes; namespaced per book ("lw:grass",
    /// "xianxia:qi-meadow"). Default registry uses `lw:` prefix.
    pub id: String,
    pub primitive: TerrainPrimitive,
    pub label: String, // display string ("Qi-rich Meadow")
    /// Per-kind biome affinity, palette hints, walking modifiers, lore
    /// hooks. Open property bag; engine ignores; frontend + LLM read.
    #[serde(default)]
    pub properties: serde_json::Value,
}

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct ObjectKindDef {
    pub id: String,
    pub primitive: ObjectPrimitive,
    pub label: String,
    /// Logical tile footprint at anchor. (1,1) = single tile;
    /// (4,4) = 4×4 area starting at anchor.
    pub footprint: GridSize,
    /// Per-tile walkability within the footprint. Default: all tiles
    /// match primitive (Blocker = all blocked; Habitable = all walkable;
    /// etc). Can override per-tile via `walkability_pattern`.
    #[serde(default)]
    pub walkability_pattern: Option<WalkabilityPattern>,
    /// Minimum Chebyshev distance to other placements of the same kind
    /// (towns can't cluster; trees can). 0 = no spacing rule.
    #[serde(default)]
    pub min_spacing: u32,
    #[serde(default)]
    pub properties: serde_json::Value,
}

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub enum WalkabilityPattern {
    /// All footprint tiles walkable (Habitable default).
    AllWalkable,
    /// All footprint tiles blocked (Blocker default).
    AllBlocked,
    /// Specific tile mask within footprint (e.g. mine entrance: only
    /// anchor tile walkable, rest blocked).
    Mask(Vec<bool>),
}
```

### 2.3 Registry container + loader

```rust
#[derive(Debug, Clone, Default)]
pub struct TerrainRegistry {
    by_tag: HashMap<String, TerrainKindDef>,
}

impl TerrainRegistry {
    pub fn load_default() -> Self { /* embedded default.toml */ }
    pub fn load_from_yaml(path: &Path) -> Result<Self, Error>;
    pub fn get(&self, tag: &str) -> Option<&TerrainKindDef>;
    pub fn primitive(&self, tag: &str) -> TerrainPrimitive {
        self.get(tag).map(|d| d.primitive).unwrap_or(TerrainPrimitive::Void)
    }
    pub fn all_tags(&self) -> impl Iterator<Item = &str>;
}

// ObjectRegistry parallel structure
```

Default registry shipped in-binary (or under `services/tilemap-service/
registry/default.toml`) — covers all current 10 terrain + 9 object kinds
under `lw:` namespace for backward source-of-truth.

### 2.4 Wire shape — break (not additive)

Same-branch refactor means frontend lands aligned with backend wire
change. Additive shim is dead weight; break cleanly.

```rust
/// Replaces previous `terrain_layer: Vec<u8>` (was TerrainKind u8 index).
pub struct TilemapView {
    // ...
    pub terrain_layer: Vec<TerrainCell>,
    pub object_placements: Vec<TilemapObjectPlacement>, // shape extended
    pub registry_ref: RegistryRef,                       // new
}

pub struct TerrainCell {
    pub primitive: TerrainPrimitive,
    pub tag: String, // refs into TerrainRegistry
}

pub struct TilemapObjectPlacement {
    pub primitive: ObjectPrimitive,
    pub tag: String,
    pub anchor: TileCoord,
    pub footprint: GridSize,           // resolved from registry at gen time
    pub orientation: Direction,         // for asymmetric props
    #[serde(default)]
    pub properties: serde_json::Value, // optional per-instance overrides
    // existing fields kept for backward-readable JSON
    pub canon_ref: Option<String>,
    pub value: Option<u32>,
}

pub struct RegistryRef {
    /// Registry identifier loaded at gen time (default `"lw"`).
    pub id: String,
    /// Version pinning for migration / compatibility checks.
    pub version: String,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum Direction { N, NE, E, SE, S, SW, W, NW }
```

### 2.5 Placer refactor

`place_and_connect_object` no longer matches `TilemapObjectKind`. It:

1. Looks up `kind_def` from `ObjectRegistry::get(&tag)`
2. Computes candidate placement region from `kind_def.footprint`
3. Rejects placement if **any** tile in footprint already occupied OR
   marked Blocker/Wall by another placement's walkability pattern
4. Reserves all footprint tiles in walkability map per `kind_def
   .walkability_pattern`
5. Enforces `kind_def.min_spacing` via Chebyshev distance check

Walkability map structure stays internal; placer's existing connectivity
algo (`would_seal_a_gap` + `find_access_path`) reads the updated map and
remains correct.

### 2.6 Biome rules refactor

Current `biome_library.rs` hard-codes terrain selection per biome. After
refactor, biome rules read terrain affinity from registry properties:

```yaml
# In registry-default.toml
- id: "lw:grass"
  primitive: land
  label: "Grass"
  properties:
    biome_affinity:
      grassland_temperate: 1.0
      coastal_water: 0.2
      drake_badlands: 0.3
```

Biome library walks registry, builds biome-to-tag table at startup. Same
result for default registry; per-book registries get their own affinity
without touching code.

## 4. Migration story

### 4.1 Default registry as backward-compat anchor

Default registry (`lw:` namespace) ships with the 10 current terrain
kinds + 9 object kinds. After refactor, calling
`/internal/v1/tilemaps/render` with default registry MUST produce
output structurally equivalent to current behavior (modulo new fields).

Test guard: a snapshot test on `tilemap_baseline.json` (the lone golden)
asserts old + new outputs differ only in:
- `terrain_layer` shape (Vec<u8> → Vec<TerrainCell>)
- `object_placements` shape (added footprint + primitive + properties)
- New `registry_ref` field

Terrain selection per biome + object placement coordinates MUST be
byte-identical for deterministic seed.

### 4.2 Wire shape break — single PR

Frontend lands consuming new shape in the same PR as backend refactor.
No prod deploys exist to migrate (V1 viewer is debug-only); break is
cheap.

### 4.3 LLM-driven registry generation hook (#037 follow-up)

This ADR doesn't implement LLM gen, but the registry schema is shaped to
support it. Book ingest pipeline (separate `mmo-rpg/book-ingest` branch
later) will:
1. Read book glossary + knowledge graph
2. LLM-extract terrain + object types
3. LLM-generate registry TOML
4. (Branch `mmo-rpg/asset-pipeline-v2`) gen sprite per registry entry

V2 ADR locks the registry contract that LLM gen will write into.

## 9. Lessons that will save cross-session

- **Spec data-shape ≠ spec semantics** — TilemapObjectPlacement shape defines
  fields, not what they mean. Cross-check semantic assumptions at every
  cross-system boundary, not just field shapes.
- **Closed enum fails multi-tenant** — hybrid (closed-primitive + open
  tag-registry) is the canonical pattern for multi-book / multi-mod /
  multi-tenant platforms (Minecraft blocks, RimWorld defs, Stellaris
  script). Per-tenant registry can be LLM-generated for LLM-driven
  platforms.
