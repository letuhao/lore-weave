# LOOM T5.4 — Dock / Float / Pop-out Windowing (spec)

**Date:** 2026-06-25 · **Branch:** `feat/composition-service` · **Size:** XL · **Type:** FE (pure)
**Status:** the FINAL drafted V1 composition-studio feature — once usable, the branch is closeable.

## Goal
Replace the fixed sub-tab strip in the composition studio with a **dockable / floatable /
OS-pop-out windowing** system: the author arranges the 19 studio panels (Compose, Cast,
Timeline, Grounding, References, …) as a dock rail, draggable in-app floating windows, or
**separate OS browser windows** — and the **live state survives every move** (no remount of a
streaming co-writer, a chat SSE, or the Tiptap editor).

## CLARIFY decisions (PO-approved 2026-06-25)
- **Scope: FULL** — dock + in-app float + **OS pop-out** (BroadcastChannel + a SharedWorker that
  owns the live state across browser windows).
- **Flag: default OFF**, per-device localStorage. The current `CompositionPanel` fixed-strip is the
  always-available fallback. Windowing is opt-in.
- **Persistence:** the layout (which panels are docked/floated/popped, positions, sizes) is
  per-device localStorage (a per-device UI-state, allowed by the CLAUDE.md persistence rules;
  server-prefs sync is **deferred**). BE: none.

## The AH-2 architecture — hoist live state above the windowing layer
The studio's no-remount invariant exists because live state (a streaming co-writer ghost, a chat
SSE, the Tiptap editor instance) dies on unmount. Windowing *moves* a panel's host node — which
WOULD remount it. So the live state is **hoisted to a shared owner** and each window renders a
**thin view** that reads/writes the shared state:

| Live state | Today | After T5.4 |
|---|---|---|
| Co-writer SSE (`useCompositionStream`: ghost/streaming/jobId + AbortController) | inside `ComposeView` | owned by **LiveStateContext** (in-app) → **SharedWorker** (cross-window) |
| Chat SSE (CoWriterChat stream) | inside `CoWriterChat` | same |
| Tiptap editor instance | `ChapterEditorPage` | stays single-instance; floated/popped editor views are deferred (editor is dock-only in V1 — see Non-goals) |

**Key invariant:** a panel's React subtree is mounted **once** (in the shared owner / a stable
host) and only its *container* (dock slot ↔ floating window ↔ popped window) changes. Moving a
panel re-parents its DOM host, never remounts its component — same discipline as the current
CSS-hidden tabs, extended across windows via portals + (for pop-out) a serialized state mirror.

## Components (files)
- `context/LiveStateContext.tsx` — hoists the co-writer + chat streams to a single owner above the
  windowing layer; exposes `{ stream, chat }` so any window's Compose/Chat view is a thin consumer.
  In-app (same document) this is plain React context; cross-window it syncs via the SharedWorker.
- `context/WorkspaceLayoutContext.tsx` — the layout model (panel → placement: `dock | float | popout`,
  order, rect) + reducer + localStorage persistence + the feature flag.
- `hooks/useWorkspaceLayout.ts` — the controller (dock/float/pop-out/close/move/resize actions).
- `components/workspace/WorkspaceShell.tsx` — the windowing host: renders the dock rail + floating
  windows + pop-out bridges; falls back to the fixed `CompositionPanel` when the flag is OFF.
- `components/workspace/DockRail.tsx` — the dnd-kit rail (reorder, show/hide, drag-to-float).
- `components/workspace/FloatingWindow.tsx` — an in-app draggable/resizable window (portal to body).
- `components/workspace/ComponentPicker.tsx` — add a panel to the workspace (the "+" menu).
- `components/workspace/PopoutBridge.tsx` — opens an OS window (`window.open`) and renders the
  panel into it via a portal to the popped document; BroadcastChannel keeps non-live UI state synced.
- `workers/liveState.shared-worker.ts` — the SharedWorker: owns the co-writer/chat fetch+SSE,
  fans tokens out to all connected window ports (so a pop-out keeps streaming when the opener closes).
- `types.ts` — `Placement`, `WorkspacePanelId`, `WorkspaceLayout`, message protocols.
- `CompositionPanel.tsx` / `ChapterEditorPage.tsx` — mount `WorkspaceShell` under the flag; keep the
  fixed strip as the OFF fallback.

## Milestones (each a risk boundary → commit; POST-REVIEW batched per milestone)
- **M1 — Foundation (this cycle's first commit).** `LiveStateContext` (hoist co-writer + chat
  streams; ComposeView/CoWriterChat become thin consumers), `WorkspaceLayoutContext` + flag +
  localStorage, `types.ts`, and the `WorkspaceShell` flag-gate with the fixed-strip fallback. NO
  visual windowing yet — proves the hoist preserves the no-remount invariant + the stream survives a
  re-parent. This is the load-bearing refactor; everything else builds on it.
- **M2 — Dock rail.** `DockRail` (dnd-kit reorder/show-hide) + `ComponentPicker`; the shell renders
  docked panels from the layout. Replaces the fixed strip when the flag is ON.
- **M3 — In-app float.** `FloatingWindow` (portal, drag/resize) + dock↔float transitions; a floated
  panel is a thin view over the same hoisted state (a floated Compose keeps streaming).
- **M4 — OS pop-out.** `PopoutBridge` + `liveState.shared-worker.ts`: pop a panel to a real OS
  window; the SharedWorker owns the SSE so the stream survives the opener closing; BroadcastChannel
  syncs layout/selection. The hardest, highest-risk milestone.

## Non-goals (V1)
- The Tiptap **editor** floats/pops only as a dock-managed panel sharing the single editor instance;
  a fully independent second editor view (collaborative multi-window editing) is OUT.
- Server-synced layout (multi-device same arrangement) — per-device localStorage only; deferred.
- Mobile windowing — the flag is desktop-oriented; mobile keeps the fixed strip.

## Risks / guards
- **No-remount regression** — the whole point; every milestone's tests assert a panel's mount-count
  stays 1 across a placement change (the existing CompositionPanel mount-counter pattern, extended).
- **SharedWorker availability** — not all browsers/contexts support SharedWorker; `PopoutBridge`
  feature-detects and degrades pop-out to a dedicated `Worker`-less BroadcastChannel relay (the
  opener window stays the SSE owner; closing it ends the stream, with a warning) — pop-out still
  works, just loses the survive-opener-close property. Flag OFF when neither is available.
- **SSE abort discipline** — the hoisted stream keeps `useCompositionStream`'s superseded-controller
  + reader-cancel guards (the SSE-reader-must-cancel-on-abort lesson); the SharedWorker mirrors them
  per port.
- **localStorage corruption** — the layout loader validates + falls back to the default dock layout
  on any parse/shape error (never crashes the studio).

## Acceptance
Flag ON: panels reorder in a dock rail, pop into floating windows, and pop out to OS windows; a
co-writer generation started in one placement keeps streaming through a dock→float→pop-out move and
survives the opener window closing (SharedWorker path). Flag OFF: the studio is byte-identical to
today's fixed strip. tsc 0; no-remount invariant tests green per milestone.
