# T5.3 — AI-provenance highlight (mark-reviewed) — design spec

**Track** 5 (Editor shell) · **Phase** P2 · **Type** [FS] · **Depends-on** the AI-insert paths (ComposeView accept, T3.2 selection, T3.3 inline) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` `.prov` paragraph + `.provtag` "✦ AI · unreviewed" (line 278) — "AI wrote this — click to mark reviewed".

> Track which prose the AI wrote vs the human, and let the author **mark AI spans reviewed**. Honesty + revision hygiene: you can see what hasn't been read yet. Provenance rides as a **Tiptap mark** in the chapter content (no new table) so it persists with the doc.

## 1. UX

```text
He let his empty fingers fall open. "I have nothing
left to draw," he said.  ┌─────────────────────────┐
▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔ │ ✦ AI · unreviewed  [✓ reviewed]│
(faint AI underlay)       └─────────────────────────┘
human-written prose has no marker · editing an AI span
keeps it AI until "reviewed" · accept-all-reviewed in the toolbar
```

- AI-inserted spans (from accept / selection-edit / inline-continue) carry an **AI-provenance mark** (`source:'ai'`, `reviewed:false`), shown as a faint underlay + a hoverable `✦ AI · unreviewed` tag.
- **Mark reviewed** (click the tag / a toolbar "mark visible reviewed") flips `reviewed:true` → the underlay fades; the span is now "owned".
- Human edits inside an AI span keep it AI-unreviewed until explicitly reviewed (Open Q1 — split-on-edit).
- A toolbar counter: "3 AI spans unreviewed".
- Toggle the provenance layer on/off (it's an overlay, not always shown).

## 2. FE shape

- **Tiptap mark** `features/composition/editor/ProvenanceMark.ts` — `aiProvenance` mark with attrs `{ source, reviewed, jobId? }`; rendered as a class. Applied by the insert paths (accept/selection/inline) on the inserted range.
- **hook** `useProvenance` — counts unreviewed spans; `markReviewed(range|all)`; toggle the overlay.
- **component** — a provenance toolbar control + the per-span tag (a Tiptap node-view / decoration for the tag).
- **api / types** — the mark serializes **into the chapter content JSON** (book-service stores Tiptap JSON) — no new endpoint; provenance persists with the chapter.
- **host** — `ChapterEditorPage` editor.

## 3. BE delta

**None new** *if* book-service round-trips unknown marks in the chapter content JSON (Open Q2). The mark rides in the stored Tiptap doc. If the content schema strips unknown marks, register the `aiProvenance` mark in the shared editor schema (FE) and confirm book-service persists it verbatim.

## 4. Data contract

`aiProvenance` Tiptap mark: `{ source: 'ai'|'human', reviewed: boolean, jobId?: string }` embedded in the chapter content JSON. Applied on insert (AI paths), flipped on review. No separate wire type — it's part of the document.

## 5. Acceptance criteria

- AI-inserted prose is visibly marked (faint underlay + tag); human prose is unmarked.
- The author marks a span (or all visible) reviewed → the marker fades; the state persists with the chapter (survives reload + device switch).
- An unreviewed counter reflects reality.
- The provenance layer can be toggled off without affecting the text.

## 6. Test outline

- **vitest**: accept/selection/inline insert applies the AI mark; markReviewed flips `reviewed`; counter accurate; overlay toggle; the mark serializes + re-parses (round-trip).
- **BE/contract**: a chapter saved with provenance marks re-loads with them intact (book-service round-trip test).
- **Playwright** `composition-provenance.spec.ts`: accept an AI draft → span marked unreviewed → mark reviewed → persists on reload.
- **i18n ×4** keys: `prov.ai_unreviewed`, `prov.ai_reviewed`, `prov.mark_reviewed`, `prov.mark_all`, `prov.toggle`, `prov.count`.

## 7. Decisions (cleared 2026-06-10)

- **Split-on-edit:** keep the span AI-unreviewed until explicitly reviewed (simplest, honest); refine later. (Lead)
- **Mark persistence:** verify book-service round-trips the custom `aiProvenance` mark in the chapter JSON; **allowlist it in the shared editor schema if stripped** (the one possible BE touch). (Lead)
- **Overlay off by default**; a toggle + an always-visible unreviewed counter. (Lead)
