- An embedded chat thread in the co-writer sub-tab, **bound to the current book/scene** (so replies are grounded in the book's memory/canon вҖ” `useEmbeddedChatBinding` already does the book-memory link).
- Per-AI-message actions: **Insert as draft** (вҶ’ `onAccept(text)`, the existing editor-insert path) and **Use as guide** (вҶ’ pre-fills the `ComposeView` guide for a grounded generate).
- Streams replies (chat-service SSE, already wired in `features/chat`).
- Degrade: if the book-memory link fails, the existing chat warning toast ("replies may lack book context") shows.
- Empty вҶ’ a few starter prompts ("brainstorm the next beat", "what would X do?").

## 2. FE shape

- **component** `features/composition/components/CoWriterChat.tsx` вҖ” wraps the embeddable `<Chat>`/`useEmbeddedChatBinding` bound to `{ bookId, sceneId }`; adds the **Insert/Use-as-guide** action bar on AI messages. Reuses the chat feature's stream/providers (don't re-implement streaming).
- **hook** вҖ” reuse `useEmbeddedChatBinding` (book-memory link); a thin `useCoWriterBridge` to wire Insert вҶ’ `onAccept`, Use-as-guide вҶ’ the compose guide state (lifted in `CompositionPanel`).
- **api / types** вҖ” reuse `features/chat/api.ts` + types. No new composition API.
- **host** вҖ” the co-writer sub-tab (mockup default). Mounted always-on (CSS-hidden) so the thread survives tab switches (no-ternary-unmount rule).

## 3. BE delta

**None** (reuse chat-service + the existing book-memory binding). Optional: a scene-scoped context hint passed into the chat binding so replies know the *current scene* (not just the book) вҖ” small, FE-side via the binding params if supported (Open Q1).

## 4. Data contract

Reuse `features/chat` contracts (session create + SSE stream + the book-memory context binding). The composition bridge only consumes the AI message text (Insert/Use-as-guide) вҖ” no new wire types.

## 5. Acceptance criteria

- Author brainstorms conversationally in the co-writer panel; replies are grounded in the book's canon/memory.
- An AI reply can be inserted as draft prose or pushed into the generate guide.
- The thread persists across sub-tab switches.
- Book-memory link failure degrades with the existing warning, not a crash.

## 6. Test outline

- **vitest**: Insert вҶ’ calls `onAccept` with the message text; Use-as-guide вҶ’ sets the guide; thread survives a tab toggle (visibility test); binding-failure warning. (Reuse chat test idioms.)
- **Playwright** `composition-cowriter-chat.spec.ts`: send a prompt вҶ’ streamed reply вҶ’ Insert as draft appears in the editor.
- **i18n Г—4** keys: `cowriter.title`, `cowriter.insert`, `cowriter.use_as_guide`, `cowriter.ask`, `cowriter.starters`.
