# T5.1 — Focus / typewriter mode — design spec

**Track** 5 (Editor shell) · **Phase** P2 · **Type** [FE] · **Depends-on** Tiptap editor (exists) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` toolbar `#focusBtn` (line 226), `.focusline` (276), `.focuspill` (269).

> A distraction-free writing mode: dim everything but the active line/paragraph, optionally keep the cursor vertically centered (typewriter scroll), and hide chrome. Pure editor UX.

## 1. UX

```text
            ┌──────────────────────────────────┐
   (dim)    │  The active line is bright;       │  ← focus line
            │  surrounding text fades to ~40%.  │
   (dim)    └──────────────────────────────────┘
   ◎ Focus  ·  typewriter: cursor stays vertically centered
   Esc / ◎ to exit · left+right panels auto-hide in focus
```

- Toggle **Focus** (toolbar button / shortcut). In focus mode:
  - the active paragraph (or line) is full-opacity; the rest dims (~40%).
  - **typewriter scroll** (optional sub-toggle): the caret line stays vertically centered.
  - side panels auto-hide (restore on exit); a minimal focus pill shows continuity status.
- Exit via Esc or the toggle; restores the previous layout.
- Per-device UI state (focus on/off, typewriter on/off) — localStorage (allowed for per-device UI).

## 2. FE shape

- **hook** `features/composition/hooks/useFocusMode.ts` — focus + typewriter flags (localStorage); on focus, drive the dim + the centering scroll (a Tiptap selection-update handler centers the caret).
- **component** — a `FocusMode` wrapper / editor-chrome class toggles (dim via a CSS class on non-active blocks; the active block from the Tiptap selection). `ModeToggle` button in the toolbar.
- **api / types** — none.
- **host** — `ChapterEditorPage` (the manuscript editor). Must not remount the editor on toggle (no-unmount; CSS class + scroll only).

## 3. BE delta

**None** — pure FE editor UX.

## 4. Data contract

None (per-device UI state in localStorage). The active block comes from the Tiptap selection; dimming is a CSS class on non-active blocks.

## 5. Acceptance criteria

- Toggling Focus dims all but the active paragraph and hides side chrome; exiting restores layout.
- Typewriter scroll keeps the caret centered while writing.
- No editor content or hook state is lost entering/exiting focus.
- The preference persists per device.

## 6. Test outline

- **vitest**: toggle sets the focus class; active block (from selection) is undimmed; typewriter centering invoked on selection change; exit restores; persists to localStorage. (Visibility-lifecycle: no remount.)
- **Playwright** `composition-focus-mode.spec.ts`: enter focus → surrounding text dims + panels hide; type → caret stays centered; Esc restores.
- **i18n ×4** keys: `focus.enter`, `focus.exit`, `focus.typewriter`.

## 7. Decisions (cleared 2026-06-10)

- **Pure FE, per-device** (localStorage); no-remount on toggle. (Lead)
- **Active unit = paragraph** (line-level optional later); typewriter = caret-centered scroll. (Lead)
- **Panels auto-hide in focus**, restore on exit (integrates with T5.4 windowing once it lands). (Lead)
