# T1.2 вҖ” Beat Sheet вҖ” design spec

**Track** 1 (Structure) В· **Phase** P2 В· **Type** [FS] В· **Depends-on** T1.1 (`useOutline`), renders inside **T5.5** power-view overlay В· **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` power-view `data-v="beats"` (lines 431вҖ“439 вҖ” `.bscard` per template beat).

> Surfaces a **structure template's beats** as cards, each showing the scene(s) mapped to it + a fill-state, so the author sees coverage (which beats are drafted / empty / unplaced). Today only the planner's `unmapped_beats` text hints at this. Mapping key: `template.beats[].key` вҶ” `node.beat_role`.

- Pick a structure template (reuse `useStructureTemplates`); render **one card per `beat`** in template order вҖ” card shows the beat `key` (title) + `purpose` (tooltip) + the **mapped node(s)** + fill-state derived from node `status` (doneвҶ’drafted, draftingвҶ’writing, empty/outlineвҶ’empty; no nodeвҶ’**unplaced**).
- **Map / unmap:** **drag a scene/chapter onto a beat card** (dnd-kit) to set `node.beat_role = beat.key`; a per-node `<select>` of beat keys is the a11y fallback. Works at **scene *and* chapter** level (PO). Clearing it unmaps.
- **Click a card's mapped node** вҶ’ jump to it (editor / outline).
- **Unmapped nodes** (carry no `beat_role`) listed so the author can place them.
- Empty (no template chosen) вҶ’ prompt to pick one.

## 2. FE shape (`features/composition/`)

- **hook** вҖ” reuse `useStructureTemplates(token)` (beats) + `useOutline(projectId, token)` (nodes). A pure selector `buildBeatSheet(template, nodes)` (unit-tested in isolation) joins `beats[].key` вҶ” `nodes[].beat_role` вҶ’ `{ beat, nodes, state }[]` + `unmappedNodes`. Map/unmap = `useOutline().patchNode(id, { beat_role })`.
- **component** `components/BeatSheetView.tsx` (view) вҖ” card grid from the selector; per-card mapped nodes + state; the assign control. Render-only + bubble to the hook.
- **api.ts** вҖ” reuse `listTemplates` + `getOutline` + `patchNode`. No new calls.
- **types.ts** вҖ” `StructureTemplate { id; name; kind; beats: Beat[] }`, `Beat { key: string; purpose: string }`.
- **host** вҖ” the **T5.5 power-view overlay** "beats" view (its natural home per the mockup). For P2 ordering, if T5.5's shell isn't built yet, ship it as a standalone panel reachable from the Planner area and **graft into the overlay when T5.5 lands** (Open Q3).

## 3. BE delta

**None for the MVP.** Read: `GET /v1/composition/templates` вҶ’ `{ templates: StructureTemplate[] }` ([canon.py:137](../../../services/composition-service/app/routers/canon.py)); `GET /outline` for nodes. Map: `PATCH /outline/nodes/{id}` with `{ beat_role }`. *(beat keys are validated by the decompose planner against `template.beats`; a manual assign should also constrain to the chosen template's keys вҖ” FE-side.)*

## 4. Data contract

`StructureTemplate.beats: { key: string; purpose: string }[]` (confirmed via `engine/plan.py:113` `b.get('key')` / `b.get('purpose')`). `node.beat_role` stores a beat **key** (or `null`). Join is by exact `key` match. Fill-state from `node.status`.

## 5. Acceptance criteria

- Author picks a template and sees every beat as a card with its mapped node + fill-state.
- Beats with no node show **unplaced**; nodes with no beat show in the **unmapped** list.
- Assigning/clearing a beat on a node persists (`beat_role` patch) and the cards update.
- Clicking a mapped node jumps to it.

## 6. Test outline

- **vitest**: `buildBeatSheet` selector вҖ” join by key; unplaced beat; unmapped node; fill-state from status. `BeatSheetView` вҖ” cards render in template order; assign вҶ’ `patchNode({beat_role})`; click вҶ’ navigate.
- **Playwright** `composition-beat-sheet.spec.ts`: pick template вҶ’ beats render; assign a scene to a beat вҶ’ card reflects it.
- **i18n Г—4** keys: `beats.title`, `beats.pick_template`, `beats.unplaced`, `beats.unmapped`, `beats.assign`, fill-state labels (`drafted`/`writing`/`empty`).

## 7. Decisions (cleared 2026-06-10)

- **Mapping level: scene AND chapter (PO)** вҖ” show/assign `beat_role` on either node kind (decompose seeds chapters; the author can assign scenes too). No BE change.
- **Assign UX: drag (dnd-kit)** a node onto a beat card (PO: DnD now); a `<select>` is the a11y fallback.
- **Host:** standalone panel вҶ’ graft into the **T5.5** power-view overlay. (Lead)
- **Template switch re-keys** the join (old beats вҶ’ unmapped) вҖ” accepted; `beat_role` is a free key. (Lead)
