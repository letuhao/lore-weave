# T4.2 — Progress & Stats (streak / goals) — design spec

**Track** 4 (Track/Flywheel) · **Phase** P2 · **Type** [FS] · **Depends-on** none (greenfield; `recharts` available) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Source:** components-catalog §D (tagged **LATER** — pulled into V1 by PO 2026-06-10). **Not in v3 mockup** (the mockup shows only a live word count) → this wireframe is the design.

> Writing momentum: a daily word-count streak, a session/daily goal, and a small trend chart. Today only a live word count exists (editor chrome). This adds the **tracking** layer (persisted, multi-device).

## 1. UX

```text
┌─ 📊 Progress ──────────────────────────────
│  today   1,840 / 1,500 words   ✓ goal met
│  streak  🔥 6 days
│  ▁▂▅▃▇▆█  last 7 days (recharts)
│  total   84,210 words · 12 chapters
│  goal: [ 1500 ] words/day            ✎
└─  per-book · counts published + draft prose
```

- **Today's words** vs a daily **goal** (editable); goal-met state.
- **Streak** (consecutive days with writing).
- A small **7/30-day trend** chart (recharts — already a dep).
- **Totals** (words, chapters).
- Per-book (Open Q2: per-book vs per-user aggregate).
- Server-persisted (multi-device): the streak/goal/daily counts must be the same on phone + PC.

## 2. FE shape

- **hook** `features/composition/hooks/useProgress.ts` — `useProgress(projectId, token)` → reads daily-progress + goal (BE, §3); `setGoal(n)`; reports a day's word delta (debounced) from the editor's word count.
- **component** `features/composition/components/ProgressPanel.tsx` — today/goal/streak/total + a `<recharts>` sparkline. Extract the chart.
- **api.ts** — progress read + goal write + a day-count report (see §3).
- **types.ts** — `Progress { today, goal, streak, total_words, total_chapters, daily: {date, words}[] }`.
- **host** — a docked right-panel sub-tab (§D).

## 3. BE delta

**NEW (composition-service):** a **daily-progress store** —
- `composition_daily_progress(user_id, project_id, date, words)` (a small table) + a `daily_goal` on the work (`work.settings.daily_goal`).
- Endpoints: `GET /v1/composition/works/{project_id}/progress` (today + goal + streak + totals + recent daily); `POST …/progress/report` (record today's word delta — idempotent per day, server clamps); goal via the work-settings update.
- Streak/totals computed server-side from the daily rows (server is the SSOT — no localStorage-only counters; the no-local-user-data rule).

> **Date handling:** "today" is user-timezone-sensitive — record/compute against the user's local date (pass it from the client or store tz) to avoid a midnight-UTC streak break (Open Q3).

## 4. Data contract

`GET /v1/composition/works/{project_id}/progress` → `{ today_words, daily_goal, streak_days, total_words, total_chapters, daily: [{date, words}] }`. `POST …/progress/report` body `{ date, words }` (the client's local date + cumulative or delta — clamp server-side). Goal write via work-settings (`daily_goal`).

## 5. Acceptance criteria

- Author sees today's words vs goal, a streak, a trend chart, and totals — the same across devices.
- Editing the daily goal persists; goal-met state reflects it.
- The streak increments on consecutive writing days and breaks correctly across the user's local midnight.
- No progress data is stored only locally (server SSOT).

## 6. Test outline

- **vitest**: today/goal/streak/total render; goal edit → `setGoal`; chart renders the daily series; word delta report debounced.
- **BE pytest** (composition): `progress/report` upserts the day idempotently; streak computed from daily rows; goal round-trips; timezone date handling.
- **Playwright** `composition-progress.spec.ts`: write → today's count updates → persists on reload (server, not localStorage).
- **i18n ×4** keys: `progress.title`, `progress.today`, `progress.goal`, `progress.streak`, `progress.total`, `progress.goal_met`, `progress.edit_goal`.

## 7. Decisions (cleared 2026-06-10)

- **Scope:** V1 = streak / daily goal / 7–30-day trend / totals — nothing heavier (catalog-LATER basic tier). (Lead)
- **Server-persisted** daily-progress store + goal (no localStorage-only — multi-device SSOT). (Lead)
- **Per-book V1**; a cross-book user streak later. (Lead)
- **Timezone:** record against the user's **local date** (client-supplied) so streaks don't break at UTC midnight. (Lead)
- **Word source:** daily manuscript (draft) edits; the label clarifies draft vs published. (Lead)
