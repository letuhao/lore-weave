# T1.3 — Scene Graph canvas — design spec

**Track** 1 (Structure) · **Phase** P2 · **Type** [FE] (scene-links BE exists) · **Depends-on** T1.1 (`useOutline`), renders inside **T5.5** power-view overlay · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` power-view `data-v="map"` (lines 401–416 — SVG `.mlinks` + `.mn` nodes).

> A typed-link **node canvas** over scenes: setup→payoff (and custom) edges made visible and editable. The `scene_links` CRUD already exists (`setup_payoff`/`custom`); today there is **no canvas** — only the flat `PlannerTree`. This is the spatial relationship view.

## 1. UX

```text
🎬 Scene Graph                                    opt-in · power view
   ┌ S1 · Arrival ┐ ──setup▶── ┌ S2 · Warning ┐
   └──────────────┘            └──────┬───────┘
                          setup▶      │
   ┌ S3 · Confront • ┐ ◀─────────────┘   ┌ S4 · Aftermath ┐
   └────────┬────────┘ ···········custom········· └────────────────┘
            └ click node → open scene · drag from a node edge → new link
legend:  ──▶ setup→payoff (solid)    ····· custom (dashed)
```

- Canvas of scene nodes (from `GET /outline`, `kind='scene'`) + edges (`scene_links`): **setup_payoff** = solid arrow, **custom** = dashed; edge label on hover.
- **Create a link:** drag from node A to node B → choose `kind` (+ optional `label`) → `POST /scene-links`. **Delete:** select an edge → ✕.
- **Click a node** → open that scene (editor/outline).
- **Layout:** the model has **no x/y** — P2 uses **auto-layout** (columns by `story_order`, rows to reduce crossings) with optional per-device drag (positions in localStorage — per-device UI state, allowed; not user data). See Open Q1.
- Empty (no scenes / no links) → hint to add a setup→payoff from two scenes.
- **Mobile:** pan/zoom; tap a node to open; link-create is a desktop affordance (Open Q3).

## 2. FE shape (`features/composition/`)

- **hook** — extend `useOutline` (nodes + `scene_links`) with `createSceneLink({from,to,kind,label})` / `deleteSceneLink(id)` mutations (invalidate the outline query).
- **component** `components/SceneGraphCanvas.tsx` (view) — renders nodes + SVG edges; link create/delete; node click. **Tech:** prefer a **lightweight hand-rolled SVG** (mirrors the mockup's raw SVG, no heavy dep) unless a graph lib is already in deps (Open Q2). Keep ≤200 lines — extract `SceneNode`/`SceneEdge` subcomponents + an `autoLayout(nodes, links)` pure helper (unit-tested).
- **api.ts** — `createSceneLink(projectId, body, token)`, `deleteSceneLink(linkId, token)`.
- **types.ts** — `SceneLink` (shared with T1.1).
- **host** — the **T5.5 power-view overlay** "map" view. Build standalone first; embed when T5.5 lands (Open Q4 — same as T1.2 Q3).

## 3. BE delta

**None.** Reuse ([outline.py](../../../services/composition-service/app/routers/outline.py)):
- `GET /outline` (nodes + scene_links).
- `POST /v1/composition/works/{project_id}/scene-links` (201) `{ from_node_id, to_node_id, kind, label }` — `409 SCENE_LINK_EXISTS` on dup, `400 BAD_REFERENCE` on bad/cross-work node.
- `DELETE /v1/composition/scene-links/{link_id}` (204).

> Open Q1 *may* add a node-position field if we want server-persisted layout — explicitly deferred for P2 (auto-layout + local positions).

## 4. Data contract

`SceneLink { id; from_node_id; to_node_id; kind: 'setup_payoff'|'custom'; label }`. Create body `{ from_node_id, to_node_id, kind?='setup_payoff', label?='' }`. Nodes from `GET /outline` (`kind='scene'`, positioned by `story_order`).

## 5. Acceptance criteria

- Author sees scenes as a graph with typed edges (setup→payoff solid, custom dashed).
- Create a setup→payoff (and custom) link between two scenes; delete a link.
- A duplicate link is rejected gracefully (409 → toast, no dup edge).
- Click a node opens its scene; the layout is legible without manual arranging.

## 6. Test outline

- **vitest**: `autoLayout` pure helper (columns by `story_order`, deterministic); `SceneGraphCanvas` — nodes+edges render with correct edge style per `kind`; create link → `POST` with `kind`; delete; node click; **409 → toast, no dup**.
- **Playwright** `composition-scene-graph.spec.ts`: link two scenes → edge appears; reload → edge persists (BE); delete → gone.
- **i18n ×4** keys: `graph.title`, `graph.new_link`, `graph.kind_setup`, `graph.kind_custom`, `graph.label`, `graph.empty`, `graph.exists`.

## 7. Decisions (cleared 2026-06-10)

- **Canvas: hand-rolled SVG** — no graph lib in `frontend/package.json`; matches the mockup, avoids a heavy dep. (Lead)
- **Layout: auto (columns by `story_order`)** + per-device drag positions in localStorage; server-persisted shared layout deferred (would need node `x/y`). (Lead)
- **Mobile:** view + open-node only; link-create is a desktop affordance. (Lead)
- **Node set: scenes only** for V1. (Lead)
- **Host:** standalone → graft into the **T5.5** overlay. (Lead)
