# T2.3 — Timeline UI — design spec

**Track** 2 (World/Knowledge) · **Phase** P2 · **Type** [FS] (mostly FE; BE ready) · **Depends-on** knowledge `/timeline` (exists), renders inside **T5.5** power-view overlay · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` power-view `data-v="timeline"` (lines 419–428 — `.tlline` points + `.tlcut` spoiler cutoff).

> The spoiler-safe chronology: the book's `:Event` nodes ordered by `event_order`, with the **"AI sees ≤ here"** cutoff visualized. The public `GET /timeline` is fully built (rich filters, chapter-title enrichment); there is **no UI**.

## 1. UX

```text
🕑 Timeline                                        opt-in · power view
filters: [ all entities ▾ ]  [ date range ]  [ ⌖ spoiler-cut at current ch ]
├─●─────────●──────────────●────────────┊··········○············▶ event_order
  Ch.9      Ch.11          Ch.12·S2     ▲ AI sees   (Ch.13 hidden:
  sword     oath broken    gatekeeper's  ≤ here      the Hollow King)
  shatters                 warning       spoiler-safe
click an event → open its chapter · filter by entity (→ Character Arc T2.4)
```

- Horizontal timeline of events ordered by `event_order`, labelled with `chapter_title` (BE-enriched) + summary. recharts (existing dep) or a custom SVG line — see Open Q1.
- **Spoiler cutoff** marker at the current chapter's `event_order` (`before_order`): events past it are dimmed/"hidden" (mirrors what the AI can see — `before_order` is the same axis the packer windows on).
- **Filters** (BE-supported): `entity_id` (→ feeds T2.4 Character Arc), `event_date_from/to`, `before/after_order`, `before/after_chronological`.
- **Click an event** → open its chapter (`/read` or editor) at that point.
- Pagination (`limit`/`offset`, `total`) for long books; reversed-range → the BE 422 surfaces as a friendly error.
- Empty → "no events yet (extract first)".

## 2. FE shape

- **hook** `features/composition/hooks/useTimeline.ts` — `useTimeline(projectId, token, filters)` → `knowledgeApi.listTimeline` (reuse `features/knowledge/api.ts`). Derives the cutoff `event_order` from the current chapter.
- **component** `features/composition/components/TimelineView.tsx` (view) — the line/axis, event points, the spoiler-cut marker, filter controls. recharts `<ScatterChart>`/custom axis (Open Q1). Extract `TimelineEvent`/`SpoilerCut`.
- **api / types** — reuse `features/knowledge` (`listTimeline`, `Event`).
- **host** — **T5.5** power-view overlay "timeline" view (already in the overlay's switcher per mockup) + reachable standalone.

## 3. BE delta

**None.** `GET /v1/knowledge/timeline` is complete: `project_id`, `after/before_order`, `after/before_chronological`, `entity_id`, `event_date_from/to`, `limit`/`offset`; events enriched with `chapter_title`; 422 on reversed ranges.

## 4. Data contract

`GET /v1/knowledge/timeline?project_id=&before_order=&entity_id=&event_date_from=&…&limit=&offset=` → `{ events: Event[], total }`. `Event { event_order, chronological_order, summary|description, participants[], chapter_id, chapter_title, event_date_iso, … }`. Spoiler cutoff = the current chapter's `event_order` passed as `before_order` (or rendered as a marker over an unfiltered fetch — Open Q2).

## 5. Acceptance criteria

- Author sees events on a chronological axis with chapter labels.
- The spoiler cutoff is visible; events beyond the current chapter are clearly "future/hidden".
- Filtering by entity / date / order narrows the axis; reversed range shows a friendly error.
- Clicking an event opens its chapter.

## 6. Test outline

- **vitest**: events render ordered; cutoff marker at the right `event_order`; entity filter → refetch with `entity_id`; date filter; 422 reversed-range → error; empty.
- **Playwright** `composition-timeline.spec.ts`: timeline renders for a seeded project; spoiler cut present; filter by entity narrows it.
- **i18n ×4** keys: `timeline.title`, `timeline.filter_entity`, `timeline.filter_date`, `timeline.spoiler_cut`, `timeline.hidden`, `timeline.empty`, `timeline.range_error`.

## 7. Decisions (cleared 2026-06-10)

- **Render: recharts-first** (existing dep); fall back to hand-rolled SVG if the spoiler-cut/label layout fights the lib. (Lead)
- **Cutoff: marker + dim** past-cutoff by default, with a **"hide spoilers" toggle** that switches to `before_order` filtering. (Lead)
- **Current chapter → `event_order`:** threaded like the M4d-1 timeline window. (Lead)
- **Host:** T5.5 power-view "timeline" view + standalone. (Lead)
