# T4.1 — Flywheel panel ("+N new facts learned") — design spec

**Track** 4 (Track/Flywheel) · **Phase** P1 · **Type** [FE + BE-read] · **Depends-on** publish→extraction (server-side, exists) + knowledge extraction jobs (exists) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** §D "Track" (Flywheel = the +N-facts surface). **Coverage:** §D MISSING (FE surface).

> Close the author-facing loop: when you publish a chapter, extraction canonizes it and **learns N new facts/entities** — surface that ("you taught the world N new things"). The flywheel runs server-side today (publish → canonization → extraction); there is **no author surface** for what it learned. (Distinct from the eval/corrections dashboard — `QualityPanel` — which is the *tuning* flywheel.)

## 1. UX

```text
┌─ ♻ Flywheel — what the world learned ─────────
│  Ch.12 "Confrontation at the Gate" · published 14:02
│  +4 entities   +7 relations   +3 events
│   ▸ new: "The Ashen Pact" (faction) · "Gate of Ash" (place)
│   ▸ Kael —betrayed→ Mira   · Mira —member-of→ Ashen Pact
│  [ view in Cast codex → ]   [ view in Timeline → ]
│  ─────────────────────────────────────────────
│  history: Ch.11 +2/+3/+1 · Ch.10 +5/+8/+2 …
└─  canon grows as you publish · advisory, not a gate
```

- After a publish, show the **latest extraction job's contribution**: counts of new entities / relations / events, with a few named highlights.
- **Deep-links** to the new items in the Cast codex (T2.1) / Relationship Map (T2.2) / Timeline (T2.3).
- A short **history** of recent publishes and what each taught.
- Pending/running state: "extracting… (canonizing Ch.12)"; never-extracted → the same "run an extraction first" hint as GroundingPanel.
- Advisory/celebratory — not a gate.

## 2. FE shape

- **hook** `features/composition/hooks/useFlywheel.ts` — `useFlywheel(projectId, token)` → `knowledgeApi.listExtractionJobs(projectId)` (reuse `features/knowledge` or extraction feature); derive the latest job's added-counts + highlights.
- **component** `features/composition/components/FlywheelPanel.tsx` — the contribution card + history + deep-links.
- **api / types** — reuse `features/knowledge` / `features/extraction` (`ExtractionJob`).
- **host** — a docked right-panel sub-tab (§D); also a good post-publish toast/celebration moment (Open Q3).

## 3. BE delta

**Reuse** `GET /v1/knowledge/projects/{project_id}/extraction/jobs` (history, newest-first). **Confirm (Open Q1):** does `ExtractionJob` carry **per-run added counts** (entities/relations/events added) + the new item ids/names? If not, add them to the job result (a small knowledge-service addition) — the panel needs the *delta*, not just job status.

## 4. Data contract

`GET /v1/knowledge/projects/{project_id}/extraction/jobs` → `ExtractionJob[]` (newest-first). Per job (confirm/extend): `{ id, status, chapter (title), entities_added, relations_added, events_added, new_items?: [{kind, id, name}], created_at }`. The panel reads the latest completed job (+ history).

## 5. Acceptance criteria

- After publishing a chapter, the author sees what the world learned (+N entities/relations/events) with a few named highlights.
- Deep-links open the new items in the codex / map / timeline.
- Running/never-extracted states are clear (not errors).
- Purely advisory — never blocks anything.

## 6. Test outline

- **vitest**: latest job's counts + highlights render; deep-links target the right views; running state; empty/never-extracted hint; history list.
- **Playwright** `composition-flywheel.spec.ts`: after a seeded publish+extraction, the panel shows +N and links to the codex.
- **i18n ×4** keys: `flywheel.title`, `flywheel.entities`, `flywheel.relations`, `flywheel.events`, `flywheel.new`, `flywheel.view_codex`, `flywheel.view_timeline`, `flywheel.extracting`, `flywheel.empty`.

## 7. Decisions (cleared 2026-06-10)

- **Added-count delta (build it):** extend `ExtractionJob` with per-run added counts (entities/relations/events) + new-item refs (knowledge-service) — the panel needs the delta, not just status. (Lead)
- **Publish→job correlation:** use the job's chapter ref (the list already enriches chapter titles). (Lead)
- **Surfacing:** docked panel + a lightweight post-publish toast. (Lead)
- **Distinct from `QualityPanel`** — this is the author-facing "+N facts" celebration, not the eval/corrections tuning dashboard. (Lead)
