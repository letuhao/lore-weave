# T1.1 — Outline Tree browser — design spec

**Track** 1 (Structure) · **Phase** P1 · **Type** [FE] (BE exists) · **Depends-on** none · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` left-panel **☰ Outline** (`data-lt="outline"`, lines 245/255–263 — the `otree`).

> A persistent **Act→Chapter→Scene→Beat** browser of the *committed* outline. Today `PlannerView`/`PlannerTree` render only the **decompose preview** (pre-commit); once committed, the tree from `GET /outline` has **no navigable browser**. This is the structural navigator + light node editing.

## 1. UX

```text
☰ Outline                                    [+]
▾ Act II — Descent                            ●
  ▾ Ch.12 — Ashfall                           ◐
      S1 · Arrival                            ●
      S3 · Confrontation              ‹cur›    ◉
      S4 · Aftermath                          ○
      + add scene
  ▸ Ch.13 — The Hollow King                   ○
status:  ● done   ◐ drafting   ◉ current   ○ empty/outline
```

- Left-panel **Outline** tab (AI-mode), rendering the committed tree from `GET /outline` — recursive `arc → chapter → scene → beat` by `parent_id`, ordered by `rank`/`story_order`.
- Each node row: a **status dot** (`status` → ●/◐/◉/○), the title, and the "current" marker for the open scene.
- **Navigate:** click a chapter → open it in the editor; click a scene/beat → select it (open in editor / planner — see Open Q3).
- **Light edit (reuse node CRUD):** inline rename (`title`); set `status`; **add child** (`+ add scene`/`+ add beat`); **archive** (✕). Collapse/expand per node.
- **Empty state:** "No outline yet" → CTA linking to the Planner (decompose) flow.
- **Concurrency:** patch carries `If-Match: <version>`; a `412` → toast "outline changed elsewhere" + refetch.
- **Reorder:** drag-to-reorder via **dnd-kit** → patch `rank`. (PO: DnD invested now.)
- **View toggle (cards ⇄ tree):** this panel also offers a **cards** layout — the Corkboard (T1.4) is the cards mode of *this* panel (shared `useOutline` + host), **not** a separate tab. (PO)
- **Mobile:** full-width; indentation by depth; 44px tap targets.

## 2. FE shape (`features/composition/`)

- **hook** `hooks/useOutline.ts` — `useOutline(projectId, token)` react-query `GET /outline` → `{ nodes, scene_links }` (scene_links unused here, consumed by T1.3). Mutations `createNode` / `patchNode` (passes `If-Match` from `node.version`) / `archiveNode`. Mirror `useCanonRules`; invalidate on success.
- **component** `components/OutlineTree.tsx` (view) — builds the nested tree from the flat `nodes` (group by `parent_id`, order by `rank`), renders rows; selection + edit callbacks. `components/OutlineNodeRow.tsx` for a single row (status dot, inline title, add/archive) — mirror `PlannerSceneRow`'s render-only + bubble-edit idiom.
- **api.ts** — `getOutline(projectId, token)`, `createNode(projectId, body, token)`, `patchNode(nodeId, patch, version, token)` (sets `If-Match`), `archiveNode(nodeId, token)`.
- **types.ts** — `OutlineNode`, `SceneLink`, `NodeKind`/`NodeStatus` unions (§4).
- **host** — a new **left-panel "Outline" tab** in `ChapterEditorPage.tsx` (which already owns the Chapters/Original/Glossary left tabs), matching the mockup. (Not a CompositionPanel right-sub-tab.)

## 3. BE delta

**None.** Reuse ([outline.py](../../../services/composition-service/app/routers/outline.py)):
- `GET /v1/composition/works/{project_id}/outline?include_archived=false` → tree + links.
- `POST /v1/composition/works/{project_id}/outline/nodes` (201).
- `PATCH /v1/composition/outline/nodes/{node_id}` (`If-Match` version → 412 on conflict; `status='done'` routes the commit-aware path that emits `composition.scene_committed`).
- `DELETE /v1/composition/outline/nodes/{node_id}` (archive, 200).

## 4. Data contract

```ts
type NodeKind = 'arc' | 'chapter' | 'scene' | 'beat';
type NodeStatus = 'empty' | 'outline' | 'drafting' | 'done';
interface OutlineNode {
  id: string; project_id: string; parent_id: string | null;
  kind: NodeKind; rank: string | null;
  title: string; pov_entity_id: string | null; present_entity_ids: string[];
  goal: string; beat_role: string | null;       // beat key (see T1.2)
  status: NodeStatus; chapter_id: string | null; // book chapter this node maps to
  tension: number | null;                        // 0..100
  story_order: number | null; synopsis: string; version: number;
}
interface SceneLink { id: string; from_node_id: string; to_node_id: string; kind: 'setup_payoff'|'custom'; label: string; }
```
`PATCH` body = any subset of the mutable fields; header `If-Match: "<version>"`. Errors: `412 NODE_VERSION_CONFLICT` (carries `current`), `400 BAD_REFERENCE` (reparent cycle / ownership).

## 5. Acceptance criteria

- Author browses the committed Act→Chapter→Scene tree, with status at a glance.
- Clicking a chapter/scene navigates to it; the current scene is marked.
- Rename / set-status / add-child / archive persist and reflect; a concurrent edit (412) is surfaced, not lost.
- No outline → a clear CTA into the Planner.
- Advisory structural tool — never blocks generation/publish.

## 6. Test outline

- **vitest** (`__tests__/OutlineTree.test.tsx`): nested tree builds from flat nodes; status→dot mapping; node select fires; rename → `patchNode` with `If-Match`; add child; archive; **412 conflict → refetch + toast**; empty → decompose CTA.
- **Playwright** `composition-outline-tree.spec.ts`: committed tree renders; navigate to a scene; rename persists.
- **i18n ×4** keys: `outline.title`, `outline.add_scene`, `outline.add_beat`, `outline.empty`, `outline.go_plan`, `outline.archive`, `outline.conflict`, status labels.

## 7. Decisions (cleared 2026-06-10)

- **Host:** left-panel **Outline tab** in `ChapterEditorPage` (per mockup). (Lead)
- **Reorder:** drag via **dnd-kit** → patch `rank`. (PO: DnD now)
- **Navigation:** scene click = select + scroll the editor to that scene's chapter. (Lead)
- **Corkboard unified:** T1.4 is the **cards layout of this panel** (cards ⇄ tree toggle, shared hook + host), not a separate feature. (PO)
