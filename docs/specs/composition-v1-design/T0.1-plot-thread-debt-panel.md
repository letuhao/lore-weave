# T0.1 — Plot-thread / promise-debt panel — design spec

**Track** 0 (quick wins) · **Phase** P0 · **Type** [FE] · **Depends-on** none (BE shipped FD-1 S4a) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` right-panel docked **🧵 Plot Threads** (`data-rt="threads"`, lines 296/303/370–376).

> Surfaces the promise/foreshadow ledger the generation flow already writes (FD-1 S2 producer + S3 re-injection). This is the human-visibility side — the **open** set is the author's unpaid-debt signal. **Advisory only (D4)** — a flag, never a gate.

## 1. UX

```text
┌─ 🧵 Plot Threads ──────────────────  [3 open]
│  ( ●Open )  ( All )
│  🔓 The missing letter                 promise
│  🔓 Mira's allegiance               foreshadow
│  🔒 Kael's lost sword            paid · promise
└─  advisory · P0 meta = kind+status (chapter# / due-soon deferred)
```

Dockable right-panel sub-tab "🧵 Plot Threads" (mockup §A). Mirrors the **CanonRulesPanel** list idiom (same right-panel `secttl` + row list).

- **Header row:** title "Open & paid threads" + an **open-count debt badge** (`open_count`, e.g. "3 open"). A small filter toggle: **Open** (default) ⇄ **All**.
- **Thread row** (`.thr`): a status token + summary + meta line:
  - `open`/`progressing` → 🔓 (warn color); `paid` → 🔒 (success); `dropped` → muted/strikethrough.
  - **summary** = the promise text (`thread.summary`).
  - **meta** = opened-at chapter ("set Ch.N" — see Open Q1) · "· due soon" when high-priority/aging (Open Q2) · "paid → …" when `payoff_node` set.
  - rows are **priority-ordered** (BE `list_open` already orders).
- **States:** loading ("Loading…"), empty ("No open promises — all paid." / "No threads yet."), error (toast).
- **Gating:** the panel/sub-tab renders **only when** `work.settings.narrative_thread_enabled` is true (matches the producer gate in `packer/pack.py:147`). When off, the sub-tab is hidden entirely (no empty tease).
- **Mobile:** full-width panel; rows wrap meta under summary.

## 2. FE shape (`features/composition/`)

- **hook (controller)** `hooks/useNarrativeThreads.ts` — `useNarrativeThreads(projectId, token, status: 'open'|'all')`. react-query read; **mirror `useCanonRules`** (queryKey `['composition','threads',projectId,status]`, `enabled: !!projectId && !!token`, `select: d => d.threads`). Also expose `openCount` from the same response (separate `select` or return the raw data + derive). Read-only — no mutations (the ledger is written by generation).
- **component (view)** `components/ThreadsPanel.tsx` — props `{ projectId, token, enabled }`. Status toggle (local `useState<'open'|'all'>`), open-count badge, the `.thr` row list, empty/loading. **Render nothing when `!enabled`.** Reuse the CanonRulesPanel row/markup conventions (`data-testid="composition-thread"`).
- **api.ts** — add `listNarrativeThreads(projectId, status, token): Promise<{ threads: NarrativeThread[]; open_count: number }>` (GET, `apiJson`, mirror `listCanonRules`).
- **types.ts** — add `NarrativeThread` (fields in §4) + `ThreadStatus`/`ThreadKind` unions.
- **host** — dock as a sub-tab in `CompositionPanel.tsx` (the existing 7-sub-tab strip). Pass `enabled={!!work.settings?.narrative_thread_enabled}`. (The mockup's "dock as panel" picker is the **T5.4 windowing** feature — out of scope here; for P0 add it as a fixed sub-tab like the others.)

## 3. BE delta

**None.** Reuse `GET /v1/composition/works/{project_id}/narrative-threads?status=open|all` ([narrative_threads.py:32](../../../services/composition-service/app/routers/narrative_threads.py)). JWT + `_require_work` ownership gate already present. `open_count` is a true COUNT (not LIMIT-capped).

## 4. Data contract

**Request:** `GET /v1/composition/works/{project_id}/narrative-threads?status=open` (default) | `all`.
**Response:** `{ "threads": NarrativeThread[], "open_count": number }`.

`NarrativeThread` ([models.py:56](../../../services/composition-service/app/db/models.py)):
```ts
type ThreadKind = 'promise' | 'foreshadow' | 'question' | 'mice_thread';
type ThreadStatus = 'open' | 'progressing' | 'paid' | 'dropped';
interface NarrativeThread {
  id: string; project_id: string; user_id: string;
  kind: ThreadKind;
  status: ThreadStatus;          // default 'open'
  opened_at_node: string | null; // scene/beat node UUID where set
  payoff_node: string | null;    // node where paid (→ "paid → …")
  priority: number;              // default 50; orders the open list
  summary: string;               // the promise text (display label)
  version: number;
}
```
No untrusted-text boundary concern in *this* read (display-only, escaped by React) — but `summary` originates from LLM detector output, so **do not** render it as HTML.

## 5. Acceptance criteria

- An author opens the Threads panel and sees the project's **open promises, priority-ordered**, with an open-count debt badge.
- Toggling **All** shows paid/dropped rows too, visually distinct (🔒 paid, muted dropped).
- The panel is **purely advisory** — no button blocks generation/commit.
- With `narrative_thread_enabled=false`, the sub-tab does not appear.
- Empty ledger → friendly empty state; backend error → toast, no crash.

## 7. Decisions (cleared 2026-06-10)

- **Meta scope (P0):** show **kind + status + summary** only. Defer node→chapter resolution ("set Ch.N") and "due soon" (no due field exists) — both additive later. (Lead)
- **Host:** fixed **gated sub-tab** in `CompositionPanel`; the dock/float picker is T5.4. (Lead)
- **Ordering:** keep BE `list_open` priority order; `open_count` badge independent of the filter.
