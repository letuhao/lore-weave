# T3.3 — Classic ⇄ AI inline mode (in-prose ghost + accept-bar) — design spec

**Track** 3 (Write & AI) · **Phase** P1 · **Type** [FE] · **Depends-on** `useCompositionStream` (exists), Tiptap editor · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` center editor (lines 274–285 — `.ghost` paragraph + `.accept` bar + `/` slash hint + ✦ Continue).

> Today the streamed ghost lives in the **side-panel** (`ComposeView`). The mockup's classic⇄AI mode puts the ghost **inline in the manuscript** at the cursor, with an accept-bar right there (✦ Continue / ↵ Accept / ✎ Edit / ⟳ Regenerate) — a document-mode toggle between plain writing (Classic) and AI-assisted (AI).

## 1. UX

```text
… he made himself walk toward it anyway.
"You came," she said, and something landed wrong.
┌ ✦ AI · drafting…────────────────────────────────┐
│ The fog held its breath. Beyond the gate,        │  ← inline ghost
│ something ancient turned to listen.              │     (dimmed, in-doc)
└──────────────────────────────────────────────────┘
[ ↵ Accept ]  [ ✎ Edit ]  [ ⟳ Regenerate ]  [ Esc Discard ]
mode: ( Classic ◯ ─ ● AI )   ·   type / for commands · ✦ Continue
```

- A document **mode toggle (Classic ⇄ AI)**. In **AI mode**, ✦ Continue (or `/continue`) streams the next beat as an **inline ghost** at the cursor (dimmed, in-document but not committed); an **inline accept-bar** offers Accept / Edit / Regenerate / Discard.
- **Classic mode** = plain Tiptap writing, no ghost affordances (the AI panel stays available but doesn't auto-inline).
- Accept commits the ghost into the doc (normal autosave); Discard removes it; Edit drops it into the doc as editable text.
- Slash commands (`/continue`, `/rewrite`→T3.2, …) as an alternate entry.
- Reuses the same `useCompositionStream` + critique-on-accept as `ComposeView` — only the **render target moves from the side panel to the editor**.

## 2. FE shape

- **Tiptap extension** `features/composition/editor/GhostExtension.ts` — a ProseMirror decoration that renders the streaming ghost at the cursor (non-editable widget) + the accept-bar; commits/【discards on action. Keep the stream state in the hook, the decoration reads it.
- **hook** — reuse `useCompositionStream`; a `useInlineGhost` adapter maps stream tokens → the decoration + Accept → `editor.commands.insertContentAt`.
- **component** `ModeToggle.tsx` (Classic ⇄ AI) in the editor chrome.
- **api / types** — none new (reuse the stream).
- **host** — `ChapterEditorPage` Tiptap editor. **No-unmount:** the ghost decoration must not destroy editor/stream state when toggling modes (CSS/branch, not remount).

## 3. BE delta

**None new (AH-1, REVIEW-ARCH):** inline-continue resolves **cursor → context** — scene-scoped `/generate` (`outline_node_id`) when the caret is in a **decomposed** scene, **else** the chapter-scoped `GenerateChapterBody` path (no node; already exists). So the co-writer works on an undecomposed chapter. Pure FE otherwise (render target + Tiptap extension).

## 4. Data contract

Reuse the cowrite SSE stream (`useCompositionStream`). The only new state is FE: the ghost decoration position + the mode flag (per-device UI state — OK in localStorage).

## 5. Acceptance criteria

- In AI mode, ✦ Continue streams an inline ghost at the cursor with an inline accept-bar.
- Accept commits the ghost into the manuscript; Discard removes it; Edit makes it editable.
- Classic mode writes plainly with no inline ghost.
- Toggling modes never loses editor content or an in-flight stream.

## 6. Test outline

- **vitest**: mode toggle renders/hides inline affordances without remount (visibility-lifecycle test); ghost decoration shows streamed tokens; Accept inserts at the cursor; Discard removes; Esc discards. (Radix/portal + Tiptap query caveats apply.)
- **Playwright** `composition-inline-mode.spec.ts`: AI mode → Continue → inline ghost → Accept commits into the doc.
- **i18n ×4** keys: `inline.mode_classic`, `inline.mode_ai`, `inline.continue`, `inline.accept`, `inline.edit`, `inline.regenerate`, `inline.discard`.

## 7. Decisions (cleared 2026-06-10)

- **Inline ghost = AI-mode default (PO)** — the primary drafting surface. The ComposeView side panel stays for **diverge/K-options + critic**; chat (T3.1) is brainstorm. (Three-layer co-writer.)
- **Tech:** ProseMirror **decoration widget** (non-editable, easy discard); **no-remount** on mode toggle. (Lead)
- **Slash menu:** `/continue` V1 + bridge to T3.2 ops (rewrite/expand); full menu later. (Lead)
- **Cursor→context resolver (AH-1, REVIEW-ARCH):** continue uses scene-scoped `/generate` when the caret is in a decomposed scene, else chapter-scoped `GenerateChapterBody` — the co-writer works on undecomposed chapters. (PO/Lead)
