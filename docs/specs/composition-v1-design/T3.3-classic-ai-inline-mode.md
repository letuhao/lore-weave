> Today the streamed ghost lives in the **side-panel** (`ComposeView`). The mockup's classicвҮ„AI mode puts the ghost **inline in the manuscript** at the cursor, with an accept-bar right there (вңҰ Continue / вҶө Accept / вңҺ Edit / вҹі Regenerate) вҖ” a document-mode toggle between plain writing (Classic) and AI-assisted (AI).

## 1. UX

- A document **mode toggle (Classic вҮ„ AI)**. In **AI mode**, вңҰ Continue (or `/continue`) streams the next beat as an **inline ghost** at the cursor (dimmed, in-document but not committed); an **inline accept-bar** offers Accept / Edit / Regenerate / Discard.
- **Classic mode** = plain Tiptap writing, no ghost affordances (the AI panel stays available but doesn't auto-inline).
- Accept commits the ghost into the doc (normal autosave); Discard removes it; Edit drops it into the doc as editable text.
- Slash commands (`/continue`, `/rewrite`вҶ’T3.2, вҖҰ) as an alternate entry.
- Reuses the same `useCompositionStream` + critique-on-accept as `ComposeView` вҖ” only the **render target moves from the side panel to the editor**.

## 2. FE shape

- **Tiptap extension** `features/composition/editor/GhostExtension.ts` вҖ” a ProseMirror decoration that renders the streaming ghost at the cursor (non-editable widget) + the accept-bar; commits/гҖҗdiscards on action. Keep the stream state in the hook, the decoration reads it.
- **hook** вҖ” reuse `useCompositionStream`; a `useInlineGhost` adapter maps stream tokens вҶ’ the decoration + Accept вҶ’ `editor.commands.insertContentAt`.
- **component** `ModeToggle.tsx` (Classic вҮ„ AI) in the editor chrome.
- **api / types** вҖ” none new (reuse the stream).
- **host** вҖ” `ChapterEditorPage` Tiptap editor. **No-unmount:** the ghost decoration must not destroy editor/stream state when toggling modes (CSS/branch, not remount).

## 3. BE delta

**None** вҖ” reuse `/generate` (cowrite stream). Pure FE (render target + Tiptap extension).

## 4. Data contract

Reuse the cowrite SSE stream (`useCompositionStream`). The only new state is FE: the ghost decoration position + the mode flag (per-device UI state вҖ” OK in localStorage).

## 5. Acceptance criteria

- In AI mode, вңҰ Continue streams an inline ghost at the cursor with an inline accept-bar.
- Accept commits the ghost into the manuscript; Discard removes it; Edit makes it editable.
- Classic mode writes plainly with no inline ghost.
- Toggling modes never loses editor content or an in-flight stream.

## 6. Test outline

- **vitest**: mode toggle renders/hides inline affordances without remount (visibility-lifecycle test); ghost decoration shows streamed tokens; Accept inserts at the cursor; Discard removes; Esc discards. (Radix/portal + Tiptap query caveats apply.)
- **Playwright** `composition-inline-mode.spec.ts`: AI mode вҶ’ Continue вҶ’ inline ghost вҶ’ Accept commits into the doc.
- **i18n Г—4** keys: `inline.mode_classic`, `inline.mode_ai`, `inline.continue`, `inline.accept`, `inline.edit`, `inline.regenerate`, `inline.discard`.

## 7. Decisions (cleared 2026-06-10)

- **Inline ghost = AI-mode default (PO)** вҖ” the primary drafting surface. The ComposeView side panel stays for **diverge/K-options + critic**; chat (T3.1) is brainstorm. (Three-layer co-writer.)
- **Tech:** ProseMirror **decoration widget** (non-editable, easy discard); **no-remount** on mode toggle. (Lead)
- **Slash menu:** `/continue` V1 + bridge to T3.2 ops (rewrite/expand); full menu later. (Lead)
