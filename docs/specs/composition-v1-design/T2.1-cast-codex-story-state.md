# T2.1 — Cast & Codex (per-entity story-state) — design spec

**Track** 2 (World/Knowledge) · **Phase** P1 · **Type** [FS] · **Depends-on** knowledge `/entities` + `:EntityStatus` store (exist) · **Status** DRAFT (REVIEW-IMPL HIGH-1 folded)
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` §B **👥 Cast & Codex** (`data-rt="cast"`, lines 362–369).

> The book's cast (characters / places / factions) as a docked codex, each with its **current story-state** + 1-hop relations. Today the editor shows only a flat glossary list; `suggest-cast` (T0.2) is the only cast-aware affordance. **Reads knowledge directly via gateway** (`/v1/knowledge`, reusing `features/knowledge/`).

## 1. UX

```text
┌─ 👥 Cast & Codex ───────────────  [search…]
│  ▸ char  Kael      active · decided to break the oath   ⟶ 3 rel
│  ▸ char  Mira      active                                ⟶ 2 rel
│  ▸ char  Hadon     gone (Ch.9)                           ⟶ 1 rel
│  ▸ place The Gate of Ash                                 ⟶ 1 rel
│  click an entity → status · known facts · relations · recent events
└─  state = :EntityStatus (active|gone) + :Fact (decisions/prefs) · spoiler-safe
```

- Docked right-panel sub-tab listing entities (`GET /entities?project_id=&kind=`), grouped/badged by `kind` (character / location / faction / concept), searchable (`?search=`).
- **Story-state line (CORRECTED — REVIEW-IMPL HIGH-1):** the entity's spoiler-safe state from **`:EntityStatus`** (`GET /entities/{id}/status?before_order=`) — V1 vocab `active | gone` (latest transition with `from_order ≤ P`), the store the A2 canon guard already uses (correct `event_order` axis). Plus **known facts** from `:Fact` (decision / preference / milestone / negation — a separate "what they've decided" list, NOT attribute state). **Fine attribute-state (e.g. "wounded") is modeled in neither store at V1 — don't promise it.** Falls back to recent events when both are empty.
- **Expand a row** → aliases, 1-hop relations (`GET /entities/{id}`), recent events.
- **Spoiler-safe:** state/events windowed to `before_order` = the current chapter (no future reveals).
- Empty (no extracted entities) → hint ("publish/extract to populate the codex").

## 2. FE shape

- **hook** `features/composition/hooks/useCast.ts` — `useCast(projectId, token, { kind?, search? })` → `knowledgeApi.listEntities` (reuse `features/knowledge/api.ts`); `useEntityDetail(entityId)` for 1-hop relations; `useEntityStatus(entityId, beforeOrder)` → the **new status route** (§3) for `active|gone`; optionally `useEntityFacts(entityId, beforeOrder)` for the known-facts list.
- **component** `features/composition/components/CastCodexPanel.tsx` (view) — list + search + expandable rows. Reuse the CanonRulesPanel list idiom + the `<meter>`/badge conventions.
- **api** — reuse/extend `features/knowledge/api.ts` (`listEntities`, `getEntityDetail`, `listTimeline`). No composition-service proxy.
- **types** — `Entity`, `EntityDetail`, `Relation`, `Event` (import from `features/knowledge/types`).
- **host** — fixed gated right-panel sub-tab in `CompositionPanel` (cross-cutting: docked = sub-tab until T5.4).

## 3. BE delta

**NEW (PO: build now — CORRECTED target, REVIEW-IMPL HIGH-1)** — `GET /v1/knowledge/entities/{entity_id}/status?before_order=` on **knowledge-service**: the spoiler-windowed current status from **`:EntityStatus`** (vocab `active|gone`; latest transition `from_order ≤ before_order`; `from_order` already on the `event_order` / `EVENT_ORDER_CHAPTER_STRIDE` axis). Genuinely a thin read — the store + axis already exist (A2 canon guard, `entity_status.py`). Optionally **`GET /entities/{id}/facts`** over the `:Fact` store (`FACT_TYPES = decision|preference|milestone|negation`) for a separate known-facts list. ⚠ **`:Fact` is NOT state** — closed-type, content-keyed, **no order axis, no entity link** (`entity_status.py:5-7`), so it can't model windowed current state. **The status route also powers T2.4's state band.** Plus reuse `GET /entities`, `/entities/{id}`, `/timeline`.

## 4. Data contract

- `GET /v1/knowledge/entities?project_id=&kind=&search=&limit=&offset=` → `{ entities: Entity[], total }`. `Entity { entity_id|canonical_id, name, canonical_name, aliases[], kind, project_id, version, … }`.
- `GET /v1/knowledge/entities/{entity_id}` → `EntityDetail { entity: Entity, relations: Relation[] /* 1-hop active RELATES_TO */ }` + `ETag`.
- `GET /v1/knowledge/timeline?entity_id=&before_order=&limit=` → `{ events: Event[], total }` (event context).
- **NEW** `GET /v1/knowledge/entities/{entity_id}/status?before_order=` → `{ status: 'active'|'gone', from_order, evidence? }` (latest `:EntityStatus` transition ≤ `before_order`; default `active`).
- *(optional)* `GET /v1/knowledge/entities/{entity_id}/facts` → `{ facts: Fact[] }`, `Fact.type ∈ {decision,preference,milestone,negation}` — the known-facts list (NOT attribute state).
- **project_id resolution:** `GET /v1/knowledge/projects?book_id=<book>` — **exists**; composition's `knowledge_client` already uses it (`knowledge_client.py:73`, LOW-6); 0-or-1 project; the FE calls it via the gateway.

## 5. Acceptance criteria

- Author sees the book's cast grouped by kind, searchable, with a current-state line + relation count.
- Expanding an entity shows aliases, 1-hop relations, and recent (spoiler-safe) events.
- State/events never reveal beyond the current chapter.
- No entities → a clear "extract first" empty state (not an error).

## 6. Test outline

- **vitest**: list renders grouped by kind; search debounced; expand → detail fetch (relations); status (active|gone) + known-facts render; spoiler window applied; gone-state shown distinctly; empty state.
- **Playwright** `composition-cast-codex.spec.ts`: cast renders for a seeded project; expand shows relations.
- **i18n ×4** keys: `cast.title`, `cast.search`, `cast.kind_character/location/faction/concept`, `cast.relations`, `cast.no_state`, `cast.empty`.

## 7. Decisions (cleared 2026-06-10)

- **State source (PO: build now — CORRECTED, REVIEW-IMPL HIGH-1): the `:EntityStatus` store**, not `:Fact`. `GET /entities/{id}/status?before_order=` → `active|gone` on the correct `event_order` axis (A2 already uses it). `:Fact` (decision/preference/…) is a separate optional known-facts list. **Fine attribute-state ("wounded") is not modeled in V1 — expectation reset.** Still a thin read, just over the right store.
- **Cast source: knowledge entities** (status/relations/events); cross-link the glossary entry for authored edits. (Lead)
- **knowledge `project_id`:** `GET /v1/knowledge/projects?book_id=` (exists; LOW-6). (Lead)
- **Host:** fixed gated right-panel sub-tab in `CompositionPanel`. (Lead)
