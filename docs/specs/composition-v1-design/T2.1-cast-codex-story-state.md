# T2.1 — Cast & Codex (per-entity story-state) — design spec

**Track** 2 (World/Knowledge) · **Phase** P1 · **Type** [FS] · **Depends-on** knowledge `/entities` (exists) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` §B **👥 Cast & Codex** (`data-rt="cast"`, lines 362–369).

> The book's cast (characters / places / factions) as a docked codex, each with its **current story-state** + 1-hop relations. Today the editor shows only a flat glossary list; `suggest-cast` (T0.2) is the only cast-aware affordance. **Reads knowledge directly via gateway** (`/v1/knowledge`, reusing `features/knowledge/`).

## 1. UX

```text
┌─ 👥 Cast & Codex ───────────────  [search…]
│  ▸ char  Kael        swordless, wounded   ⟶ 3 rel
│  ▸ char  Mira        hidden allegiance     ⟶ 2 rel
│  ▸ place The Gate of Ash                   ⟶ 1 rel
│  ▸ fac   The Ashen Pact                    ⟶ 4 rel
│  click an entity → expand: aliases · relations · recent events
└─  source: knowledge entities (project-scoped) · anchored to glossary
```

- Docked right-panel sub-tab listing entities (`GET /entities?project_id=&kind=`), grouped/badged by `kind` (character / location / faction / concept), searchable (`?search=`).
- **Story-state line:** the entity's current state ("swordless, wounded") from the **new `GET /entities/{id}/facts?before_order=` route** (PO: build it now) — the spoiler-windowed attribute ledger. Falls back to recent events when an entity has no facts yet.
- **Expand a row** → aliases, 1-hop relations (`GET /entities/{id}`), recent events.
- **Spoiler-safe:** state/events windowed to `before_order` = the current chapter (no future reveals).
- Empty (no extracted entities) → hint ("publish/extract to populate the codex").

## 2. FE shape

- **hook** `features/composition/hooks/useCast.ts` — `useCast(projectId, token, { kind?, search? })` → `knowledgeApi.listEntities` (reuse `features/knowledge/api.ts`); `useEntityDetail(entityId)` for the expand (1-hop relations); `useEntityFacts(entityId, beforeOrder)` → `knowledgeApi.getEntityFacts` (the **new facts route**, §3) for the attribute-state ledger.
- **component** `features/composition/components/CastCodexPanel.tsx` (view) — list + search + expandable rows. Reuse the CanonRulesPanel list idiom + the `<meter>`/badge conventions.
- **api** — reuse/extend `features/knowledge/api.ts` (`listEntities`, `getEntityDetail`, `listTimeline`). No composition-service proxy.
- **types** — `Entity`, `EntityDetail`, `Relation`, `Event` (import from `features/knowledge/types`).
- **host** — fixed gated right-panel sub-tab in `CompositionPanel` (cross-cutting: docked = sub-tab until T5.4).

## 3. BE delta

**NEW (PO: build now)** — `GET /v1/knowledge/entities/{entity_id}/facts?before_order=&limit=` on **knowledge-service**: the entity's attribute facts (state), **spoiler-windowed** by `before_order`. Reads the existing Neo4j fact store (facts are already extracted; this is the read surface KSA §6.4 slated as a follow-up route). **This route also powers T2.4's state band** — shared Track-2 BE work. Plus reuse `GET /entities`, `/entities/{id}`, `/timeline` (no change).

## 4. Data contract

- `GET /v1/knowledge/entities?project_id=&kind=&search=&limit=&offset=` → `{ entities: Entity[], total }`. `Entity { entity_id|canonical_id, name, canonical_name, aliases[], kind, project_id, version, … }`.
- `GET /v1/knowledge/entities/{entity_id}` → `EntityDetail { entity: Entity, relations: Relation[] /* 1-hop active RELATES_TO */ }` + `ETag`.
- `GET /v1/knowledge/timeline?entity_id=&before_order=&limit=` → `{ events: Event[], total }` (event context).
- **NEW** `GET /v1/knowledge/entities/{entity_id}/facts?before_order=&limit=` → `{ facts: Fact[] }`. `Fact { fact_id, predicate|attribute, value, source_chapter_id, event_order, confidence, … }` (exact shape confirmed at BUILD against the Neo4j fact node) — the spoiler-windowed current-state ledger.
- **project_id resolution:** composition `work` ↔ knowledge `project` — confirm the FE has the knowledge `project_id` for the book (Open Q2).

## 5. Acceptance criteria

- Author sees the book's cast grouped by kind, searchable, with a current-state line + relation count.
- Expanding an entity shows aliases, 1-hop relations, and recent (spoiler-safe) events.
- State/events never reveal beyond the current chapter.
- No entities → a clear "extract first" empty state (not an error).

## 6. Test outline

- **vitest**: list renders grouped by kind; search debounced; expand → detail fetch (relations); state-proxy from recent events; spoiler window applied; empty state.
- **Playwright** `composition-cast-codex.spec.ts`: cast renders for a seeded project; expand shows relations.
- **i18n ×4** keys: `cast.title`, `cast.search`, `cast.kind_character/location/faction/concept`, `cast.relations`, `cast.no_state`, `cast.empty`.

## 7. Decisions (cleared 2026-06-10)

- **State source: build the facts route now (PO)** — `GET /entities/{id}/facts?before_order=` (knowledge-service) is the spoiler-windowed attribute ledger; recent-events is the fallback when an entity has no facts. **Also unblocks T2.4's state band.**
- **Cast source: knowledge entities** (state/relations/events); cross-link the glossary entry for authored edits. (Lead)
- **knowledge `project_id`:** threaded from the work/book resolve. (Lead)
- **Host:** fixed gated right-panel sub-tab in `CompositionPanel`. (Lead)
