# T3.5 — Style & Voice — design spec

**Track** 3 (Write & AI) · **Phase** P2 · **Type** [FS] (mostly FE) · **Depends-on** `BookProfile` / `work.settings` (exists) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` §C **🎚 Style & Voice** (`data-rt="style"`, lines 378–385 — Density/Pace sliders + Voice chips).

> Make the book's prose style **editable**. `BookProfile` (`work.settings`: `voice`, `tone`, `density`, `structure_pref`, `source_language`) already threads into every draft/judge prompt — but the FE only **reads** it (settings view). This adds the editor so the author shapes the co-writer's voice.

## 1. UX

```text
┌─ 🎚 Style & Voice ─────────────────────────
│  Density   lean ◂───●────▸ lush
│  Pace      slow ◂──────●─▸ fast
│  Tone      [ grim ] [ wry ] [ lyrical ] [+]
│  Structure [ scene-driven ▾ ]
│  Voice — narrator
│   "Spare, concrete. Short declaratives under
│    pressure. No purple metaphor."        ✎
│  source language: [ auto ▾ ]
└─  applies to every generate + the critic's voice score
```

- Editable controls mapping to `BookProfile`:
  - **Density** / **Pace** sliders → discrete `density` values (e.g. lean/balanced/lush) (Open Q1 on slider→field mapping).
  - **Tone** chips (grim/wry/lyrical/…) → `tone`.
  - **Structure** select → `structure_pref`.
  - **Voice** free-text → `voice` (the narrator's voice description fed to the drafter).
  - **Source language** select → `source_language`.
- Writes through to `work.settings` (server is the source of truth); the co-writer + critic immediately use the new profile.
- A "reset to neutral" (the NEUTRAL default).

## 2. FE shape

- **hook** `features/composition/hooks/useStyleProfile.ts` — reads `work.settings` (reuse `useWork`) → `BookProfile`; `save(partial)` → work-settings PATCH (write-through; server SSOT, not localStorage).
- **component** `features/composition/components/StyleVoicePanel.tsx` — sliders / chips / selects / textarea bound to the profile; debounced save.
- **api.ts** — reuse the existing work-settings update (composition `PATCH work`/settings). No new endpoint if settings already round-trip these keys (verify — Open Q2).
- **host** — a docked right-panel sub-tab (mockup `data-rt="style"`).

## 3. BE delta

**Likely none** — `BookProfile.from_settings` already reads `voice/tone/density/structure_pref/source_language` from `work.settings`, and the work-settings update path exists. Confirm those keys are **writable** via the public settings update (some settings are server-managed) — if not, widen the allowed-keys (Open Q2). No new prompt wiring (the packer/M6 already thread `BookProfile`).

## 4. Data contract

`work.settings` keys: `voice: string`, `tone: string`, `density: string`, `structure_pref: string`, `source_language: string`. Read via the work payload; write via the work-settings update. Slider positions map to the discrete `density`/pace vocabulary (FE-side mapping).

## 5. Acceptance criteria

- Author edits density/pace/tone/structure/voice/language; changes persist to the server (visible on another device).
- The next generate + the critic's voice score reflect the new profile.
- Reset returns to the neutral default.
- No style setting can force a language/genre when left blank (NEUTRAL safety preserved).

## 6. Test outline

- **vitest**: each control writes the right `work.settings` key (debounced); reset → neutral; slider→discrete-value mapping; write-through (no localStorage-only).
- **BE pytest** (composition, if allowed-keys widened): the settings update accepts the style keys; `from_settings` round-trips them.
- **Playwright** `composition-style-voice.spec.ts`: set a voice → reload → persists → a generate prompt includes it (network-assert or critic reflects).
- **i18n ×4** keys: `style.title`, `style.density`, `style.pace`, `style.tone`, `style.structure`, `style.voice`, `style.language`, `style.reset`.

## 7. Decisions (cleared 2026-06-10)

- **Slider → discrete vocabulary** (lean/balanced/lush; slow/measured/fast) — no schema change. (Lead)
- **Writable keys:** verify the work-settings update permits `voice/tone/density/structure_pref/source_language`; **widen the allowed-keys if not** (the one possible BE delta). (Lead)
- **Book-level narrator voice V1**; per-entity voice deferred. (Lead)
