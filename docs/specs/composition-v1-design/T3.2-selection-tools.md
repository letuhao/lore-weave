# T3.2 — Selection tools (rewrite / expand / describe) — design spec

**Track** 3 (Write & AI) · **Phase** P1 · **Type** [FS] · **Depends-on** `/generate` (exists; needs a selection field) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` §C slash/toolbar affordances (lines 284 `/` commands; the toolbar at 220–226).

> Inline AI tools that act on the **selected prose**: rewrite, expand, describe. Today `/generate` only drafts a whole scene from grounding+guide — there is **no selection-scoped operation**. `GenerateBody.operation` is a free string, but it carries **no input text** to operate on.

## 1. UX

```text
The Gate of Ash rose out of the fog ▓▓▓▓▓▓▓▓▓ ← selection
                       ┌──────────────────────────────┐
                       │ ✦ Rewrite  ✦ Expand  ✦ Describe│  ← floating toolbar
                       └──────────────────────────────┘
   → streams a replacement as a ghost over the selection → [Accept] [Discard]
```

- Select text in the editor → a **floating toolbar** offers **Rewrite** (restyle, same meaning), **Expand** (more detail/length), **Describe** (add sensory/scene detail).
- Choosing one streams a result (reuse `useCompositionStream`) shown as a **ghost over the selection**; Accept replaces the selection, Discard reverts.
- Optional one-line instruction ("rewrite — terser") before running.
- Grounded: the operation includes the scene's grounding pack (so a rewrite stays canon-consistent) + the BookProfile voice (T3.5).
- Disabled when the selection is empty or too long (cap).

## 2. FE shape

- **component** `features/composition/components/SelectionToolbar.tsx` — a Tiptap bubble-menu/floating toolbar bound to the editor selection; buttons per operation; ghost-over-selection + Accept/Discard. Reuse the ComposeView ghost/accept idiom.
- **hook** — extend `useCompositionStream` to accept `{ operation, selection }` (the selected text) and stream a selection-scoped result; or a thin `useSelectionEdit`. Accept = replace the Tiptap range.
- **api.ts** — `generate` body gains `operation` + `selection` (see §3).
- **types.ts** — `SelectionOperation = 'rewrite' | 'expand' | 'describe'`.
- **host** — the manuscript editor (`ChapterEditorPage`/Tiptap), not a side panel.

## 3. BE delta

**NEW (composition-service `/generate`):**
- `GenerateBody` **+ `selection: str | None`** (the text to operate on; capped) **+ `operation`** extended to accept `rewrite|expand|describe` (already a free `str`).
- The engine **dispatches** these operations to dedicated prompt builders (rewrite/expand/describe), each fed `selection` + grounding + BookProfile voice. ⚠ **Register each operation in the dispatch + the prompt map** and **assert the dispatched instruction in tests** — per the `dict.get(key, DEFAULT)`-hides-a-missing-enum lesson (LOOM-39), an unregistered operation must NOT silently fall back to `draft_scene`.
- Returns a streamed replacement (same SSE shape as cowrite). No persistence until Accept (FE replaces the range, then the normal autosave).

## 4. Data contract

`POST /v1/composition/works/{project_id}/generate` (or the SSE variant) · body adds `{ operation: 'rewrite'|'expand'|'describe', selection: string }`. Response = the existing stream (ghost tokens). FE replaces the Tiptap selection range on Accept.

## 5. Acceptance criteria

- Selecting prose offers rewrite/expand/describe; running one streams a grounded, voice-consistent result over the selection.
- Accept replaces exactly the selection; Discard reverts cleanly.
- An unknown/empty operation does **not** silently draft a whole scene (dispatch is explicit).
- Over-long selections are capped with a clear message.

## 6. Test outline

- **vitest**: toolbar appears on selection; each operation calls `/generate` with the right `operation` + `selection`; Accept replaces the range; Discard reverts; empty/over-long disabled.
- **BE pytest** (composition): each operation dispatches its **own** prompt (assert the dispatched instruction, not just a 200) — the regression-lock for the missing-enum class; `selection` is fed into the prompt.
- **Playwright** `composition-selection-tools.spec.ts`: select → Rewrite → ghost → Accept replaces the text.
- **i18n ×4** keys: `sel.rewrite`, `sel.expand`, `sel.describe`, `sel.instruction`, `sel.too_long`, `sel.accept`, `sel.discard`.

## 7. Decisions (cleared 2026-06-10)

- **Selection field + explicit dispatch** — `/generate` gains `selection`; rewrite/expand/describe each get a registered prompt builder + a **test asserting the dispatched instruction** (missing-enum lesson, LOOM-39). (Lead)
- **Grounding:** scene pack for V1. (Lead)
- **Cap:** generous char cap + a clear message. (Lead)
- **Hook:** extend `useCompositionStream` with optional `{operation, selection}`. (Lead)
