# T0.2 — suggest-cast wiring — design spec

**Track** 0 (quick wins) · **Phase** P0 · **Type** [FE] · **Depends-on** none (BE shipped; reuses FD-15 cast-edit path) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` §B **👥 Cast & Codex** (`data-rt="cast"`, lines 295/302/362–369) — the cast roster the suggestion feeds.

> The BE `POST .../suggest-cast` ([engine.py:878](../../../services/composition-service/app/routers/engine.py)) has **zero FE callers** — built-but-hidden debt. Wire a one-click "Suggest cast" into the planner scene row so the model proposes the scene's present entities from its goal/synopsis/title (+ optional guide), and the author accepts/edits.

## 1. UX

```text
┌─ Scene 2 · "The Gate opens"
│  synopsis … … …            tension ▓▓▓▓░░ 62
│  Cast: [Kael ✕] [Mira ✕]   [ + add ▾ ]   [ ✨ Suggest cast ]
│           └→ merges model-proposed ids into present_entity_ids (de-duped)
└─  ✨ disabled when the scene has no persisted node_id
```

Add a **"✨ Suggest cast"** action to the cast-chips row in **`PlannerSceneRow`** (next to the existing `+ add cast` select). On click:
- calls suggest-cast for that scene node → **merges** the returned `suggested_entity_ids` into the row's `present_entity_ids` (set-union, de-duped) via the existing `onEdit` path.
- pending state: button shows a spinner / disabled while in flight.
- merged entities render as the existing removable cast chips (FD-15) — the author removes any unwanted one with the existing ✕.
- **empty result** → subtle toast/hint "No cast suggested" (no chip change).
- **disabled** when the scene has no persisted `node_id` (see Open Q1) — suggest-cast is keyed by a saved scene node.

(Alt surface considered: ComposeView grounding. Chosen: PlannerSceneRow, because that's the existing cast-edit locus and reuses FD-15 verbatim.)

## 2. FE shape (`features/composition/`)

- **hook (controller)** — add a `suggestCast` mutation. **Preferred:** extend `hooks/usePlanner.ts` (it already owns scene-row edits) with `suggestCast({ nodeId, guide? })` → `compositionApi.suggestCast(projectId, nodeId, { guide }, token)` returning `string[]`. (Alt: a tiny standalone `useSuggestCast(projectId, token)` if usePlanner shouldn't grow — decide at DESIGN.)
- **component (view)** `components/PlannerSceneRow.tsx` — add the "Suggest cast" button in the `planner-cast` row; on success call `onEdit({ present_entity_ids: dedupe([...scene.present_entity_ids, ...suggested]) })`. New props: `onSuggestCast: () => void` + `suggesting: boolean` (lifted to `PlannerView`, which owns the mutation + per-row pending id). Labels via `useGlossaryRoster` already in scope (so newly-added ids show names, not raw UUIDs — confirm Open Q2).
- **api.ts** — add `suggestCast(projectId, nodeId, body, token): Promise<{ suggested_entity_ids: string[] }>` (POST, `apiJson`).
- **types.ts** — `SuggestCastBody = { guide?: string }`.
- **wiring** — `PlannerView.tsx` owns `usePlanner().suggestCast`; tracks `suggestingNodeId`; passes `onSuggestCast`/`suggesting` down to each `PlannerSceneRow`.

## 3. BE delta

**None.** Reuse `POST /v1/composition/works/{project_id}/scenes/{node_id}/suggest-cast`. It loads the work+node (ownership + 404), builds a query from `node.goal/synopsis/title + body.guide`, calls `glossary.select_for_context(book_id, user_id, query)`, returns the entity ids. JWT-gated.

## 4. Data contract

**Request:** `POST /v1/composition/works/{project_id}/scenes/{node_id}/suggest-cast` · body `{ "guide"?: string }`.
**Response:** `{ "suggested_entity_ids": string[] }` (entity UUIDs; may be empty; may include ids already in cast → FE de-dupes).
Merge rule (FE): `present_entity_ids = unique(existing ∪ suggested)`. The author's existing chips are never removed by a suggest.

## 5. Acceptance criteria

- One click on a planner scene proposes a cast from the model and **adds** the suggestions to that scene's cast (de-duped, existing chips preserved).
- The author can remove any suggested chip via the existing ✕ (no new removal path).
- Suggesting on a scene with no model-relevant entities → empty result, friendly hint, no chip change.
- The action is disabled/absent when the scene isn't persisted (no `node_id`).
- Errors surface as a toast; the row is unchanged.

## 6. Test outline

- **vitest** (`__tests__/PlannerSceneRow.test.tsx` / `PlannerView.test.tsx`): (a) suggest merges returned ids into `present_entity_ids` via `onEdit`; (b) **de-dupe** — an id already in cast isn't duplicated; (c) empty result → no `onEdit` (or no-op) + hint; (d) disabled when no `node_id`; (e) error → toast, cast unchanged; (f) suggested id renders as a roster label when known.
- **Playwright** `composition-suggest-cast.spec.ts` (Part-B gate): suggest-cast proposes + author accepts into cast (model-gated OR stub the endpoint).
- **i18n ×4** keys: `plan.suggest_cast`, `plan.suggest_cast_pending`, `plan.suggest_cast_empty`. Parity-checked.

## 7. Decisions (cleared 2026-06-10)

- **Gate on persisted `node_id`:** the "Suggest cast" action is enabled **only for committed scenes** (decompose-preview drafts have no node_id). Document the pre-commit limitation. (Lead)
- **Label fallback:** a suggested entity outside `useGlossaryRoster` renders its raw id (existing `labelFor` behavior); verify roster scope (book-wide) at BUILD. (Lead)
- **No `guide` in P0** — query is built from node goal/synopsis/title; passing the ComposeView guide is a later enhancement. (Lead)
