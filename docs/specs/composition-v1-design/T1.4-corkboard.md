> The classic "index cards on a corkboard" planning view: each scene is a card (synopsis), draggable to reorder, grouped by chapter. It is a **card-layout alternative over the same outline nodes** as T1.1 вҖ” so it reuses the outline BE entirely (no new store).

> **Decision (PO 2026-06-10):** Corkboard is the **cards layout of the Outline panel (T1.1)** вҖ” a cards вҮ„ tree toggle over the same `useOutline` nodes, **not a separate tab**. This spec defines the cards layout + drag-reorder; it shares T1.1's host + hook.

- Scenes as **cards** (title + `synopsis` + status dot), grouped by chapter (chapter = a column or a labeled row band).
- **Add card** вҶ’ create a `scene` node under the chapter (`POST /outline/nodes`, `kind='scene'`).
- **Edit card text** вҶ’ patch `synopsis` (and `title`) inline.
- **Reorder** вҶ’ drag вҶ’ patch `rank` (lexorank) / `story_order` (Open Q2).
- **Click card** вҶ’ open the scene in the editor.
- **Archive** card вҶ’ `DELETE /outline/nodes/{id}`.
- **Mobile:** single-column stack per chapter; drag via a handle; 44px targets.

## 2. FE shape (`features/composition/`)

- **hook** вҖ” reuse `useOutline` (nodes + `createNode`/`patchNode`/`archiveNode`). A pure `groupScenesByChapter(nodes)` selector (unit-tested).
- **component** `components/Corkboard.tsx` (view) вҖ” card grid grouped by chapter; add/edit/reorder/archive bubble to the hook. Extract `SceneCard.tsx`. Reuse the dnd approach chosen in T1.1 Q2 (or a simple move-up/down for P2 if DnD is deferred).
- **api.ts / types.ts** вҖ” reuse outline (no new calls/types).
- **host** вҖ” a CompositionPanel sub-tab **or** an alternate view toggle inside the Outline panel (Open Q1).

## 3. BE delta

**None.** Pure reuse of the outline node CRUD ([outline.py](../../../services/composition-service/app/routers/outline.py)): `synopsis` = card text, `rank`/`story_order` = order, `status` = the dot, `kind='scene'`.

## 4. Data contract

Same `OutlineNode` as T1.1. Card text = `synopsis`; order = `rank` (string lexorank) and/or `story_order` (int); group = `chapter_id` / `parent_id` chapter node.

## 5. Acceptance criteria

- Author sees scenes as index cards grouped by chapter, with status.
- Add / edit-synopsis / archive a card persists via the outline node CRUD.
- Reorder persists (order reflects on reload).
- Click a card opens the scene.

## 6. Test outline

- **vitest**: `groupScenesByChapter` selector; `Corkboard` вҖ” cards render grouped; add вҶ’ `createNode(kind='scene')`; edit synopsis вҶ’ `patchNode`; reorder вҶ’ `patchNode(rank/story_order)`; archive.
- **Playwright** `composition-corkboard.spec.ts`: add a card вҶ’ appears; edit synopsis вҶ’ persists; reorder вҶ’ order holds on reload.
- **i18n Г—4** keys: `corkboard.title`, `corkboard.add_card`, `corkboard.empty`, `corkboard.edit`, `corkboard.archive`.

## 7. Decisions (cleared 2026-06-10)

- **Unified with Outline (PO):** Corkboard is the **cards mode** of the T1.1 Outline panel (cards вҮ„ tree toggle, shared `useOutline` + host) вҖ” not a separate tab.
- **Reorder: drag (dnd-kit)** вҶ’ patch `rank` for display order; `story_order` stays the reading/packer axis (recompute on reorder if needed). (PO + Lead)
- **Card scope: scenes only.** (Lead)
