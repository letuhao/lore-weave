# T2.4 — Character Arc — design spec

**Track** 2 (World/Knowledge) · **Phase** P2 · **Type** [FS] · **Depends-on** T2.3 Timeline (shares `entity_id` axis), knowledge `/timeline` + `/entities/{id}` (exist) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Source:** components-catalog (tagged **LATER** — pulled into V1 by PO 2026-06-10). **Not in v3 mockup** → this wireframe is the design.

> One character's journey across the book: their events in narrative order + how their relations/state evolve. It is the **entity-scoped projection of the Timeline** (`GET /timeline?entity_id=X`) plus the entity's relations — a focused arc view rather than the whole-book axis.

## 1. UX

```text
📈 Character Arc — [ Kael ▾ ]
state ▔▔▔╲▁▁▁▁▁▁╱▔▔ (tension proxy · optional)
events ●──────●──────────●────────●▌spoiler-cut
       Ch.9   Ch.11      Ch.12·S2  (≤ current)
       sword  oath       gatekeeper
       breaks broken     warning
relations now: ally→Mira(was: trusted) · member→Ashen Pact
click an event → open chapter · switch character ▾
```

- Pick a character → its **events in `event_order`** (`GET /timeline?entity_id=`) on a compact arc, spoiler-cut at the current chapter.
- **Relations strip:** the character's current 1-hop relations (`GET /entities/{id}`), with "(was: …)" if a superseded/invalidated relation is available (Open Q2).
- Optional **state/tension band** over the arc (a visual proxy from event density / a tension signal — Open Q3).
- Click an event → open its chapter; switch character via the picker.
- Empty (character has no events) → hint.

## 2. FE shape

- **hook** `features/composition/hooks/useCharacterArc.ts` — `useCharacterArc(projectId, entityId, token)` → composes `listTimeline({entity_id})` + `getEntityDetail(entityId)` (reuse `features/knowledge/api.ts`). Pure `buildArc(events, beforeOrder)` selector.
- **component** `features/composition/components/CharacterArcView.tsx` (view) — the arc line (recharts/custom, shared with T2.3), the relations strip, the character picker (reuse `useCast`/roster). Extract `ArcEvent`.
- **api / types** — reuse `features/knowledge` (`listTimeline`, `getEntityDetail`, `Event`, `Relation`).
- **host** — a docked panel **or** a power-view; given it's per-entity and compact, lean a **docked sub-tab** that the Cast codex (T2.1) "view arc" action opens (Open Q4).

## 3. BE delta

**None new** — T2.1 builds the facts route this track. Reuse `/timeline?entity_id=`, `/entities/{id}`, and the **new `GET /entities/{id}/facts`** (T2.1) for the state band. The arc spine is event-based; facts add the state-over-time band.

## 4. Data contract

`GET /v1/knowledge/timeline?project_id=&entity_id=&before_order=&limit=` → `{ events, total }` (the arc spine). `GET /v1/knowledge/entities/{entity_id}` → relations (the strip). Spoiler cutoff = current chapter `event_order` (shared with T2.3).

## 5. Acceptance criteria

- Author picks a character and sees their events in narrative order, spoiler-safe.
- The current relations strip reflects 1-hop relations.
- Clicking an event opens its chapter; switching characters reloads the arc.
- No events for the character → a clear empty state.

## 6. Test outline

- **vitest**: `buildArc` orders + windows events; relations strip from detail; character switch refetch; empty.
- **Playwright** `composition-character-arc.spec.ts`: pick a character → arc renders; spoiler-cut present.
- **i18n ×4** keys: `arc.title`, `arc.pick_character`, `arc.relations_now`, `arc.was`, `arc.empty`.

## 7. Decisions (cleared 2026-06-10)

- **Distinct view (PO)** — Character Arc is its own compact per-entity view, **reusing T2.3's arc/axis component** (not just a Timeline filter).
- **State band:** feasible now via the new `/entities/{id}/facts` route (T2.1) — show a state-over-time band. Relation history "(was:)" stays deferred (needs a historical-relation read). (Lead)
- **Host:** docked sub-tab, launched from the Cast codex (T2.1). (Lead)
