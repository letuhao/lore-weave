# T3.6 — References (comps / influences) — design spec

**Track** 3 (Write & AI) · **Phase** P2 · **Type** [FS] · **Depends-on** none (greenfield) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Source:** components-catalog (tagged **LATER** — pulled into V1 by PO 2026-06-10). **Not in v3 mockup** → this wireframe is the design.

> A place to record the book's **comps / influences** — reference works, authors, or vibes the author is writing toward — and (optionally) feed them as **style hints** into generation. Pairs naturally with Style & Voice (T3.5): T3.5 is the explicit profile, References is the by-example influence.

## 1. UX

```text
┌─ 📚 References ──────────────────────────  [ + add ]
│  • "The Broken Empire" — Lawrence    grim, unreliable narrator   ✎ ✕
│  • Le Guin — Earthsea                restraint, moral weight     ✎ ✕
│  • vibe: "rain-soaked noir fantasy"                              ✎ ✕
│  ☑ use references as style hints in generation
└─  influences the co-writer's tone (by example, not canon)
```

- A simple list of references: each `{ title, note }` (title = a work/author/vibe; note = what to draw from it). Add / edit / remove.
- An **opt-in toggle**: "use references as style hints" → the references are injected as a small style-hint block into the drafter prompt (NOT canon — influence only, sanitized).
- Empty → a hint ("add the books/authors you're writing toward").

## 2. FE shape

- **hook** `features/composition/hooks/useReferences.ts` — CRUD over the work's references (read via `useWork`/a refs endpoint; mutations add/edit/remove + the use-as-hints flag).
- **component** `features/composition/components/ReferencesPanel.tsx` — list + add/edit/remove (reuse the CanonRulesPanel list idiom) + the hints toggle.
- **api.ts** — references CRUD (see §3).
- **types.ts** — `Reference { id, title, note }`.
- **host** — a docked right-panel sub-tab (alongside Style & Voice).

## 3. BE delta

**NEW (composition-service):** a small references store on the work —
- start as a `work.settings.references = [{id, title, note}]` blob + `use_references_as_hints: bool` (no new table; reuse the settings update), promote to a table only if it grows.
- **If hints are enabled**, the packer injects a **sanitized** style-hint block (titles+notes) into the drafter prompt — through the **same neutralize/sanitize path as `<lore>`/`<guide>`** (untrusted author text → prompt boundary; the new-prompt-block-must-reuse-sanitize lesson).

## 4. Data contract

`work.settings.references: { id, title, note }[]` + `use_references_as_hints: bool`. Read via the work payload; write via the work-settings update. When the flag is on, the packer emits a `<references>` style-hint block (sanitized, token-bounded, influence-only — never canon/grounding).

## 5. Acceptance criteria

- Author adds/edits/removes references (title + note); they persist on the work.
- Toggling "use as hints" injects them as a sanitized style block into generation (verified in the prompt, not just stored).
- References never act as canon/grounding (influence only).
- The reference text is sanitized at the prompt boundary (no delimiter injection).

## 6. Test outline

- **vitest**: add/edit/remove references persist; hints toggle; list renders; empty state.
- **BE pytest** (composition): with hints on, the packer emits a **sanitized** `<references>` block (assert the neutralize path ran on a forged-delimiter note); with hints off, no block.
- **Playwright** `composition-references.spec.ts`: add a reference → persists → toggle hints → a generate prompt includes the sanitized block.
- **i18n ×4** keys: `refs.title`, `refs.add`, `refs.title_field`, `refs.note_field`, `refs.use_as_hints`, `refs.empty`, `refs.remove`.

## 7. Decisions (cleared 2026-06-10)

- **Scope (PO): notes + opt-in style hints** — list comps/influences (title+note) + a toggle injecting a **sanitized** `<references>` style block into generation; nothing external/fetched.
- **Store:** `work.settings.references` blob V1. (Lead)
- **Distinct from T3.5** — References (by-example) + Style & Voice (explicit profile) are adjacent panels; both feed the drafter. (Lead)
