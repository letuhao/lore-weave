# T2.5 вҖ” World Map вҖ” design spec

**Track** 2 (World/Knowledge) В· **Phase** P2 В· **Type** [FS] В· **Depends-on** knowledge `/entities?kind=location` + `/entities/{id}` (exist); shares SVG graph primitives with T2.2/T1.3 В· **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Source:** components-catalog (tagged **LATER** вҖ” pulled into V1 by PO 2026-06-10). **Not in v3 mockup** вҶ’ this wireframe is the design. **Highest-unknown task in Track 2.**

> A spatial view of the book's **places** (location entities) and how they connect (contains / borders / route-to relations). No coordinates exist in the data model вҶ’ like the Scene Graph, this is a **hand-rolled SVG graph with author-arranged (per-device) positions**, optionally over a backdrop image.

- Places = `GET /entities?project_id=&kind=location`. Edges = locationвҶ”location relations (`GET /entities/{id}` 1-hop filtered to location neighbors, predicates like contains/borders/route-to).
- **Layout = author-arranged, server-persisted (PO)**: drag places to position; positions saved to a **work-scoped store** (shared across devices, В§3). Auto initial placement (grid / by mention order) until the author arranges.
- **Backdrop image (PO):** upload a hand-drawn / AI-generated map (вҶ’ MinIO); it renders behind the nodes. Stored on the work.
- Click a place вҶ’ open it in the Cast codex (T2.1); hover an edge вҶ’ predicate.
- Group/nest by "contains" (a region encloses its places) вҖ” Open Q3.
- Empty (no location entities) вҶ’ hint to extract/declare places.

## 2. FE shape

- **hook** `features/composition/hooks/useWorldMap.ts` вҖ” `useWorldMap(projectId, token)` вҶ’ `listEntities({kind:'location'})` + per-place `getEntityDetail` for edges (reuse `features/knowledge/api.ts`); **server-persisted positions + backdrop** via composition-service (В§3) вҖ” `savePositions` (debounced) + `uploadBackdrop`. Pure `buildPlaceGraph(places, details)`.
- **component** `features/composition/components/WorldMap.tsx` (view) вҖ” SVG places + edges + optional backdrop; drag-position (dnd-kit or pointer events), click. **Reuse the shared `<GraphCanvas>` primitive** (T2.2/T1.3).
- **api / types** вҖ” reuse `features/knowledge` (location-filtered `Entity`, `Relation`).
- **host** вҖ” **T5.5** power-view overlay (full width); build standalone, graft at T5.5.

## 3. BE delta

**NEW (PO: build now)** вҖ” a work-scoped map store on **composition-service** (the studio owns the canvas; knowledge owns the entities):
- **positions:** persist `{ entityId: {x,y} }` on the work вҖ” start with a `work.settings.world_map.positions` blob (no new table; PATCH via the existing work-settings path); promote to a `world_map` row only if it grows.
- **backdrop:** `POST /v1/composition/works/{project_id}/world-map/backdrop` (image вҶ’ **MinIO, public-read UUID key** per the MinIO-presigned-URL lesson вҖ” static URL, no SigV4 host-rewrite) вҶ’ store the URL on the work.
Plus reuse knowledge `GET /entities?kind=location` + `/entities/{id}` (no change).

## 4. Data contract

`GET /v1/knowledge/entities?project_id=&kind=location` вҶ’ `{ entities, total }`. `GET /v1/knowledge/entities/{id}` вҶ’ relations; keep edges where both endpoints are locations. **Persisted on the composition `work`** (`settings.world_map`): `{ positions: { entityId: {x,y} }, backdrop_url?: string }`. Backdrop upload вҶ’ MinIO public-read static URL.

## 5. Acceptance criteria

- Author sees the book's places and their connections, arrangeable by drag (positions persist per-device).
- Clicking a place opens its codex entry; edges show their predicate.
- No location entities вҶ’ a clear empty/CTA state.
- (If enabled) a backdrop image renders behind the nodes.
