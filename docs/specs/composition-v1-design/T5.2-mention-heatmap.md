# T5.2 — Mention heatmap — design spec

**Track** 5 (Editor shell) · **Phase** P2 · **Type** [FE] · **Depends-on** the existing in-prose mention decoration (exists) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` Grounding pane heatmap (lines 340–344) + the `.mention h1/h2/h3` heat classes in prose (276–279) + "◍ Show heatmap in prose" (344).

> Show which entities dominate the current chapter — a ranked **mention frequency** list, and an in-prose **heat shading** of mentions (hot = frequent). The editor already decorates mentions; this adds the frequency ranking + the heat toggle.

## 1. UX

```text
┌─ ◍ Mention heatmap · this chapter ───────────
│  Kael          ████████████ 11
│  Mira          ███████ 7
│  Gate of Ash   ████ 4
│  [ ◍ Show heatmap in prose ]
└─  in prose: hotter mentions shade darker (h3>h2>h1)
```

- A ranked list of entities mentioned in the current chapter with counts + bars (mockup `.hm`/`.hb`/`.hc`).
- A **"Show heatmap in prose"** toggle: mention decorations shade by frequency band (the existing `h1/h2/h3` classes = heat levels).
- Click an entity → jump to its first mention / open in Cast codex (T2.1).
- Updates as the author writes (re-counts on content change, debounced).

## 2. FE shape

- **hook** `features/composition/hooks/useMentionHeatmap.ts` — derive per-entity mention counts from the editor content (reuse the existing mention decoration's entity mapping; count occurrences per chapter). A pure `countMentions(doc)` selector (unit-tested).
- **component** `features/composition/components/MentionHeatmap.tsx` — the ranked bars + the in-prose toggle (sets a heat class on the editor root so mention marks shade by band).
- **api / types** — none new (mentions already resolved client-side; counts derived).
- **host** — a section in the Grounding sub-tab (mockup places it there) or its own compact panel.

## 3. BE delta

**None** — counts derived from the rendered content + existing mention resolution. (If mention resolution is server-assisted, reuse that; no new endpoint.)

## 4. Data contract

Derived: `{ entityId, label, count, band }[]` from the editor doc's mention marks. Heat band = count quantile → `h1|h2|h3` shading class. No wire types.

## 5. Acceptance criteria

- The author sees the top mentioned entities in the current chapter with counts.
- Toggling in-prose heatmap shades mentions by frequency; toggling off restores plain mentions.
- Clicking an entity navigates to it.
- Counts update as the chapter changes.

## 6. Test outline

- **vitest**: `countMentions` ranks correctly + assigns bands; toggle adds/removes the prose heat class; entity click navigates; updates on content change.
- **Playwright** `composition-mention-heatmap.spec.ts`: heatmap lists top entities; toggle shades the prose.
- **i18n ×4** keys: `heatmap.title`, `heatmap.show_in_prose`, `heatmap.mentions`.

## 7. Decisions (cleared 2026-06-10)

- **FE-derived counts** from the existing mention decoration (no BE). (Lead)
- **Heat bands** = frequency quantiles → the existing `h1/h2/h3` mention classes. (Lead)
- **Host:** a section in the Grounding sub-tab (per mockup) + reuse for the in-prose toggle. (Lead)
- **Click → Cast codex** (T2.1) / first mention. (Lead)
