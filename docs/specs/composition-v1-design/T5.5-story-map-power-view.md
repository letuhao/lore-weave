# T5.5 — Story-Map power-view overlay — design spec

**Track** 5 (Editor shell) · **Phase** P2 · **Type** [FE] · **Depends-on** T1.3 Scene Graph · T2.3 Timeline · T1.2 Beat Sheet (+ T2.2 Relationship Map, T2.5 World Map) · **Status** DRAFT
**Program:** [composition-design-program](../../plans/2026-06-10-composition-design-program.md). **Mockup:** `composition-studio-mockup-v3.html` the `.overlay` power-view (lines 388–441) — full-width, opt-in, with a Scene Graph / Timeline / Beat Sheet switcher.

> The **host shell** for the big structural/world views: a full-width, opt-in overlay ("power view") with a switcher across Scene Graph, Timeline, Beat Sheet (and Relationship Map, World Map). The individual views are designed standalone (their own specs); this is the **frame** that gives them room and a common switcher + "back to editor".

## 1. UX

```text
┌─ 🎬 Scene Graph ─────── opt-in · power view ──────────  [✕ Back to editor]
│  ( 🎬 Scene Graph )( 🕑 Timeline )( ⌗ Beat Sheet )( 🕸 Relations )( 🗺 World )
│  ┌──────────────────────────────────────────────────────────────┐
│  │                                                              │
│  │     « the selected view renders full-width here »            │
│  │       (Scene Graph / Timeline / Beat Sheet / …)              │
│  │                                                              │
│  └──────────────────────────────────────────────────────────────┘
└─  opened from the + picker ("Open as view") or a panel's "expand ⧉"
```

- A full-width overlay above the editor; a **view switcher** across the registered power-views; **✕ Back to editor** restores the writing view.
- Opened from the T5.4 picker ("Open as view") or a docked panel's "expand to view".
- Each view = its own component (Scene Graph T1.3 / Timeline T2.3 / Beat Sheet T1.2 / Relationship Map T2.2 / World Map T2.5), lazy-mounted but **kept alive while the overlay is open** (no remount when switching views — preserve pan/zoom/selection).
- Keyboard: Esc closes; the switcher is reachable.

## 2. FE shape

- **component** `features/composition/components/PowerViewOverlay.tsx` — the overlay frame + switcher + close; renders the active view; mounts views lazily, keeps mounted-while-open (CSS show/hide between views — no-unmount, preserves each view's internal state).
- **integration** — registered in the T5.4 workspace layout as the "view layer" (`openView: ViewId|null`). Each power-view registers a `{ id, title, component }`.
- **hook** — reuse `useWorkspaceLayout().openView` (T5.4) — the overlay reads/sets it.
- **api / types** — none (frame only; views own their data).
- **host** — the workspace shell (T5.4). Until T5.4 lands, a minimal standalone overlay toggle works (the views can be opened directly).

## 3. BE delta

**None** — frame only; each hosted view owns its BE (already specced).

## 4. Data contract

None of its own. `ViewId = 'scene_graph' | 'timeline' | 'beat_sheet' | 'relationship_map' | 'world_map'`; the overlay maps `ViewId → component`. View state (pan/zoom/selection) lives in each view (per-device).

## 5. Acceptance criteria

- The author opens a full-width power-view, switches between Scene Graph / Timeline / Beat Sheet / Relationship Map / World Map, and returns to the editor.
- Switching views preserves each view's internal state (no remount); Esc closes.
- The editor and panels are untouched underneath (re-revealed on close).

## 6. Test outline

- **vitest**: overlay opens/closes; switcher changes the active view without remounting the others (visibility-lifecycle); Esc closes; registered views map correctly.
- **Playwright** `composition-power-view.spec.ts`: open Scene Graph → switch to Timeline → state preserved → Back to editor.
- **i18n ×4** keys: `view.power_view`, `view.back_to_editor`, view titles (reuse each view's title key).

## 7. Decisions (cleared 2026-06-10)

- **Frame only** — hosts the standalone views (T1.2/T1.3/T2.2/T2.3/T2.5); each owns its data + spec. (Lead)
- **Build order:** the hosted views ship standalone first; this overlay grafts them last (it depends on them). (Lead — sequence near the end with T5.4.)
- **No-unmount across view switches** (preserve pan/zoom/selection via CSS show/hide). (Lead)
- **Integrates with T5.4** as the "Open as view" layer. (Lead)
