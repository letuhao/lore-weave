## 4. Stage 3 — climate (climate.rs)

Per-cell `ClimateZone` from latitude × elevation × ocean-distance (Köppen-Geiger-inspired, fully deterministic — no RNG).

**Latitude** — `eff_lat ∈ [0,1]` (0 = equator/hot, 1 = pole/cold), from `cell.center.1` (`y`) and `hemisphere_orientation`:
- `Northern`   → `eff_lat = y`            (top of map = pole)
- `Southern`   → `eff_lat = 1 - y`        (bottom = pole)
- `Equatorial` → `eff_lat = 2 * |y - 0.5|` (equator across the middle)

**Ocean-distance** — multi-source BFS over `neighbors` from every water cell (`elevation < sea_level`); `ocean_dist_norm = min(graph_dist / (grid_side/2), 1.0)`. Deterministic (sources swept in index order, BFS over sorted `neighbors`).

**Classification** (`elev_norm = elevation / 65535`):
1. `elev_norm > 0.62` **and** `eff_lat < 0.75` → **Highland** (altitude dominates — *except* at polar latitudes, where a high cell stays Polar so glaciated peaks remain reachable; design-r1 WARN-1).
2. else `temp = (1 - eff_lat) - 0.35 * elev_norm`; `dry = ocean_dist_norm`:
   - `temp < 0.16` → Polar
   - `temp < 0.34` → Boreal
   - `temp < 0.58` → Temperate
   - `temp < 0.78` → `Mediterranean` if `dry > 0.5` else `Subtropical`
   - else          → `Arid` if `dry > 0.55` else `Tropical`
3. Dry-interior override: if zone ∈ {Temperate, Mediterranean, Subtropical} and `dry > 0.72` and `temp > 0.42` → **Arid**.
4. `climate_bias = Some(z)`: nudge `temp` and `dry` by a fixed per-zone delta toward `z` *before* step 1–3 (e.g. bias `Arid` → `dry += 0.15`; bias `Tropical` → `temp += 0.12`; bias `Polar` → `temp -= 0.15`; …). One `match` mapping each `ClimateZone` to a `(temp_delta, dry_delta)`.

Deterministic: a pure function of stored data + one BFS. No RNG, no `HashMap`.

## 5. Stage 4 — hydrology (hydrology.rs) + biomes (biome.rs)

### 5a. Depression-filled flow elevation (priority-flood)

Steepest-descent flow over a raw heightmap dead-ends in local minima → rivers that never reach the sea. **Priority-flood** fixes this: compute `flow_elev[c] ≥ elevation[c]` such that every land cell has a strictly-downhill path to an ocean cell.

- Min-heap of `(flow_elev, cell)` — entries ordered by `flow_elev` then `cell` index (deterministic tie-break, **not** a raw `BinaryHeap<u16>` whose equal-key order is unspecified).
- A `closed: Vec<bool>` flag. **A cell is marked `closed` when it is first *pushed*, never when popped** (standard Barnes 2014). A closed cell is never pushed again — so every cell is pushed exactly once and every heap entry has a unique `cell`, making the heap fully deterministic (design-r2 WARN-1).
- Seed: every water cell with `flow_elev = elevation`, pushed + closed.
- Pop lowest `(fe, c)`; for each neighbour `n` not `closed`: `flow_elev[n] = max(elevation[n], fe)`; push `(flow_elev[n], n)` + mark `n` closed.
- Result: no land cell is a local minimum in `flow_elev` — every land cell has a strictly-downhill `flow_elev` path to a water cell.

### 5b. River flux (flow accumulation)

- `rainfall[c]` = `1.0 * climate_wetness(climate[c])` — per-climate multiplier (Tropical 1.4, Subtropical 1.2, Temperate 1.0, Boreal 0.9, Highland 0.9, Mediterranean 0.8, Polar 0.5, Arid 0.3).
- Process land cells in **descending `flow_elev`** order (ties by index — deterministic, so the `f32` sum order is pinned). For each cell `c`: `river_flux[c] += rainfall[c]`; let `d` = the neighbour with the lowest `flow_elev` (ties by index); if `flow_elev[d] < flow_elev[c]` → `river_flux[d] += river_flux[c]`.
- `river_threshold` = the 96th percentile of land-cell `river_flux` (self-tuning ⇒ ~4 % of land is river; deterministic).

### 5c. Water network — ocean vs lake (design-r1 BLOCK-2)

- Water cell = `elevation < sea_level`.
- Compute connected components of water cells (flood-fill over water-cell adjacency, using the **same ascending-start-cell sweep + sorted-neighbour DFS** as `terrain.rs::land_components` — design-r2 WARN-3, for cross-routine consistency; component size + border-touch are set-properties so the tag is order-independent, but the sweep order is pinned anyway).
- A water component is **Ocean** (`is_in_ocean = true` for all its cells) if it touches the map border (contains a ring cell) **OR** its size exceeds `LAKE_MAX = max(24, cell_count / 150)`. Otherwise it is a **Lake**.
- *Why the size rule:* `enforce_coherence` submerges non-largest land components into *interior* water; a submerged secondary continent is never border-reachable, so border-only seeding would mis-tag a continent-sized enclosed sea as a `Lake`. Size-gating keeps `Lake` for genuinely small isolated basins and treats any large enclosed water body as `Ocean` (an inland sea). Border-only seeding + interior-submerging coherence were otherwise mutually inconsistent.
- *Accepted tradeoff (design-r2 WARN-3):* a genuinely large naturally-isolated lake above `LAKE_MAX` also reads as an inland-sea `Ocean`. Phase 2 has no author-declared lakes (CreativeSeed declares none), so every large isolated basin is procedural; calling a continent-scale enclosed basin an inland sea is geographically defensible (cf. the Caspian). Accepted as-is for Phase 2.
- *`Lake` coverage (code-review r2 WARN-3):* `Lake` is exercised by `hydrology.rs` `water_network` **unit tests**, not by a full-pipeline structure test. The Azgaar blob+falloff heightmap rarely carves a *small* (`≤ LAKE_MAX`) enclosed sub-sea basin, so generated maps seldom contain a `Lake` cell — accepted for Phase 2. A lake-carving pass (or author-declared lakes) is a later-phase concern.
- `is_coast[c]` = land cell with ≥1 neighbour that is an Ocean water cell.

### 5d. Biome derivation (biome.rs) — the §4.2 matrix

`derive_biome(climate, elevation, sea_level, river_flux, river_threshold, is_water, is_in_ocean, is_coast) -> BiomeKind`, a total deterministic function. `land_t = (elevation - sea_level) / (65535 - sea_level)` tiers: low `<0.22`, mid `<0.55`, high `≥0.55`.

```
water cell        → is_in_ocean ? Ocean : Lake
land, flux > river_threshold              → River
land, is_coast, land_t < 0.06             → Beach
land, is_coast                            → Coast
else by climate:
  Highland     → high? Mountain : Hill
  Polar        → high? Glacier  : Tundra
  Boreal       → high? Mountain : (mid? Hill : Forest)
  Temperate    → high? Mountain : (mid? Hill : Plain)
  Mediterranean→ high? Mountain : (mid? Hill : Plain)
  Subtropical  → high? Mountain : (mid? Hill : (wet_low? Marsh : Forest))
  Tropical     → high? Mountain : (mid? Hill : (wet_low? Marsh : Jungle))
  Arid         → high? Mountain : (mid? Hill : Desert)
```
`wet_low` = `land_t < 0.22 && river_flux > 0.5 * river_threshold` (wet but not a river). `derive_biome` is a **total** function; all 14 `BiomeKind` are reachable across the parameter space (`biome.rs` unit tests exercise each variant by calling `derive_biome` directly with hand-picked inputs — independent of whether one full map happens to surface every variant). Matches the §4.2 hints.

## 6. Connectivity-aware sea level — clears DEFERRED #013 (`DEFER-GEO-P1-1`)

Phase 1 picked `sea_level` by elevation percentile, *then* `enforce_coherence` submerged non-largest components — so realised land fraction drifted (Inland 0.23 vs 0.70). Phase 2 replaces the percentile pick:

- **non-Archipelago:** binary-search `sea_level ∈ [8192, 57344]` so the **largest land connected component** ≈ `coastline_profile.land_fraction() * cell_count`. After `enforce_coherence` keeps only that component, realised land = the largest component — so targeting the largest component *is* targeting the realised fraction.

  *Monotonicity (binary-search validity — design-r1 BLOCK-1):* let `L(s) = {cell : elevation ≥ s}`; `s↑ ⇒ L(s)` shrinks. Any connected component `C` of `L(s₂)` (`s₂ > s₁`) is, since `L(s₂) ⊆ L(s₁)` over the *same* induced graph, also connected in `L(s₁)` → `C` lies within one component of `L(s₁)` → `|C| ≤ maxcomp(L(s₁))`. Hence `maxcomp(L(s))` is **monotone non-increasing** in `s` (the *identity* of the largest component may change as an isthmus floods, but its *size* never rises) → binary search converges (~16 iterations, each an O(count) component pass).

  *Unreachable target (best-effort):* if even `s = 8192` yields `maxcomp < target` (a heightmap too fragmented to ever form a component that big), the search floors at 8192 and returns it — the largest achievable continent. This is a documented best-effort outcome, not a failure; criterion #7 is phrased to accept it.
- **Archipelago:** unchanged — the 5 fixed island discs already define the structure; keep the percentile pick.

Deterministic (binary search over `u16`, deterministic component pass).

## 8. Acceptance criteria (VERIFY gate)

1. `cargo build` + `cargo clippy --all-targets` clean.
2. **Determinism:** `generate` byte-identical across two runs (hash now covers climate/biome/river_flux/is_coast) — ≥4 fixture seeds.
3. **Climate valid:** every cell has a `ClimateZone`; `Highland` ⇒ `elev_norm > 0.62`; hemisphere flip is correct (Northern vs Southern mirror).
4. **Biome valid:** every cell has a `BiomeKind`; water cells are Ocean/Lake; Ocean ⇔ `is_in_ocean`; every land cell with `river_flux > river_threshold` is River; biome matches `derive_biome` for its inputs.
5. **Rivers descend (no land dead-end):** every `River` cell has ≥1 neighbour that is an Ocean/Lake/River biome **or** has strictly greater `river_flux`. This proves no `River` cell is a flux local-maximum stranded on land — checkable from the `WorldMap` alone (biome + river_flux + neighbors). Combined with the §5a priority-flood (every land cell drains downhill to a water cell), it means every river reaches the sea. *(design-r2 WARN-2 — phrased to be WorldMap-checkable without storing the hydrology-intermediate `flow_elev`.)*
6. **Biome patch coherence:** biomes form contiguous patches, not speckle — **≥ 85 %** of land cells have ≥1 neighbour of the same `BiomeKind`. A tolerant structural check (not a hard per-pair adjacency table — a steep elevation/latitude gradient can legitimately abut contrasting biomes, so a zero-tolerance rule would fail valid maps). *(design-r1 WARN-1 — the earlier "GEO_001 HIGH-1" citation was wrong: HIGH-1 is the `SetBiomeOverride` water-transition delta rule, not an adjacency constraint.)*
7. **Land fraction:** for non-Archipelago, realised land fraction ≥ `coastline_profile.land_fraction() − 0.08` **or** `sea_level == 8192` (the documented best-effort floor — §6). Clears DEFERRED #013.
8. CLI end-to-end → JSON + biome PNG; PNG flagged for human morning review.
