## BE (knowledge-service) — 2 thin endpoints, NO new Cypher
Both **reuse existing, user-scoped repo functions**:
- `POST /v1/knowledge/entities` `{project_id, name, kind}` → `merge_entity(user_id, project_id, name, kind, source_type='manual', confidence=1.0, provenance='human_authored')` → `Entity`. Idempotent (canonical_id dedup) — a later extraction of the same place converges on the same node. `kind` constrained to `{character,location,faction,concept}`.
- `POST /v1/knowledge/relations` `{subject_id, object_id, predicate}` → `recreate_relation(user_id, subject_id, predicate, object_id)` → `Relation`; `None` (endpoint missing / cross-user) → **409**. Idempotent; confidence 1.0, pending_validation false.
- Multi-tenant: `user_id` from JWT threaded into Cypher (verified in both repo fns). No project-ownership check (entities are user-scoped → no cross-tenant leak).
- Gateway: catch-all `/v1/knowledge/*` proxy → no gateway change.

## FE specifics
- `buildPlaceGraph(places, details)` (pure): nodes = locations; edges = relations where BOTH endpoints ∈ location-id set; dedup by relation id. Reuse the `GraphEdge` shape + `RelationEdge` (define a `relmap-arrow` marker in WorldMap defs).
- Positions: seed from `work.settings.world_map.positions`, fall back to auto grid layout; drag → local; drag-**end** → persist via `useSetWorkSettings` (`patch:{ world_map:{...wm, positions} }`).
- Backdrop: `uploadChapterMedia(token, bookId, chapterId, file)` → store `backdrop_url` on `work.settings.world_map`; render as a `<image pointer-events:none>` via the GraphCanvas `background` slot.
- Add-place: text input → `createPlace(name)` → invalidate places query. Link-places: pick-2 nodes (SceneGraph pattern) + predicate select (`contains`/`borders`/`route_to`) → `linkPlaces` → invalidate details.
- Click place → `onViewCast(name)` (CompositionPanel sets castSearch + tab='cast').
- Empty: no locations → CTA hint ("extract this book or add a place").
