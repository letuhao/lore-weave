# RAID v1.6 — portability fix

## Problem

RAID v1.5 hardcodes `docs/plans/2026-05-29-foundation-mega-task/` in **5 script files** and **1 prompt template**. Switching branch → paths break. No single "task config" file declares which task is active.

## Solution

Introduce per-branch task config + Python loader; refactor 5 files to consume it.

### New files (2)

1. **`.raid/active-task.yaml`** — committed per branch. Declares:
   ```yaml
   task_id: foundation-mega-task
   task_slug: 2026-05-29-foundation-mega-task
   plan_dir: docs/plans/2026-05-29-foundation-mega-task
   workflow_doc: docs/plans/2026-05-29-foundation-mega-task/RAID_WORKFLOW.md
   decomposition_doc: docs/plans/2026-05-29-foundation-mega-task/CYCLE_DECOMPOSITION.md
   locked_qs_doc: docs/plans/2026-05-29-foundation-mega-task/OPEN_QUESTIONS_LOCKED.md
   pre_flight_doc: docs/plans/2026-05-29-foundation-mega-task/PRE_FLIGHT_CHECKLIST.md
   brief_dir: docs/raid/cycle_briefs
   cycle_log: docs/raid/CYCLE_LOG.md
   cycle_count: 38
   ```

2. **`scripts/raid/task-config.py`** — loader CLI:
   - `path <key>` → emits resolved absolute path (or relative, configurable)
   - `get <key>` → emits raw value
   - `dump` → emits full config as JSON
   - `validate` → asserts all declared paths exist (used by startup-verifier)
   - Exit code: 0 OK, 2 missing key, 3 missing config file

### Refactored files (5)

| File | Change |
|------|--------|
| `scripts/raid/brief-generator.py` | Replace `PLANS_DIR = REPO_ROOT / "docs"/"plans"/"2026-05-29-foundation-mega-task"` → `from task_config import load_config; PLANS_DIR = load_config()["plan_dir"]` |
| `scripts/raid/startup-verifier.sh` | Replace `PLANS_DIR="$REPO_ROOT/docs/plans/2026-05-29-foundation-mega-task"` → `PLANS_DIR="$REPO_ROOT/$(python "$SCRIPT_DIR/task-config.py" get plan_dir)"` |
| `scripts/raid/recovery-protocol-runner.sh` | Same swap as above |
| `scripts/raid/cycle-runner-prompt.md` | Replace literal `docs/plans/2026-05-29-foundation-mega-task/OPEN_QUESTIONS_LOCKED.md` → `<LOCKED_QS_DOC>` placeholder + add interpolation note. Coordinator reads `.raid/active-task.yaml` and substitutes. |
| `.claude/commands/raid.md` | Add Step 0 "Read `.raid/active-task.yaml` via `python scripts/raid/task-config.py dump`" before LOOP. All path references use config values, not literals. |

### Why a Python loader instead of pure bash

PowerShell users + git-bash users + WSL — Python is the lowest common denominator already used by every RAID script. Bash YAML parsing on Windows is fragile (no `yq` shipped). One Python file with stdlib-only YAML... actually `pyyaml` already required (used by `in-progress-state-writer.py`). Confirmed safe.

## Out of scope

- Not refactoring docs (CYCLE_DECOMPOSITION, BEGIN_CYCLE_0, SESSION_PATCH, etc.) — those are task-specific by nature; keep hardcoded.
- Not adding multi-task support (running 2 RAID tasks concurrently in same repo) — defer to v1.7 if needed.
- Not changing `.raid/active-task.yaml` schema to support nested layer plans — Coordinator can derive from `plan_dir + "/layer_NN_*.md"` glob.
