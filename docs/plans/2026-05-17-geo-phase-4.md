## 1. Goal & scope

Finalize the generator: a stable `WorldMap` ⇄ JSON round-trip, SVG export, the full CLI, and **optional LLM CreativeSeed authoring** (a prose brief → schema-valid `CreativeSeed`).

**In scope:** JSON round-trip stability test + `CreativeSeed` JSON loading; `render::political_svg`; CLI restructured into `generate` / `author` subcommands with `--config` + `--svg`; `author.rs` — LLM authoring via a direct OpenAI-compatible HTTP call; `reqwest` dependency.

**Out of scope / deferred:** polygon-accurate SVG (no Voronoi polygons are stored — a Phase 1 decision); the `loreweave_llm` platform SDK (the standalone generator stays decoupled per GEO_GENERATOR_PLAN §1 — a direct HTTP call instead).

**Invariant:** regeneration-determinism — `generate(seed, cs)` stays a pure function. LLM authoring is non-deterministic, but it only produces the *input* `CreativeSeed`; once a `CreativeSeed` exists, generation is byte-identical as always. JSON round-trip must be **identity**.

## 2. JSON round-trip

`WorldMap`, `CreativeSeed`, and every nested type already `derive(Serialize, Deserialize)` (Phases 1–3). Phase 4 adds **no new data types** — only the guarantee, a hash-verification method, and tests:

- `serde_json::to_string_pretty(&map)` → `serde_json::from_str::<WorldMap>` must be **`==`** the original. `content_hash: [u8;32]` serializes as a JSON array, enums as their variant name → both round-trip.
- **Hash verification (design-r1 BLOCK-1).** `from_str` deserializes `content_hash` straight from the bytes — it is never recomputed, so a hand-edited `WorldMap` JSON would carry a *lying* hash and a `from_str == original` test proves nothing about it. Phase 4 moves the hash logic to `WorldMap::compute_hash(&self) -> [u8;32]` (in `world_map.rs`) and adds `WorldMap::verify_hash(&self) -> bool` (`compute_hash() == content_hash`). **`compute_hash` hashes every `WorldMap` field *except* `content_hash` itself** — folding the digest field into its own digest is circular and would make `verify_hash` permanently false (design-r2 BLOCK-2). `generate()` is rewritten to call `map.content_hash = map.compute_hash()` and the old free `content_hash` fn + its MAINTENANCE doc-comment are deleted from `lib.rs` (only `lib.rs` itself referenced the fn; `main.rs` reads the *field*). Criterion #2 asserts the *loaded* map's `verify_hash()` is true (recompute-after-load), not merely struct equality.
- **`f32` round-trip is identity only for finite values (design-r1 WARN-1).** `serde_json` formats finite `f32` via a shortest-round-trip algorithm (parses back identical), but serializes `NaN`/`±Inf` as JSON `null`, which then fails to deserialize into a non-`Option` `f32`. `river_flux` (a bounded sum of finite per-climate rainfall) and `cell.center` (jittered-grid coordinates) are always finite — `generate()` carries a `debug_assert!` that every `river_flux`/`center` value `is_finite()`, and §2's identity guarantee is explicitly conditioned on it.
- `CreativeSeed` loads from a JSON file (`--config`). `climate_bias: Option<ClimateZone>` ⇒ JSON `null` or a variant string.
- *Coupling note (design-r3 WARN-2):* `verify_hash()`-after-load is identity *because* `serde_json` round-trips a finite `f32` to the bit-identical value — `compute_hash` hashes `f32` bit-patterns (`to_le_bytes`) while JSON stores decimal text, so the check is coupled to the `serde_json` float formatter. If the wire format ever changes (compact binary, a different JSON lib) this must be re-verified.

## 3. SVG export — `render::political_svg`

A vector political map as an SVG string:
- `<svg width=S height=S>` + an ocean-blue background `<rect>`.
- Each cell → a small `<rect>` (side ≈ `S / grid_side`, centred on the cell) filled by `political_cell_color` (ocean cells skipped — the background shows through). A cell-grid approximation of the Voronoi partition (polygon-exact fill is deferred — §1).
- Routes → `<line>` coloured by `RouteKind`.
- Settlements → `<circle>` (radius by `population_tier`) coloured by role.
- Deterministic: cells/routes/settlements emitted in `Vec` order; coordinates formatted with fixed precision.

Polygon-exact SVG would need stored Voronoi polygons; the cell-`<rect>` approximation is the honest Phase-4 deliverable.

## 4. CLI — clap subcommands (`main.rs`)

```
world-gen generate --seed S [--config cs.json] [--scale ..] [--archetype ..]
                   [--coastline ..] [--hemisphere ..] [--climate-bias ..]
                   [--settlement-density ..] [--culture-count N]
                   --out map.json [--png p.png] [--political-png pp.png] [--svg m.svg]
world-gen author --brief "<prose>" --out creative_seed.json
                 [--llm-url http://localhost:1234/v1] [--llm-model ibm/granite-4-h-tiny]
```

- `generate`: if `--config <path>` is given, the `CreativeSeed` is loaded from that JSON file and the individual creative flags are ignored; otherwise the flags build it (Phases 1–3 behaviour). `--svg` writes `political_svg`.
- `author`: calls the LLM, writes the resulting `CreativeSeed` JSON to `--out`. The user then runs `generate --config` on it.
- The top level is a required subcommand — the CLI invocation gains a `generate`/`author` verb (a deliberate finalization of the Phase 1–3 flag-only CLI).

## 5. LLM authoring — `author.rs`

```rust
/// Parse + validate an LLM `content` string into a CreativeSeed.
pub fn parse_creative_seed(content: &str) -> Result<CreativeSeed, String>;

/// Request a CreativeSeed from an OpenAI-compatible chat endpoint.
pub fn request_creative_seed(brief: &str, llm_url: &str, model: &str)
    -> Result<CreativeSeed, String>;

/// The JSON Schema constraining the LLM output to the CreativeSeed shape.
fn creative_seed_schema() -> serde_json::Value;
```

- `creative_seed_schema()` — a JSON Schema, `additionalProperties: false`, every key **required** (including `climate_bias`):
  - `world_scale`/`coastline_profile`/`hemisphere_orientation`/`settlement_density` — `enum` of the exact **serde variant names**.
  - `world_archetype` — `enum` of the serde variant names **minus `Custom`** (design-r1 WARN-2: `Custom` is an author-declares-everything placeholder nothing branches on — the LLM must pick a real archetype).
  - `climate_bias` — `enum` of the 8 `ClimateZone` variant names **plus JSON `null`** (the encoding is pinned: serde's `Option<ClimateZone>` accepts an explicit `null` → `None` and a variant string → `Some`; the key is required so schema and serde agree exactly).
  - `culture_count` — `integer`, `minimum: 1`, `maximum: 16`.
  - The enum value lists are hand-mirrored from the Rust enums — `creative_seed_schema()` carries a `MAINTENANCE:` comment naming the source enums (`WorldScale`, `WorldArchetype`, `CoastlineProfile`, `HemisphereOrientation`, `SettlementDensity`, `ClimateZone`), and a unit test (criterion #6) asserts every schema enum string deserializes into its Rust enum so a stale/bogus entry is caught (design-r3 WARN-3).
- `request_creative_seed`: a `reqwest::blocking` POST to `{llm_url}/chat/completions` — system prompt (explains the field meanings), user message = `brief`, `response_format: {type: "json_schema", json_schema: {name, strict, schema}}`. Then a strict **error contract** (design-r2 WARN-4) — **no `unwrap()` or `[idx]` on the response `Value`**, navigate with `.get()`/`.as_array()`/`.as_str()`:
  1. `send()` / connection failure (LM Studio not running — the common case) → `Err` with the reqwest message.
  2. non-success HTTP status → `Err("LLM HTTP {status}: {body snippet}")` (a non-2xx body is an error envelope with no `choices`).
  3. missing/empty `choices`, or absent/`null` `message.content` → `Err("LLM response had no message content")`.
  4. otherwise hand the `content` string to `parse_creative_seed`.
- `parse_creative_seed`: `serde_json::from_str::<CreativeSeed>(content)` — serde rejects unknown enum variants / missing fields, so a successful parse is **structural** validation. It then **clamps `culture_count` to `1..=16`** so the stored `CreativeSeed` can never carry an absurd value (the schema bounds it too — defence in depth; design-r1 WARN-2).
- Determinism note: the LLM call is non-deterministic; the *generator* downstream is not affected — `generate(seed, parsed_creative_seed)` is as pure as ever.

**Default LLM:** `ibm/granite-4-h-tiny` at `http://localhost:1234/v1` (LM Studio, OpenAI-compatible) — verified during the run's pre-flight to emit clean `json_schema`-constrained `content`.

## 7. Acceptance criteria (VERIFY gate)

1. `cargo build` + `cargo clippy --all-targets` clean.
2. **JSON round-trip identity:** `generate → to_string_pretty → from_str` is `==` the original `WorldMap`, **and** the loaded map's `verify_hash()` is `true` (recompute-after-load, not just struct equality — design-r1 BLOCK-1), for ≥3 fixture seeds × ≥2 scales incl. a heavy-hydrology seed (river_flux finiteness — design-r1 WARN-1).
3. **CreativeSeed JSON:** a `CreativeSeed` → JSON → `from_str` is `==` the original (incl. `climate_bias: Some` and `None`).
4. **CLI `generate --config`:** loading a `CreativeSeed` JSON file yields a `CreativeSeed` `==` (direct `Eq`) the one the equivalent flags build, **and** `generate()` on it yields a `WorldMap` with an identical `content_hash` (design-r2 WARN-3 — "same map" alone is unverifiable since `world_archetype` is unobserved); end-to-end emits JSON + PNG + SVG.
5. **SVG:** `political_svg` output starts with `<svg` and ends with `</svg>`, is non-empty, contains `<line` (route) and `<circle` (settlement) elements.
6. **`author::parse_creative_seed`:** parses a valid fixture JSON into the expected `CreativeSeed`; rejects malformed JSON and an unknown enum variant with an `Err`; clamps an out-of-range `culture_count`. **Error envelope:** a canned non-2xx / no-`choices` LLM response JSON is rejected with a clear `Err` (testable without a live endpoint — design-r2 WARN-4). **Schema drift:** every enum string in `creative_seed_schema()` deserializes into its Rust enum (design-r3 WARN-3).
7. **LLM (manual / `#[ignore]`):** an integration test hits LM Studio if running; the run's morning review can exercise `world-gen author` manually.

## 8. Build order

1. `Cargo.toml` — add `reqwest` (blocking).
2. `world_map.rs` — add `WorldMap::compute_hash()` (hashes all fields **except** `content_hash`) + `verify_hash()`; carry the hash-MAINTENANCE doc-comment here, and add to it (design-r3 WARN-1): "`verify_hash` proves only produce==verify *agreement*, NOT field-list completeness — if a new field is added but `compute_hash` is not extended, `verify_hash` still passes; completeness is guaranteed only by `determinism.rs`'s full-`PartialEq` assertion."
3. `lib.rs` — delete the old free `content_hash` fn; rewrite `generate()` to call `map.content_hash = map.compute_hash()`; add `pub mod author;` + a `pub use author::…`.
4. `author.rs` — `creative_seed_schema`, `parse_creative_seed` (parse + clamp), `request_creative_seed` (the §5 error contract).
5. `render.rs` — `political_svg`.
6. `main.rs` — restructure into `generate` / `author` subcommands; `--config`, `--svg`; refresh the `#[command(about=…)]` string.
7. Tests: JSON round-trip + `verify_hash`-after-load + finiteness (`tests/serde.rs`); `author` parse/validate + error-envelope unit tests; one `#[ignore]` LLM integration test.
8. VERIFY: build, clippy, test, CLI `generate --config` + `--svg` smoke run.
