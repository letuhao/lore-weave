@router.post("/arcs/{node_id}/extract-template", status_code=201)
async def extract_arc_template(
    node_id: UUID,
    body: ArcExtractTemplateBody,
    user_id: UUID = Depends(get_current_user),
    grant: GrantClient = Depends(get_grant_client_dep),

    structures = _structures()
    node = await _gate_arc(structures, grant, user_id, node_id, GrantLevel.VIEW)

            get_pool(), arc_node=node, owner_user_id=user_id,
            code=body.code, name=body.name,
            language=body.language, visibility=body.visibility,
        )
    except asyncpg.UniqueViolationError:
        raise HTTPException(status_code=409, detail={
            "code": "ARC_TEMPLATE_DUPLICATE",
            "message": "an arc template with this code + language already exists in your library",

THREE adjudications the builder must not get wrong:

(1) 409 — the spec's phrase "mirror the MCP handler verbatim including the UniqueViolationError→409 map" is SELF-CONTRADICTORY and must NOT be taken literally. The MCP handler does NOT map to 409; it returns a 200-style envelope {"success": False, "outcome": "applied_conflict"} (server.py:4663-4667) because MCP has no HTTP status. Mirroring it verbatim would ship a 200 on duplicate, violating the spec's own "409 on duplicate" error contract. Mirror the handler's LOGIC, not its return envelope: catch asyncpg.UniqueViolationError → raise HTTPException(409). This is what the engine itself mandates (arc_apply.py:660-663: "raises asyncpg.UniqueViolationError, which the caller maps to a 409 — this seam does not swallow it"). Do NOT add a try/except inside the engine.

(2) Gate is VIEW, not EDIT — mirrors the MCP handler (server.py:4648 _arc_or_deny(..., GrantLevel.VIEW)): you only READ the arc to extract it, and the new template is owner-stamped to the caller in their own always-writable library. The adjacent delete_arc/restore_arc use GrantLevel.EDIT, so copy-pasting the wrong sibling silently over-gates the route. _gate_arc (arc.py:383) already delivers the spec's "404 H13 uniform, VIEW derived from the ROW": missing node and denied grant BOTH return the uniform NOT_ACCESSIBLE_MESSAGE 404 (no existence oracle); under-tier → 403.

(3) Do NOT re-map the response — the engine already returns the spec's exact shape verbatim ({success, outcome:'extracted', template_id, member_chapter_node_ids[], layout_placements, pacing_chapters}, arc_apply.py:674-681). Return it as-is. Unlike the MCP handler, do NOT append the "_meta"/undo_hint key (that is an MCP-envelope concern, not REST).
