## BE (composition-service) — new selection-scoped path, decoupled from outline_node_id
- `POST /v1/composition/works/{project_id}/selection-edit` (SSE, same token stream as cowrite).
- `SelectionEditBody { operation: Literal['rewrite','expand','describe'], selection: str (Field max_length=SELECTION_MAX_CHARS → 422 backstop), scene_context?: UUID, model_source, model_ref, guide?, reasoning?, max_output_tokens, model_kind?, model_name? }`. The **Literal** makes a bad op a 422 at validation.
- `cowrite.build_selection_messages(selection, profile, operation, guide, grounding="")` — **explicit dispatch**: `_SELECTION_INSTRUCTIONS[op]`; `if op not in _SELECTION_INSTRUCTIONS: raise ValueError` (NO `draft_scene` fallback — the LOOM-39 `dict.get(key,DEFAULT)`-hides-a-missing-enum lesson). "Output ONLY the revised passage" + voice/lang from BookProfile. The SELECTED passage is in the user message.
- Endpoint: `from_settings(work.settings)` → profile; if `scene_context` resolves to a valid node → `pack()` its grounding (degrade to voice-only on pack failure); `build_selection_messages`; budget pre-check (413); reasoning resolve; `jobs.create(operation=op, outline_node_id=scene_context|None, mode='cowrite')`; stream via `stream_draft`; SSE `job`→`token…`→`done`.
- **Test (pytest):** each op dispatches its OWN instruction; `selection` is in the prompt; an unknown op raises (regression-lock for the missing-enum class).

## SelectionToolbar logic
- `shouldShow`: selection non-empty AND ≤ cap, OR a ghost is pending (stay open through Accept/Discard).
- on op-click: capture `range = {from,to}` + the selected text; `start({projectId, selection, operation, sceneContext, modelSource:'user_model', modelRef, guide})`.
- ghost preview popover (streaming text) + **Accept** → bounds-check `range.to ≤ doc.content.size`; valid → `editor.chain().focus().deleteRange(range).insertContentAt(range.from, ghost).run()`; stale → toast `sel.stale` + abort. **Discard** → clearGhost + reset.
- cap: selected text length > `SELECTION_MAX_CHARS` → ops disabled + `sel.too_long`.
- gated on `projectId` (no co-writer work → toolbar not rendered).
