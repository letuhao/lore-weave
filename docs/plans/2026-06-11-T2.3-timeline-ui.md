## PO decisions (CLARIFY 2026-06-11)

The spec pre-cleared recharts-first + FE-stride-compute; the PO **overrode both** at the CLARIFY checkpoint:

1. **Render = hand-rolled SVG** — a 1-axis line + event points + cutoff marker. Consistent with the T1.3 SceneGraph / T2.2 RelationshipMap SVG components; no recharts surface; full control over the spoiler-cut + label layout (the spec anticipated this fallback).
2. **Cutoff = decoupled via `before_chapter_id`** — NO `EVENT_ORDER_CHAPTER_STRIDE` constant on the FE. This **eliminates the MED-3 coupling risk** the spec flagged (FE drifting from the BE's internal `event_order` scheme).

## Decoupled-cutoff mechanism (verified against BE)

- `GET /v1/knowledge/timeline` orders `coalesce(event_order, INT64_MAX) ASC, title, id` (total, deterministic) and returns `total` = the filtered count **before** skip/limit, **including** the `before_order` filter that `before_chapter_id` resolves into (`events.py` `_LIST_EVENTS_COUNT_CYPHER`).
- ∴ the visible set (`event_order < cutoff`) is an **ascending prefix** of the full list under identical other-filters. So:
  - **main query** (default): `listTimeline({...filters, limit, offset})` → full axis page + grand `total`.
  - **cutoff-count query**: `listTimeline({...filters, before_chapter_id: chapterId, limit:1})` → `total` = `visibleCount`.
  - event at global index `offset+i` is **hidden/future** iff `offset+i >= visibleCount`.
  - `visibleOnPage = clamp(visibleCount - offset, 0, pageLen)` drives both the dim set and the marker slot — **pagination-safe**, no stride.
- **hide-spoilers toggle ON** → pass `before_chapter_id` to the *main* query (BE filters hidden out); cutoff-count query disabled; no marker.
- **No `getChapter`/`sort_order` dependency** — the BE resolves the window from `chapterId`.

### Documented edges
- Unresolvable chapter (fail-closed → `before_order=-1`) and legitimately-chapter-1 both yield `visibleCount=0` (all dimmed). Indistinguishable from `total` alone; both are **safe/fail-closed**, marker sits at the start.
- NULL-`event_order` events sink to the tail (INT64_MAX sentinel) and are excluded from the windowed count → rendered as hidden/future. Consistent.
