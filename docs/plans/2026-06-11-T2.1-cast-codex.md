# T2.1 ‚Äî Cast & Codex (per-entity story-state) ‚Äî plan

**Size:** XL (~14 files, [FS], load-bearing: touches knowledge-service extraction write path). **Branch:** `feat/composition-service`.
**Spec:** `docs/specs/composition-v1-design/T2.1-cast-codex-story-state.md`. **/review-impl (design):** 2 HIGH + 5 MED/LOW folded (below).

## Goal
The book's cast (characters/places/factions) as a docked codex sub-tab ‚Äî each row shows its **spoiler-safe current story-state** (`active|gone` from `:EntityStatus`) + 1-hop relation count; expand ‚Üí aliases, relations, recent events, **known facts**. Reads knowledge directly via the gateway (no composition-service proxy).

## Spoiler axis (PINNED ‚Äî /review-impl HIGH-2)
- `event_order = chapter_sort_order √ó EVENT_ORDER_CHAPTER_STRIDE + within_chapter_idx` (pass2_writer.py:552-571).
- **Window ceiling** = `(chapter_sort_order + 1) √ó STRIDE ‚àí 1` ‚Üí inclusive-through-the-current-chapter (matches spec "= the current chapter; no future reveals"). Base-agnostic (works for 0- or 1-based sort_order).
- **Resolver** `resolve_before_order(book_client, chapter_id) -> (before_order:int, available:bool)`:
  - `sort_order = (await book_client.get_chapter_sort_orders([chapter_id])).get(chapter_id)`.
  - resolved ‚Üí `((sort_order+1)*STRIDE - 1, True)`.
  - **FAIL-CLOSED** (book-service `{}` / chapter unknown) ‚Üí `(-1, False)` (restrictive: status defaults `active`, no events, no facts) + surface `window_available:false`. NB inverts `get_chapter_sort_orders`' fail-OPEN posture.
- Shared helper used by status / facts / timeline.

## BE ‚Äî knowledge-service

1. **`facts.py`**
   - `merge_fact(..., subject_id: str | None = None, from_order: int | None = None)`: when `subject_id` given, MERGE `(f)-[:ABOUT]->(:Entity {id:subject_id})` (idempotent); set `f.from_order = from_order` on create. Fact `id` stays **content-keyed** (unchanged idempotency).
   - `Fact` model gains `from_order: int | None = None`; `subject_id` is NOT on the node (it's the ABOUT edge).
   - `list_facts_for_entity(session, *, user_id, entity_id, before_order, min_confidence=0.8, exclude_pending=True, include_archived=False, limit=100)`: `MATCH (f:Fact)-[:ABOUT]->(e:Entity {id, user_id})` + L2-loader filters (`/review-impl LOW-6`) + `f.from_order <= before_order` (NULL from_order excluded under any finite window ‚Üí legacy facts hidden until re-extract). Returns `list[Fact]`.

2. **`entity_status.py`** ‚Äî extend `status_at_order` to also return `from_order`: `{entity_id: {status, from_order}}` (it already does `head(collect(s))`; surface `latest.from_order`). Keep the no-silent-drop default `active` (from_order null).

3. **`pass2_writer.py`** ‚Äî stop dropping `fact.subject_id` (lines 721-726): validate `fact.subject_id ‚àà merged_entity_ids` (mirror relations; unresolved ‚Üí keep the fact, skip the link), pass `subject_id` + `from_order=chapter_base` to `merge_fact`.

4. **`entities.py` (repo) `merge_entities`** ‚Äî add an ABOUT-transfer step BEFORE the step-7 DETACH DELETE source (`/review-impl MED-3`): `MATCH (f:Fact)-[:ABOUT]->(s {id:source}) MERGE (f)-[:ABOUT]->(t {id:target})`. Else facts orphan to a merged-away entity.

5. **Routes (public/entities.py + public/timeline.py)** ‚Äî reuse JWT user-scoping + `get_book_client` dep:
   - `GET /v1/knowledge/entities/statuses?project_id=&before_chapter_id=&kind=` ‚Üí resolve window, list the project's entities (optionally by kind), batch `status_at_order`, return `{statuses:{id:{status,from_order}}, window_available}`. **Project-scoped** (not client `entity_ids` ‚Äî `/review-impl MED-4`, URL-length).
   - `GET /v1/knowledge/entities/{id}/facts?before_chapter_id=` ‚Üí resolve window, `list_facts_for_entity(id, before_order)`, `{facts, window_available}`.
   - Extend `GET /v1/knowledge/timeline` to accept `before_chapter_id` ‚Üí resolve ‚Üí `before_order` (explicit `before_order` wins ‚Äî `/review-impl LOW-7`).

6. **`neo4j_schema.cypher`** ‚Äî document the `(:Fact)-[:ABOUT]->(:Entity)` edge; index `Fact(user_id, from_order)` (mirror Event order index).

7. **BE tests** ‚Äî resolver incl. **fail-closed** path; `merge_fact` ABOUT idempotency + `from_order`; `list_facts_for_entity` window + filters; `merge_entities` ABOUT-transfer; `status_at_order` from_order; route shapes.

## FE ‚Äî frontend (features/composition reuses features/knowledge)
- **api** `knowledgeApi.getEntityStatuses(params)` + `getEntityFacts(entityId, {beforeChapterId})`; thread `before_chapter_id` into `listTimeline`. **types** `EntityStatus`, `Fact` (mirror BE), windowed responses.
- **hook** `useCast(projectId, token, {kind?, search?})` ‚Äî `listEntities` + a batch `useEntityStatuses` keyed by `(projectId, beforeChapterId, kind)`; `useEntityDetail`/`useEntityFacts`/events for the expanded row.
- **component** `CastCodexPanel.tsx` ‚Äî grouped/searchable list, `active|gone` state line + relation count; expand ‚Üí aliases, 1-hop relations, recent events, known facts; `window_available:false` hint; empty state ("extract first").
- **host** `CompositionPanel` ‚Äî `cast` sub-tab (pass `chapterId` as the window).
- **i18n √ó4** `cast.*`; **vitest** (grouped list, search, expand‚Üídetail, status active|gone, facts, window hint, empty).

## /review-impl folded
- HIGH-1 facts spoiler leak ‚Üí facts windowed by `from_order` (stamped at merge from `source_chapter`/`chapter_base`).
- HIGH-2 resolver fail-closed + pinned formula (above).
- MED-3 ABOUT transfers on entity merge. MED-4 project-scoped batch status. MED-5 content-keyed fact id + multi-subject conflation = **documented/accepted** (one node, ‚â•1 ABOUT edge).
- LOW-6 facts filters mirror L2 loader. LOW-7 timeline before_order precedence.

## Caveats (recorded ‚Äî no silent seam)
- Existing facts have NULL `from_order` + no ABOUT edge ‚Üí **won't appear under entities until re-extraction**; unrecoverable from a stored `:Fact` alone.
- Same-wording facts with different subjects collapse to one node with multiple ABOUT edges (content-keyed id).
- Chat-tool facts (`executor.py`) stay subject-less (no codex link) ‚Äî acceptable V1.

Foundation = BE (repo + extraction + merge + routes + BE tests) ‚Üí green/commit-able. Integration = FE. Offer a checkpoint at the seam.
