# LOOM T5.4 — Dock/Float/Pop-out Windowing (plan)

Spec: `docs/specs/2026-06-25-loom-t5.4-dock-float-windowing.md` · XL · pure FE · 4 milestones.

## Milestone M1 — Foundation (live-state hoist + layout scaffold + flag) ← FIRST commit
The load-bearing refactor. Everything else builds on it; ships safe behind the default-OFF flag.
- `features/composition/context/LiveStateContext.tsx` (NEW) — a provider mounted ABOVE the panel
  layer that owns `useCompositionStream(token)` (co-writer) once; exposes it via context. (Chat SSE
  hoist follows in the same provider if it's a hook; if CoWriterChat owns its own EventSource-like
  stream, hoist its controller too — verify during build.)
- `features/composition/context/WorkspaceLayoutContext.tsx` (NEW) — `{ enabled, layout, dispatch }`;
  layout = `{ panels: Record<PanelId, {placement:'dock'|'float'|'popout', order, rect?}> }`; reducer
  + localStorage load/save (validated, default-on-corruption); the feature flag (per-device
  localStorage `loom.workspace.enabled`).
- `features/composition/workspace/types.ts` (NEW) — `Placement`, `WorkspacePanelId` (the 19 ids),
  `WorkspaceLayout`, default layout.
- `features/composition/components/workspace/WorkspaceShell.tsx` (NEW) — when the flag is OFF, render
  the existing `CompositionPanel` UNCHANGED (fallback). When ON (M1), still render CompositionPanel
  but wrapped in the providers (no visual change yet) — proving the hoist + providers don't regress.
- `ComposeView.tsx` — consume the co-writer stream from `LiveStateContext` instead of calling
  `useCompositionStream` locally (the hoist's first consumer; the ghost/accept flow unchanged).
- `CompositionPanel.tsx` — read the hoisted stream where it currently owns it (thread through), so
  ComposeView + the panel agree on one stream instance.
- **Tests:** LiveStateContext (the stream survives a provider re-render / a consumer remount —
  mount-count of the OWNER stays 1); WorkspaceLayoutContext (flag default OFF, layout persists +
  validates, corrupt localStorage → default); WorkspaceShell (OFF → CompositionPanel rendered
  unchanged; ON → wrapped, panels still all mounted). tsc 0.
- **Risk boundary:** the hoist. Commit M1 (with spec+plan) once the co-writer still streams + the
  no-remount invariant holds.

## Milestone M2 — Dock rail
- `components/workspace/DockRail.tsx` (dnd-kit reorder, show/hide), `ComponentPicker.tsx` (+menu).
- WorkspaceShell renders docked panels from layout order; flag-ON replaces the fixed strip.
- Tests: reorder updates layout + persists; hidden panel stays MOUNTED (CSS-hidden, no remount).

## Milestone M3 — In-app float
- `components/workspace/FloatingWindow.tsx` (portal to body, drag/resize), dock↔float actions.
- A floated Compose keeps streaming (thin view over the hoisted stream).
- Tests: dock→float→dock keeps mount-count 1; a stream mid-flight survives the move.

## Milestone M4 — OS pop-out (hardest)
- `components/workspace/PopoutBridge.tsx` (`window.open` + portal to the popped document),
  `workers/liveState.shared-worker.ts` (owns the co-writer/chat fetch+SSE; fans tokens to all ports).
- BroadcastChannel syncs layout/active-scene across windows; SharedWorker survives opener close.
- Feature-detect SharedWorker → degrade to opener-owned BroadcastChannel relay (loses survive-close).
- Tests: pop-out renders the panel in the new document (mock window.open); SharedWorker message
  protocol (token fan-out, port add/drop, abort per port); degrade path when SharedWorker absent.

## Cross-cutting
- Every milestone keeps the no-remount invariant (mount-counter tests, the CompositionPanel pattern).
- Flag OFF ⇒ byte-identical to today (the fixed-strip fallback is the contract).
- localStorage is per-device UI state (CLAUDE.md-allowed); server sync deferred (`D-T5.4-SERVER-SYNC`).
- i18n: window chrome labels (dock/float/pop-out/close/add-panel) ×4 locales.

## Deferred (pre-noted)
