# LOOM T3.6 — References (plan)

**Date:** 2026-06-25 · **Branch:** `feat/composition-service` · **Size:** L (reclassified from M at CLARIFY) · **Type:** FS

## Goal
A docked **References** subtab in the composition studio: the author curates **reference
material** (external influences/passages, with source attribution), and per scene the panel
surfaces the **semantically-relevant** ones, each **pin/exclude**-able into the grounding pack.

## CLARIFY decisions (PO-approved 2026-06-25)
- **Boundary (the key call):** references are **composition-owned authoring data**, NOT knowledge-graph
  content. composition embeds them by calling **provider-registry `/internal/embed` directly** (the
  provider-gateway invariant is satisfied by hitting provider-registry, not by routing through
  knowledge). knowledge-service is **untouched** — it stays the knowledge graph, not a generic
  embedding bucket. The only shared thing is the embedding *capability* (provider-registry).
- **Storage/search:** a per-Work `reference_source` table; the vector is a plain `real[]` column;
  retrieval is **brute-force cosine top-K** in Python (a reference shelf is dozens–low-hundreds of
  rows — no pgvector extension, no ivfflat index, no fixed-dimension pinning).
- **Embedding model:** persisted in `composition_work.settings.reference_embed_model_ref` /
  `reference_embed_model_source` (server-SSOT, multi-device). Set write-through on the FIRST add;
  reused for every add + search so all vectors share one space. Unset → add 422
  `REFERENCE_EMBED_MODEL_UNSET`, search returns empty + a flag.
- **Query seed:** auto from the scene (synopsis/beat-goal/cast — same `query` the lore lens builds)
  **plus** a manual search box.
- **Scope:** per-Work (`project_id`), like canon_rule / style_profile.
- **Pins:** extend `scene_grounding_pins.item_type` + `GroundingItemType` with `'reference'`;
  honored in `pack()` exactly like `lore` (id = `reference_source.id`).

## Architecture seam
```
FE ReferencesPanel ─PUT grounding-pins (reuse T3.4)─┐
   │  add/list/delete/search                        │
   ▼                                                 ▼
composition references.py router ── ReferencesRepo (cosine) ── reference_source table
   │  embed content + query                          ▲
   ▼                                                 │ gather_references (auto query)
provider-registry /internal/embed          packer pack.py → assemble <references> block
```

## Backend (composition-service)
1. **`app/db/migrate.py`** — `reference_source` table:
   `(id uuidv7 PK, user_id, project_id, title, author, source_url, content, embedding real[],
   embedding_model text, embedding_dim int, created_at)`; indexes on `(user_id, project_id)`.
   Extend `scene_grounding_pins` CHECK to `('present','canon','lore','reference')` via an
   idempotent DO-block (DROP + ADD the named constraint, like the `outline_beatrole_kind` example).
2. **`app/db/models.py`** — `ReferenceSource` Pydantic model; extend
   `GroundingItemType = Literal["present","canon","lore","reference"]`.
3. **`app/clients/embedding_client.py`** (NEW) — copy of knowledge's: `POST /internal/embed`
   (`llm_gateway_internal_url` + `X-Internal-Token`), `embed(user_id, model_source, model_ref, texts)`
   → `EmbeddingResult{embeddings, dimension, model, prompt_tokens}`; `EmbeddingError(retryable)`.
4. **`app/db/repositories/references.py`** (NEW) — `ReferencesRepo`: `create` (stores vector),
   `list`, `get`, `delete` (hard — no critic history to preserve), `search` (load the Work's rows,
   cosine vs the query vector in Python, return top-K with score). All `user_id`-first + `project_id`
   filtered.
5. **`app/routers/references.py`** (NEW) — `/v1/composition`:
   `GET  /works/{pid}/references` (list), `POST /works/{pid}/references` (embed+store; resolves/sets the
   Work embed model write-through; 422 if model unset/unavailable), `DELETE /references/{id}`,
   `GET  /works/{pid}/scenes/{nid}/references?q=` (auto query from the node when `q` omitted; embed →
   cosine top-K → hits with attribution + score + the scene's pin/exclude state). Work-ownership 404s.
6. **`app/deps.py`** — `get_references_repo`, `get_embedding_client_dep`.
7. **`app/packer/lenses.py`** — `gather_references(refs_repo, embed, *, user_id, project_id, query,
   model)` → `(hits, seen)`; degrade-safe ([] on any failure / unset model / empty query).
   `LensBundle.references: list[dict]`.
8. **`app/packer/pack.py`** — thread `references_repo` + `embedding_client` (gated, dormant when
   unwired); call `gather_references` with the auto `query`; inject into the bundle; honor `'reference'`
   pins in `_apply_grounding_pins` (id = reference id; pinned → protected).
9. **`app/packer/assemble.py`** — render `<references>` (sanitized via `sanitize_lore`, a new
   `PRIO_REFERENCES` just below lore); add to `_BLOCK_ORDER`; protected when pinned.
10. **`app/routers/grounding.py`** + **`app/routers/engine.py`** — thread the new repo+client into
    both `pack()` call-sites (preview == generation).
11. **register** the references router (main app / `routers/__init__`).

## Invariants honored
- Provider-gateway: every embed call goes through provider-registry; no provider SDK in composition.
- No hardcoded model: the embed model is resolved from Work settings (a user_model ref), never literal.
- Tenancy: `reference_source` carries `user_id` + `project_id`; every query filters both (per-Work tier).
- Degrade-safe packer: a references failure thins the pack, never 500s a generate (the `_safe_*` posture).
