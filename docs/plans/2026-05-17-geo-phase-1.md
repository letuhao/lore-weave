## 1. Goal & scope

Build the `world-gen` Rust crate: a library `world_gen` exposing `generate(seed, &CreativeSeed) -> WorldMap`, plus a thin CLI bin `world-gen`. Phase 1 delivers **stages 1–2** of the 8-stage pipeline: Voronoi dual-mesh + heightmap.

**In scope:** crate scaffold; `CreativeSeed` (geometry fields) + enums; `WorldMap` + `Cell`; deterministic mesh (cell centers + adjacency); heightmap (u16 elevation) + sea-level threshold; CLI that dumps a land/sea PNG + JSON; determinism + structure tests.

**Out of scope (later phases):** climate, biomes, rivers (P2); political/settlement/route/culture (P3); JSON round-trip-stability guarantee, SVG, LLM authoring (P4). Also dropped permanently per GEO_GENERATOR_PLAN §1: event-sourcing aggregate, GeographyDelta, validator pipeline, schema_version, snapshot-fork.

**The one invariant:** regeneration-determinism — `generate(seed, cs)` is a pure function; same inputs → byte-identical `WorldMap` (same binary, same platform; cross-platform f32 determinism is a documented later concern per GEO_001 D-S04-3).

## 3. Core types

```rust
// creative_seed.rs --------------------------------------------------------
pub struct CreativeSeed {
    pub world_scale: WorldScale,
    pub world_archetype: WorldArchetype,
    pub coastline_profile: CoastlineProfile,
}
pub enum WorldScale { Pocket, Region, Continent, SuperContinent, Megaplanet }
impl WorldScale {
    /// Grid side `g = round(sqrt(target))` for the GEO_001 §6 target cell count.
    /// The mesh is a perimeter ring + a (g-2)x(g-2) jittered interior (§4).
    pub fn grid_side(self) -> usize { /* 32 / 45 / 91 / 111 / 128 */ }
    /// Exact, deterministic total cell count = (g-2)^2 + 4*(g-1). See §4 table.
    pub fn cell_count(self) -> usize { let g = self.grid_side(); (g-2)*(g-2) + 4*(g-1) }
}
pub enum WorldArchetype { Wuxia, HighFantasy, LowFantasy, Cyberpunk, SteamPunk,
    Postapocalyptic, ScienceFiction, Historical, Mythological, Romance, Mystery, Custom }
pub enum CoastlineProfile { Island, Peninsula, Coastal, Inland, Archipelago }

// world_map.rs ------------------------------------------------------------
pub struct WorldMap {
    pub seed: u64,
    pub scale: WorldScale,
    pub cells: Vec<Cell>,            // index == cell id
    pub neighbors: Vec<Vec<u32>>,    // neighbors[i] sorted-ascending, deduped; parallel to cells
    pub sea_level: u16,              // elevation < sea_level => water
    pub content_hash: [u8; 32],      // blake3 over the canonical byte view (see §6)
}
impl WorldMap {
    pub fn is_land(&self, cell: usize) -> bool { self.cells[cell].elevation >= self.sea_level }
}
pub struct Cell {
    pub center: (f32, f32),          // normalized [0,1]^2
    pub elevation: u16,              // 0..=65535
}
```

`seed: u64` is a **separate generate() argument**, not a CreativeSeed field — CreativeSeed is creative direction, seed is the RNG seed (`generate(seed, &CreativeSeed)` per GEO_GENERATOR_PLAN). `WorldMap` stores `seed` for provenance.

No `Cell.polygon` in Phase 1: the GEO_GENERATOR_PLAN Phase 1 deliverable does not list it, the land/sea image is produced by rasterization (§5), and correct hull-cell polygon clipping is a real bug surface deferred to Phase 4 ("image/SVG export"). The Voronoi *partition* is fully represented by centers + Delaunay adjacency (the dual mesh); polygons are a rendering projection.

All public types derive `serde::{Serialize, Deserialize}` + `Clone` + `Debug` + `PartialEq`. JSON round-trip *stability as a guarantee* is Phase 4; the derive is free now and lets the CLI dump JSON.

## 4. Stage 1 — mesh (mesh.rs)

**Point placement: perimeter ring + jittered interior grid** (the Azgaar / Red-Blob boundary-points technique). CLARIFY named Poisson-disk; DESIGN overrides — a jittered grid gives exact deterministic cell count, and the surrounding ring is what makes neighbour-degree provable.

**Why the ring (resolves r1 Finding 1):** a bare jittered grid's convex-hull cells have unbounded Voronoi cells and unprovable low degree. Surrounding the interior with a closed ring of points fixes the interior cells *by construction*:
- every **interior** point is now strictly inside the hull → its incident triangles form a closed fan → degree ≥ 3 (a closed fan is ≥3 triangles);
- every **non-corner ring** point has 2 ring-neighbours (its consecutive hull edges) plus the interior point(s) it faces → degree ≥ 3.

The **4 ring corners** are the one residual case (r2 Finding A): a corner is a convex-hull vertex possibly incident to a single Delaunay triangle `(corner, ring-nbr-1, ring-nbr-2)` → degree 2. They are fixed by a deterministic **degree repair** (step 7), not asserted away. Upper bound: jittered interior points reach ~8–9, ring points ~4–6 — all well under 12 (the structure test still asserts `≤ 12`).

**One formula (resolves r1 Finding 2):** `g = scale.grid_side()`. The mesh = a `(g-2)×(g-2)` jittered interior grid + a closed perimeter ring of `4*(g-1)` evenly-spaced un-jittered points. Total cell count is exact and deterministic:

| Scale | GEO_001 target | `g` = round(√target) | interior `(g-2)²` | ring `4(g-1)` | **`cell_count`** |
|---|---|---|---|---|---|
| Pocket | 1024 | 32 | 900 | 124 | **1024** |
| Region | 2048 | 45 | 1849 | 176 | **2025** |
| Continent | 8192 | 91 | 7921 | 360 | **8281** |
| SuperContinent | 12288 | 111 | 11881 | 440 | **12321** |
| Megaplanet | 16384 | 128 | 15876 | 508 | **16384** |

All five totals are within `[1024, 16384]`. Acceptance criterion #3 asserts `count == scale.cell_count()` **exactly** — no tolerance needed, it is a pure function of `scale`.

The GEO_001 §6 cell-count figures are **approximate targets** ("~N cells"); `cell_count()` is the exact value the ring + rounding produce. The only material deviation is Region (2025 vs ~2048, −1.1%, the ring/rounding remainder) — accepted as within "~" guidance, not a defect (r3 Finding D).

Algorithm:
1. `g = scale.grid_side()`. RNG = `rng_for(seed, b"mesh")`.
2. **Interior:** for grid slot `(i,j)`, `i,j ∈ 0..g-2`, place a point at normalized
   `( (i+1 + 0.5 + jx) / g , (j+1 + 0.5 + jy) / g )` with `jx,jy ∈ [-0.5,0.5) * JITTER`, `JITTER = 0.55`.
   The `+1` offset keeps the interior grid inside the ring; `JITTER ≤ 0.55` keeps interior points off the border.
3. **Ring:** `4*(g-1)` points evenly spaced (un-jittered) around the perimeter of `[0,1]²` — `g-1` points per edge, corners shared, walked in a fixed order (bottom edge L→R, right edge B→T, top R→L, left T→B).
4. **Cell ordering:** `cells = interior points (row-major) ++ ring points (perimeter order)`. Index = cell id. The ordering is fixed → deterministic.
5. Delaunay-triangulate all centres via `delaunator::triangulate`.
6. **Adjacency:** for every triangle `(a,b,c)` in `triangulation.triangles`, add edges a–b, b–c, c–a into `Vec<Vec<u32>>`; then **sort + dedup each cell's list** (a `Vec`+sort+dedup, NOT a `HashSet` — `HashSet` iteration order is non-deterministic). Result: `neighbors[i]` sorted ascending, symmetric.
   - `delaunator` exposes `triangles` as `Vec<usize>`. The `usize → u32` narrowing happens at **one site** here, written as a single deliberate cast carrying `#[allow(clippy::cast_possible_truncation)]` + a comment: cell indices are provably `≤ 16388 ≪ u32::MAX` (§4 table). Keeps acceptance criterion #1 (clippy-clean) green (r3 Finding E).
7. **Degree repair (deterministic; resolves r2 Finding A):** sweep cells in **ascending cell-index order** (a plain `for i in 0..count` linear sweep — never a `HashSet`-collected set, whose order would diverge `content_hash` per r3 Finding F). Any cell with `degree < 3` — in practice only the ≤4 convex-hull corners — gets edges added to its nearest non-neighbour cell ("nearest" = squared Euclidean centre distance, ties broken by lower cell index), adding the symmetric reverse edge each time, until `degree == 3`. Re-sort+dedup the touched lists afterward. Pure function of deterministic inputs (centres + existing adjacency) → preserves the determinism invariant; touches only hull-corner cells, never raises any cell above degree ~7.

Determinism: `delaunator` is a pure function of the input point slice; `ChaCha8Rng` is deterministic; ring points are fixed; adjacency uses sort/dedup not hashing. No `HashMap` anywhere in the pipeline. (Perfectly-regular ring points create cocircular sets, but `delaunator`'s tie-break is a fixed deterministic rule → identical output for identical input.)

## 5. Stage 2 — heightmap (terrain.rs)

Work in `f32` `[0,1]`, convert to `u16` at the end. RNG: `rng_for(seed, b"terrain")`.

1. **Blob seeds — exact algorithm (Azgaar "add hill/range"; resolves r2 Finding B).** `k = clamp(count / 380, 6, 40)`. All RNG draws come from `rng_for(seed, b"terrain")` consumed in exactly this order. For each blob `b` in `0..k`:
   - a. Draw 4 `f32` uniforms in `[0,1)`: `ux, uy, ua, uf` (in that order).
   - b. Blob centre `(bx,by)` is **profile-conditional** (code-review r1 BLOCK-1 — emergent fragmentation is not reliable, so it is made structural):
     - non-Archipelago — center-biased: `(0.5 + (ux-0.5)*0.70, 0.5 + (uy-0.5)*0.70)` ⇒ one coherent mass.
     - Archipelago — the blob is assigned to island `b mod 5` and seeded *inside that island's disc*: `angle = ux*τ`, `r = uy*ARCH_RADIUS*0.6`, `(bx,by) = island + (r·cos, r·sin)`.
   - c. Seed cell = the cell whose centre is nearest `(bx,by)` (squared Euclidean; ties → lower index).
   - d. Peak amplitude `a = 0.45 + ua*0.55`. Per-blob falloff `fb = 0.82 + uf*0.08`, **constant for the whole blob** (no per-ring jitter — keeps the output fully pinned).
   - e. BFS from the seed cell over `neighbors` (FIFO queue, `visited: Vec<bool>`, neighbours iterated in their stored sorted order). A cell first reached at BFS ring `r` contributes `a * fb^r`; stop expanding a cell once `a * fb^r < 0.02`.
   - f. Accumulate the contribution into `elevation[cell]` via **max** (overlapping ranges combine cleanly; no runaway summation).
2. **Radial falloff (coastline_profile):** multiply each cell elevation by a mask:
   - `Island`   — `mask = 1 - (d/R)^2` clamped ≥0, `R≈0.55`, centred (strong: edges → sea).
   - `Peninsula`— falloff strong on 3 edges, open on one.
   - `Coastal`  — falloff on one edge only.
   - `Inland`   — mild uniform mask (`0.85..1.0`), little forced sea.
   - `Archipelago` — 5 fixed island discs (`ARCH_ISLANDS`, pairwise separation ≥ 0.353) each masked `1-(d/ARCH_RADIUS)²` clamped ≥0, `ARCH_RADIUS = 0.15`, **max-combined**. Because `2·ARCH_RADIUS = 0.30 <` the island separation, the mask is **exactly 0** in the sea bands between discs → the islands are always separate components.
   `d` = distance from cell center to the mask centre(s).
3. **Normalize** elevations to `[0,1]`, then `elevation_u16 = (v * 65535.0).round() as u16`.
4. **Sea level by target land fraction:** land-fraction target from coastline_profile (`Island 0.38, Peninsula 0.46, Coastal 0.55, Inland 0.70, Archipelago 0.32`). `sea_level` = the `(1 - land_fraction)` percentile of the sorted `u16` elevation list. Deterministic; clamped to GEO_001's `[8192, 57344]` band. (`pick_sea_level` guards a `len < 2` mesh so the percentile index cannot panic — code-review r1 WARN-1.)
5. **Enforce land coherence (code-review r1 BLOCK-1)** — criterion #5 is a *universal* property, so it is made structural, not emergent:
   - non-Archipelago — connected-components flood-fill over land cells; submerge (set to `sea_level-1`) every component except the largest ⇒ exactly one continent (largest = 100% of land ≥ 85%).
   - Archipelago — left as-is; the radial mask already guarantees ≥3 separate island components, each ≪ 60% of land.
   The structure test (`land_coherence_per_profile`) sweeps many seeds × every profile × every scale, since one fixture seed/scale is no evidence for a universal claim.

`WorldArchetype` is stored on `CreativeSeed` but **does not branch the Phase 1 heightmap** — archetype-conditioned terrain is a later concern; keeping P1 logic minimal avoids scope creep.

**Deferred to Phase 2 (code-review r2 Finding A) — `DEFER-GEO-P1-1`:** `sea_level` is picked *before* `enforce_coherence` submerges the non-largest components, so the *realised* land fraction drifts below `coastline_profile.land_fraction()` (worst observed: Inland 0.23 vs 0.70 target). Coherence (criterion #5) still holds; only land-fraction *precision* drifts, and that was never a gated criterion. A correct fix is a connectivity-aware sea-level search, which is entangled with Phase 2's river/water-network and biome work (biomes need a sensible land fraction) — so it is deferred to Phase 2, not patched here.

## 6. Determinism & content hash

`WorldMap.content_hash` = `blake3` over a canonical byte view, appended in fixed order:
`seed` (LE bytes) · `scale` (discriminant u8) · `sea_level` (LE) · for each cell in index order: `center.0`/`center.1` as `f32::to_le_bytes`, `elevation` LE · for each `neighbors[i]` in index order: length then each u32 LE.

Hashing the f32 **bit pattern** (`to_le_bytes`) sidesteps any float-formatting ambiguity. The determinism test runs `generate` twice and asserts (a) equal `content_hash`, (b) full `WorldMap` `PartialEq`.

## 7. CLI (main.rs) — `clap`

```
world-gen --seed <u64> --scale <pocket|region|continent|supercontinent|megaplanet>
          [--archetype <...>] [--coastline <island|peninsula|coastal|inland|archipelago>]
          --out <map.json> [--png <map.png>]
```
Defaults: `archetype=high-fantasy`, `coastline=coastal`. `--out` writes `serde_json` pretty WorldMap. `--png` (optional) writes the land/sea raster.

**Land/sea raster (render.rs):** `W×H` (default 1024×1024). Build a spatial-hash bucket grid from cell centers (≈`sqrt(count)` buckets/axis). Per pixel → normalized `(x,y)` → nearest cell center via the 3×3 bucket neighbourhood → colour: water = blue ramp by depth below `sea_level`; land = green→brown→white ramp by elevation above `sea_level`. This *is* the Voronoi diagram (nearest-center partition); no polygon needed.

## 9. Acceptance criteria (Phase 6 VERIFY gate)

1. `cargo build -p world-gen` + `cargo clippy -p world-gen` clean.
2. **Determinism:** `cargo test -p world-gen` — `generate` byte-identical across two runs for ≥3 fixture seeds × ≥2 scales.
3. **Cell count** equals `scale.cell_count()` **exactly** for every `WorldScale` (1024/2025/8281/12321/16384) — a pure function of `scale`, no tolerance.
4. **Neighbour degree** ∈ `[3, 12]` for every cell (≥3 guaranteed by the ring construction, §4); `neighbors` symmetric (i∈neighbors[j] ⇔ j∈neighbors[i]); `neighbors[i]` sorted + deduped.
5. **Land coherence (resolves design-r1 Finding 3 / design-r2 Finding C / code-r2 Findings B,C):** a connected-components flood-fill over land cells (using `neighbors`), machine-checked in `structure.rs` across a seed sweep × **every `WorldScale`**:
   - `coastline_profile ∈ {Island, Peninsula, Coastal, Inland}` → largest land component **≥ 85%** of all land cells (`enforce_coherence` makes it 100% by construction — one dominant continent).
   - `Archipelago` → **exactly 5** land components (the fixed `ARCH_ISLANDS` discs) **and** largest land component **< 60%** of land cells.
6. `sea_level` within `[8192, 57344]`.
7. CLI runs end-to-end on a fixture seed; emits JSON + a land/sea PNG. The PNG is flagged for **human morning review** (the loop cannot judge "looks like a continent"); criterion #5 is the machine guarantee that backs it.
