# LOOM T3.4 — Grounding Pin / Exclude (plan)

> Spec: `docs/specs/2026-06-24-loom-t3.4-grounding-pin-exclude.md` · Size **XL** · /loom v2.2
> Verified against code 2026-06-24: `CanonRule.id: UUID` ✓ · lore hit has `source_id`+`chapter_index`,
> **no per-chunk id** → lore addressed at `source_id` granularity (deduped per source).

## Build order (foundation → integration → FE) — natural mid-BUILD checkpoint at the ▸ seam

### Slice A — persistence foundation (isolated, no integration)
1. **Migration** `scene_grounding_pins` (composition-service `db/migrate.py`, idempotent DO-block).
2. **Model** `SceneGroundingPin` in `db/models.py`.
3. **Repo** `db/repositories/grounding_pins.py`:
   - `list_for_scene(user_id, project_id, outline_node_id) -> list[SceneGroundingPin]`
   - `set_action(user_id, project_id, outline_node_id, item_type, item_id, action)` — upsert on the
     UNIQUE key (flip pin⇄exclude in place).
   - `clear(user_id, project_id, outline_node_id, item_type, item_id)` — delete.
4. **Tests** `tests/integration/db/test_repositories.py` — upsert/flip/clear + tenancy isolation + real-PG.

### ▸ CHECKPOINT (offer): Slice A is committable on its own (schema+repo, no behavior change).

### Slice B — pack() integration (the behavior)
5. **`pack()`** (`packer/pack.py`): new optional `grounding_pins_repo` dep. After spoiler filters,
   before `build_segments`:
   - read the scene set (keyed on `node['id']`), build `{('present',id)…}` exclude + pin sets;
   - drop excluded items from `bundle.present/canon/lore`;
   - collect pinned `(type,id)` → pass to `build_segments` so matching lore segments get `protected=True`.
   - Order invariant: gather → spoiler → exclude/pin → budget (a spoiler-dropped item can't be pinned).
6. **`assemble.build_segments`**: accept an optional `pinned_lore_ids: set[str]` → a lore segment whose
   `source_id` is pinned is emitted `protected=True` (keep present/canon as-is; already protected).
   - To keep the `protected` flag attachable, carry `source_id` on the lore item into the segment loop.
7. **`PackedContext`**: add `grounding_items: list[dict]` (`{type,id,label,pinned,excluded}`), built from the
   post-exclude bundle (+ the excluded items re-added flagged so the FE can un-exclude). Lore deduped by `source_id`.
8. **Tests** `tests/unit/test_pack.py`: exclude-drops-item, pin-survives-forced-trim, spoiler-beats-pin,
   sanitize-still-applies, grounding_items shape + lore dedup.

### Slice C — API + wiring
9. **`deps.py`**: `get_grounding_pins_repo`.
10. **`routers/grounding.py`**: thread the repo into the `pack()` call; the GET response includes
    `grounding_items`. New `PUT …/scenes/{nid}/grounding-pins` ({item_type,item_id,action}) — ownership-gated.
11. **`routers/engine.py`**: thread `grounding_pins_repo` into **ALL THREE** engine `pack()` calls
    (lines ~326/664/840 — draft-scene / chapter-stitch / selection-or-critique), so generation honors the
    set too. `pack()` reads keyed on `node['id']`; a synthetic/chapter node with no real `outline_node_id`
    → repo returns **empty set → safe no-op** (no pins to honor). Audit-all-callsites: **4 total** call
    sites (1 grounding + 3 engine), not 2.
12. **Tests** `tests/unit/test_grounding_router.py` + `test_engine_router.py`: PUT upsert/clear; engine pack
    receives the repo at every site (regression-lock all 3 engine sites + the grounding site).

### Slice D — frontend
13. **`api.ts`**: `setGroundingPin`. **types.ts**: `GroundingItem`.
14. **`hooks/useGroundingPins.ts`**: `setAction(item, action)` optimistic + PUT + invalidate grounding query.
15. **`GroundingPanel.tsx`**: render `grounding_items[]` as per-line rows with pin/exclude toggles; excluded
    rows dimmed+struck with restore. Non-addressable blocks unchanged.
16. **i18n** ×4 (`pin.*`).
17. **Tests** `GroundingPanel.test.tsx`: rows+toggles, exclude strike, optimistic+invalidate, persist (mocked).

## Acceptance → test mapping
AC1 pin survives trim → test_pack pin-survives-forced-trim + live-smoke. AC2 exclude absent → test_pack
exclude-drops + engine path. AC3 persist → repo round-trip + FE persist. AC4 spoiler beats pin →

## Risks
- Lore `source_id` granularity (no per-chunk id) — documented in panel copy; `D-T3.4-LORE-CHUNK-ID` if finer
  control is ever needed (requires a knowledge-side hit id).
- `grounding_items` must be built from the SAME post-spoiler bundle the blocks come from, or preview drifts
  from the packed prompt.
