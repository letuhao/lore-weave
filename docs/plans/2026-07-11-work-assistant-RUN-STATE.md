## 3. Standing invariants (the bar ‚Äî never lower these silently)

These are the things that *feel* skippable at 2am and must not be:

1. **A slice is DONE only when its evidence string exists in ¬ß5.** Not when the code compiles. Not when tests
   are green. **Evidence = the behavior was observed.**
2. **Live-smoke for any slice touching ‚â•2 services.** Unit-green has hidden cross-service bugs 4√ó in this repo.
3. **Consumed-by-effect** ‚Äî every flag/guard/setting gets a test asserting the *behavior*, not the stored value.
4. **Fail-closed** ‚Äî every privacy/spend toggle defaults off and fails closed.
5. **No silent success** ‚Äî a success status with no work done is a **bug**. Every skip logs a reason.
6. **Erasure tests assert ABSENCE** (row gone, node gone, rebuild doesn't resurrect) ‚Äî never "invisible".
7. **Never `git add -A`** ‚Äî enumerate files (concurrent sessions share this checkout).
8. **Grep before trusting a list.** The red team found "canon = published" in 6+ places after I'd found one.
   *Assume my enumeration is incomplete.*

**Mechanical enforcement:** use `scripts/workflow-gate.py` (+ the pre-commit hook) ‚Äî it blocks phase jumps and
commits without VERIFY/POST-REVIEW evidence. If I find myself wanting to bypass it, that **is** the drift.
