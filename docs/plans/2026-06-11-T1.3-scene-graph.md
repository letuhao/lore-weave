# T1.3 — Scene Graph canvas (LOOM composition V1)

**Size:** XL (≈10 files, FE-only — BE delta none). **Branch:** `feat/composition-service`.

## Goal
Scenes as a 2-D graph with typed setup→payoff / custom edges, so the author can see
and author causal dependencies the linear outline can't show. Pick-two-nodes link
creation; auto-layout with drag-to-reposition persisted to `work.settings` (shared
across devices per R2). Scene-specific now; generic `GraphCanvas<N,E>` extracted later
when T2.2 (world relations) / T2.5 (world map) give a 2nd real consumer.

## Existing BE (no change)
- `GET /works/{pid}/outline` → `{ nodes, scene_links }` (both, regardless of archived flag).
- `POST /works/{pid}/scene-links` `{from_node_id,to_node_id,kind('setup_payoff'|'custom'),label}` → 201; **409 SCENE_LINK_EXISTS** (unique from,to,kind); 400 BAD_REFERENCE (not-owned / not-in-project).
- `DELETE /scene-links/{id}` → 204 / 404.
- `PATCH /works/{pid}` merges nothing — REPLACES `settings`; caller must merge (precedent `useSetAssemblyMode`/`useSetWorkSettings`).

## Files
1. **types.ts** — `SceneLink` type; `getOutline` return `scene_links: SceneLink[]`.
2. **api.ts** — `createSceneLink(pid, body, token)` POST; `deleteSceneLink(linkId, token)` DELETE.
3. **hooks/useOutline.ts** — `useSceneLinks(pid, token)` (same query key as default `useOutline`, `select: d => d.scene_links`); `createSceneLink`/`deleteSceneLink` mutations added to `useOutlineMutations` (invalidate outline key).
4. **components/sceneGraphLayout.ts** — pure `autoLayout(scenes, links)` → `Record<id,{x,y}>`: columns by distinct `story_order` (NULLS → trailing column), rows by appearance. Deterministic, unit-tested.
5. **components/SceneNode.tsx** — `<g>` view: rect + title + status dot + open (↗) button + selection ring; pointer-down for drag, click toggles select.
6. **components/SceneEdge.tsx** — `<path>` + arrowhead marker; solid (setup_payoff) / dashed (custom); click selects → ✕ delete at midpoint.
7. **components/SceneGraphCanvas.tsx** — host: reads scenes + links, derives effective positions (`local ?? auto`), 5px drag-vs-click threshold, link-create toolbar (2 selected → kind/label → POST, 409 toast), persists positions on drag-end via `useSetWorkSettings` merge into `settings.scene_graph.positions`, navigate-to-scene.
8. **components/CompositionPanel.tsx** — `graph` SubTab (CSS-hidden mount), pass `work`/`bookId`/`token`.
9. **i18n ×4** (en/vi/ja/zh-TW) — top-level `graph` tab label + `scenegraph.*` object.
10. **components/__tests__/SceneGraphCanvas.test.tsx** — pure `autoLayout` + render: edge style by kind, pick-two→add (createSceneLink called), 409→toast/no-dup, edge delete, open→navigate.

## Acceptance
- Scenes render as graph nodes with typed edges (setup solid arrow, custom dashed).
- Create setup_payoff + custom link via pick-two + "+ link"; delete an edge.
- Duplicate link → 409 → toast, no dup edge added.
- Click a node opens its scene; auto-layout legible; drag repositions + persists (work.settings, device-shared).
- i18n ×4; tsc clean; vitest green.

## Notes / accepted trade-offs
- `patchWork` has no If-Match → position saves last-write-wins (cosmetic, R2-acceptable).
- Node set = scenes only (`kind==='scene'`, `!is_archived`).
- Hand-rolled SVG (no graph lib) — keeps bundle lean; generic extraction deferred.
