## useInlineGhost logic
- `continueDraft()`: capture `pos = editor.state.selection.from` + `coords = coordsAtPos(pos)`; `start({projectId, outlineNodeId: sceneId, operation:'continue', modelSource:'user_model', modelRef})`.
- `accept(runCritique)`: bounds-check `anchor.pos ≤ doc.content.size` (else toast `inline.stale` + abort); `editor.chain().focus().insertContentAt(anchor.pos, ghost).run()`; `runCritique && jobId` → `critique.mutate({jobId, passage: ghost})`; close.
- `edit()` = `accept(false)` (insert, no critique — in-doc editable). `discard()` = `stop()` + close. `regenerate()` = clearGhost + re-`start`. `reposition()` = recompute coords on scroll/resize.
- `canContinue = !!projectId && !!sceneId && !!modelRef`.

## InlineAiLayer logic
- `mode` from localStorage `loreweave.editor.aiMode` (default 'classic'); toggle writes through.
- ✦ Continue button shown in AI mode; `disabled={!canContinue || streaming}`; hint when disabled (no-scene vs no-model).
- InlineGhost overlay rendered whenever `anchor != null` (ANCHOR-based, NOT mode-gated → an in-flight ghost survives a Classic toggle).
- No-remount: mode is internal state; the editor + stream never remount.

## Test outline (vitest)
- mode toggle renders/hides the Continue button without remount (visibility-lifecycle); Classic hides Continue.
- Continue (AI) calls start with operation='continue' + the scene; ghost overlay shows streamed tokens.
- Accept inserts at the saved pos (mock editor) + runs critique; Edit inserts without critique; Discard removes; Esc discards.
- Continue disabled with no active scene / no model.
- toggling Classic mid-ghost keeps the overlay (anchor-based).

## Acceptance (spec §5)
AI mode Continue → inline ghost + accept-bar · Accept commits / Discard removes / Edit editable · Classic plain · toggling never loses content or an in-flight stream.
