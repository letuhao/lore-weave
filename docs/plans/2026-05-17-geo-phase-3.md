Add pipeline stages 5–8 to `world_gen`: **provinces + states** (political), **settlements**, **routes**, **culture regions**.

## 3. Shared — terrain cost & pathfinding

**Integer** costs (so a `BinaryHeap` key is `Ord` — no f32-in-heap nondeterminism). Methods on `BiomeKind`:

```
terrain_cost()  — movement cost to ENTER a cell (stages 5,7):
  Plain/Coast/Beach 1 · Forest/River 2 · Hill/Desert/Tundra 3 · Jungle/Marsh 4
  · Mountain 8 · Glacier 20 · Ocean/Lake → None (impassable)
culture_barrier() — culture-spread cost (stage 8):
  Plain/Forest/Coast/Beach/River 1 · Hill/Marsh/Jungle/Tundra 2 · Desert/Mountain 3
  · Ocean/Lake/Glacier → None (hard barrier)
population_potential() — burg-score base (stage 6): Plain 1.0 … Glacier/water 0.0
```

`pathfind.rs`:
- `multi_source_assign(seeds, cost_fn, neighbors) -> Vec<u32>` — multi-source Dijkstra; returns the nearest-seed *index* per cell (`u32::MAX` if unreachable). Heap `BinaryHeap<Reverse<(u32 cost, u32 cell)>>`; cost-tie → lower cell id. Deterministic.
- `single_source_dist(src, cost_fn, neighbors) -> (Vec<u32> dist, Vec<u32> prev)` — Dijkstra from one cell; `dist` + predecessor `prev` for path reconstruction.

## 4. Stage 5 — political (political.rs)

RNG: `rng_for(seed, b"political")`.

**Provinces:**
1. Compute **land components** (connected components of land cells over `neighbors`). `n_prov = (land_cell_count / 200).clamp(4, 80).max(n_components)` so every component can receive a seed. The `clamp(4,80)` is a *target*: if `n_components > 80` the `.max` legitimately raises realised `n_prov` above 80 (design-r3 WARN-1) — criterion #3 checks the realised `n_prov`, not the 80 target.
2. Place **exactly `n_prov`** province seed cells, **apportioned across land components** (resolves design-r1 BLOCK-1 + design-r2 BLOCK-1 — `terrain_cost` is `None` for water so flood-fill cannot cross sea; and per-component quotas must sum to `n_prov`). Apportionment: give every component 1 seed, then distribute the remaining `n_prov - n_components` by **largest-remainder** — component `i`'s extra = `floor(rem × land_fraction_i)`, leftover `rem - Σextra` seeds go to components by descending fractional remainder (ties → lower component index). Total = exactly `n_prov`. Within a component, seeds are a weighted spread: iterate that component's land cells in deterministic RNG-shuffled order, accept a cell ≥ `min_sep` Euclidean (`min_sep = 0.85/sqrt(n_prov)`) from every already-accepted seed, until the quota is met or the component's cells exhausted (a component with quota > cells keeps its ≥1). Every land component ends with ≥1 seed; `provinces.len() == n_prov`.
3. `province_of = multi_source_assign(seed_cells, terrain_cost, neighbors)` — since every land component has ≥1 seed, every land cell is reachable from a seed *in its own component* ⇒ every land cell gets a province; water cells `u32::MAX`. Provinces partition the land totally.
4. `Province { id, capital_cell: seed_cell, state }`. `capital_cell` is always a land cell (seeds are placed only on land).

**States:** cluster provinces by capital proximity.
5. Union-find over provinces: link province `p,q` iff they lie **in the same land component** AND the Euclidean distance between their `capital_cell` centres ≤ `STATE_LINK_DIST` (= `2.4 / sqrt(n_prov)` — tuned so a continent yields ~4–10 states). The same-component guard (design-r2 WARN-2; matches GEO_002 §5.1's TerrainCost-path intent) keeps every `State` land-coherent — no state spans a sea strait.
6. Each connected component → one `State`; `capital_province` = the **lowest province id** in the component (deterministic). Every province's `state` field set to its component's state id ⇒ **every state has exactly one capital, every province belongs to exactly one state**.

## 5. Stage 6 — settlement (settlement.rs)

RNG: `rng_for(seed, b"settlement")`.

**Burg score** per land cell: `burg = population_potential(biome) × water_bonus × climate_friendly(climate)`.
- `water_bonus`: `is_coast` → ×1.3; else `river_flux > 0` → ×1.15; else ×1.0.
- `climate_friendly`: Temperate/Mediterranean 1.0 · Subtropical 0.9 · Tropical 0.7 · Boreal 0.6 · Arid/Highland 0.5 · Polar 0.3.

**Placement (weighted Poisson-disk):**
1. `target = land_cell_count / density.cells_per()` (Sparse 800 / Medium 400 / Dense 200); `min_sep` Sparse 0.08 / Medium 0.05 / Dense 0.03.
2. Iterate land cells in a deterministic RNG-shuffled order; accept cell `c` iff, evaluated as a **short-circuit `&&` in exactly this order** (normative for byte-stability; design-r3 WARN-2): (a) `burg[c] > 0.05`, (b) `c` unoccupied, (c) Euclidean min-distance to every placed settlement ≥ `min_sep`, (d) one RNG draw `< burg[c]/max_burg`. Draw (d) is the **last** conjunct — exactly one RNG value is consumed, and only when (a)–(c) all hold — so the RNG-stream alignment is a fixed contract, not an accident of conjunct order. Stop at `target` accepted or land exhausted.

**Roles** (after placement):
3. For each `State`: in its `capital_province`, the settlement **nearest** the province `capital_cell` → `Capital` — "nearest" by `(Euclidean distance, cell_id)`, lowest wins (deterministic tie-break; design-r1 BLOCK-2). If that province has **zero** settlements, place one at `capital_cell` → `Capital`; this is collision-free — zero-settlements-in-province ⇒ `capital_cell` (which lies in its own province) is unoccupied, and `capital_cell` is always land (§4 step 4). ⇒ every state has exactly one Capital, every `Settlement.cell` stays unique.
4. Non-Capital settlements ranked by `burg`: top 12 % → `City`, next 22 % → `Town`, next 33 % → `Village`, rest → `Hamlet`.
5. Override: any non-Capital settlement on a `Mountain`-biome cell → `Fortress`.
6. `population_tier`: Capital 5 · City 4 · Town 3 · Village 2 · Fortress 2 · Hamlet 1.

## 6. Stage 7 — routes (routes.rs)

7a **Road** — connects `population_tier ≥ 2` settlements. For each such settlement run `single_source_dist` over `terrain_cost`; build the inter-settlement distance graph. **Roads = a Kruskal MST over that graph** ∪ each settlement's single nearest non-MST neighbour. On a multi-component world the graph is disconnected, so Kruskal yields a spanning **forest — one MST per land component** (criterion #6 is per-component; design-r1 BLOCK-1). Determinism: Kruskal sorts edges by `(cost, lo_cell, hi_cell)`; the augmentation iterates settlements by ascending `cell_id`, picking the nearest non-MST neighbour by `(cost, neighbour_cell_id)`. If a settlement has no reachable non-MST neighbour (an isolated component, or all its neighbours already MST-linked) its augmentation step is a **no-op** (no panic on an empty min). One `Route` per unordered pair (dedup; emit with `from_cell < to_cell`). Unreachable pairs skipped.

7b **Trail** — each `population_tier ∈ {0,1}` settlement (iterated by ascending `cell_id`) → a `Trail` to its nearest `tier ≥ 2` settlement by terrain-cost distance, ties → `(cost, cell_id)` lowest. Unreachable → skipped.

7c **SeaLane** — coastal `City` settlements sorted by `cell_id`; for each pair `i<j` within Euclidean `0.5`, BFS over ocean-water-cell adjacency between them; if connected emit one `SeaLane` (`from_cell < to_cell`).

7d **MountainPass** — while reconstructing 7a Road paths, tally adjacency edges (keyed `(lo_cell, hi_cell)`) on Road paths with ≥1 `Mountain` endpoint; sort by `(tally desc, lo_cell, hi_cell)`; the top `min(8, count)` → `MountainPass` routes.

7e **RiverNavigation** — scanning each Road/Trail path's cell sequence (paths processed in their deterministic 7a/7b emission order): every run of ≥ 3 consecutive cells with `river_flux > river_threshold` emits a parallel `RiverNavigation` route over that segment.

All pathfinding is integer-cost Dijkstra/BFS with `(cost, cell)` heap keys + Kruskal sorted by `(cost, lo_cell, hi_cell)`. **No `HashMap`/`HashSet` in any emission-order path** — edge tallies use a sorted `Vec`/`BTreeMap` — so the route Vec is byte-stable (design-r1 WARN-1).

## 7. Stage 8 — culture (culture.rs)

RNG: `rng_for(seed, b"culture")`.
1. `k = culture_count.clamp(1, 16)`. Place `k` hearth cells, **distributed per land component** like province seeds (§4 step 2): if `k ≥ n_components` each component gets `max(1, round(k × component_land_fraction))` hearths; if `k < n_components` the `k` largest components get one hearth each (the rest fall back in step 2).
2. `culture_of = multi_source_assign(hearths, culture_barrier, neighbors)` — every land cell → nearest hearth's culture; water `u32::MAX`. A land cell in a component with no hearth (only possible when `k < n_components`) is unreachable → culture `0` (fallback; `culture_regions[0]` always exists since `k ≥ 1`).
3. `CultureRegion { id, hearth_cell }`.

## 9. Acceptance criteria (VERIFY gate)

1. `cargo build` + `cargo clippy --all-targets` clean.
2. **Determinism:** `generate` byte-identical across two runs (hash covers all 7 new fields) — ≥4 fixture seeds.
3. **Provinces partition the land:** every land cell has `province_of != u32::MAX`; every water cell has `province_of == u32::MAX`; every `province_of` value indexes a real `Province`; `provinces.len()` equals the apportioned `n_prov`.
4. **States:** every `State` has exactly one `capital_province`; every `Province.state` indexes a real `State`; every `State.capital_province` is a province whose `state` points back (consistency).
5. **Settlements:** every `Settlement.cell` is a unique land cell; every `State` has exactly one `Capital`-role settlement; `population_tier` matches `role`.
6. **Routes:** at most one route per unordered cell-pair per kind; every `Route.from_cell`/`to_cell` is a valid cell. **Road connectivity** (machine check): build an undirected graph whose nodes are the `tier ≥ 2` settlement cells, with one edge per emitted `Route` of `kind == Road` whose *both* endpoints are such cells; for every land component with ≥2 such settlements those settlements form a single connected union-find class (components with ≤1 are trivially satisfied).
7. **Culture:** every land cell has a `culture_of`; every value indexes a real `CultureRegion`; water cells `u32::MAX`; `culture_regions.len()` equals `culture_count.clamp(1,16)`.
8. CLI end-to-end → JSON + political PNG; PNG flagged for human morning review.
