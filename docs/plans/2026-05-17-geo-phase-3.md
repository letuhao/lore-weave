## 1. Goal & scope

Add pipeline stages 5‚Äì8 to `world_gen`: **provinces + states** (political), **settlements**, **routes**, **culture regions**.

**In scope:** `SettlementRole` (6) + `RouteKind` (5) + `SettlementDensity` (3) enums; `CreativeSeed += settlement_density, culture_count`; `WorldMap += province_of, provinces, states, settlements, routes, culture_of, culture_regions`; modules `pathfind.rs` + `political.rs` + `settlement.rs` + `routes.rs` + `culture.rs`; a political/route CLI render; extended `content_hash` + tests.

**Pure procedural.** GEO_002/003/004's Canonical/Hybrid author-declaration modes are **out of scope** ‚Äî those belong to Phase 4 (LLM CreativeSeed authoring). Phase 3 generates all 4 layers from the seed alone.

**Out of scope:** serialization round-trip stability, LLM authoring (Phase 4); event-sourcing/deltas (dropped per GEO_GENERATOR_PLAN ¬ß1).

**Invariant:** regeneration-determinism ‚Äî `generate(seed, cs)` byte-identical, now including all 4 new layers.

## 3. Shared ‚Äî terrain cost & pathfinding

**Integer** costs (so a `BinaryHeap` key is `Ord` ‚Äî no f32-in-heap nondeterminism). Methods on `BiomeKind`:

```
terrain_cost()  ‚Äî movement cost to ENTER a cell (stages 5,7):
  Plain/Coast/Beach 1 ¬∑ Forest/River 2 ¬∑ Hill/Desert/Tundra 3 ¬∑ Jungle/Marsh 4
  ¬∑ Mountain 8 ¬∑ Glacier 20 ¬∑ Ocean/Lake ‚Üí None (impassable)
culture_barrier() ‚Äî culture-spread cost (stage 8):
  Plain/Forest/Coast/Beach/River 1 ¬∑ Hill/Marsh/Jungle/Tundra 2 ¬∑ Desert/Mountain 3
  ¬∑ Ocean/Lake/Glacier ‚Üí None (hard barrier)
population_potential() ‚Äî burg-score base (stage 6): Plain 1.0 ‚Ä¶ Glacier/water 0.0
```

`pathfind.rs`:
- `multi_source_assign(seeds, cost_fn, neighbors) -> Vec<u32>` ‚Äî multi-source Dijkstra; returns the nearest-seed *index* per cell (`u32::MAX` if unreachable). Heap `BinaryHeap<Reverse<(u32 cost, u32 cell)>>`; cost-tie ‚Üí lower cell id. Deterministic.
- `single_source_dist(src, cost_fn, neighbors) -> (Vec<u32> dist, Vec<u32> prev)` ‚Äî Dijkstra from one cell; `dist` + predecessor `prev` for path reconstruction.

## 4. Stage 5 ‚Äî political (political.rs)

RNG: `rng_for(seed, b"political")`.

**Provinces:**
1. Compute **land components** (connected components of land cells over `neighbors`). `n_prov = (land_cell_count / 200).clamp(4, 80).max(n_components)` so every component can receive a seed. The `clamp(4,80)` is a *target*: if `n_components > 80` the `.max` legitimately raises realised `n_prov` above 80 (design-r3 WARN-1) ‚Äî criterion #3 checks the realised `n_prov`, not the 80 target.
2. Place **exactly `n_prov`** province seed cells, **apportioned across land components** (resolves design-r1 BLOCK-1 + design-r2 BLOCK-1 ‚Äî `terrain_cost` is `None` for water so flood-fill cannot cross sea; and per-component quotas must sum to `n_prov`). Apportionment: give every component 1 seed, then distribute the remaining `n_prov - n_components` by **largest-remainder** ‚Äî component `i`'s extra = `floor(rem √ó land_fraction_i)`, leftover `rem - Œ£extra` seeds go to components by descending fractional remainder (ties ‚Üí lower component index). Total = exactly `n_prov`. Within a component, seeds are a weighted spread: iterate that component's land cells in deterministic RNG-shuffled order, accept a cell ‚â• `min_sep` Euclidean (`min_sep = 0.85/sqrt(n_prov)`) from every already-accepted seed, until the quota is met or the component's cells exhausted (a component with quota > cells keeps its ‚â•1). Every land component ends with ‚â•1 seed; `provinces.len() == n_prov`.
3. `province_of = multi_source_assign(seed_cells, terrain_cost, neighbors)` ‚Äî since every land component has ‚â•1 seed, every land cell is reachable from a seed *in its own component* ‚áí every land cell gets a province; water cells `u32::MAX`. Provinces partition the land totally.
4. `Province { id, capital_cell: seed_cell, state }`. `capital_cell` is always a land cell (seeds are placed only on land).

**States:** cluster provinces by capital proximity.
5. Union-find over provinces: link province `p,q` iff they lie **in the same land component** AND the Euclidean distance between their `capital_cell` centres ‚â§ `STATE_LINK_DIST` (= `2.4 / sqrt(n_prov)` ‚Äî tuned so a continent yields ~4‚Äì10 states). The same-component guard (design-r2 WARN-2; matches GEO_002 ¬ß5.1's TerrainCost-path intent) keeps every `State` land-coherent ‚Äî no state spans a sea strait.
6. Each connected component ‚Üí one `State`; `capital_province` = the **lowest province id** in the component (deterministic). Every province's `state` field set to its component's state id ‚áí **every state has exactly one capital, every province belongs to exactly one state**.

## 5. Stage 6 ‚Äî settlement (settlement.rs)

RNG: `rng_for(seed, b"settlement")`.

**Burg score** per land cell: `burg = population_potential(biome) √ó water_bonus √ó climate_friendly(climate)`.
- `water_bonus`: `is_coast` ‚Üí √ó1.3; else `river_flux > 0` ‚Üí √ó1.15; else √ó1.0.
- `climate_friendly`: Temperate/Mediterranean 1.0 ¬∑ Subtropical 0.9 ¬∑ Tropical 0.7 ¬∑ Boreal 0.6 ¬∑ Arid/Highland 0.5 ¬∑ Polar 0.3.

**Placement (weighted Poisson-disk):**
1. `target = land_cell_count / density.cells_per()` (Sparse 800 / Medium 400 / Dense 200); `min_sep` Sparse 0.08 / Medium 0.05 / Dense 0.03.
2. Iterate land cells in a deterministic RNG-shuffled order; accept cell `c` iff, evaluated as a **short-circuit `&&` in exactly this order** (normative for byte-stability; design-r3 WARN-2): (a) `burg[c] > 0.05`, (b) `c` unoccupied, (c) Euclidean min-distance to every placed settlement ‚â• `min_sep`, (d) one RNG draw `< burg[c]/max_burg`. Draw (d) is the **last** conjunct ‚Äî exactly one RNG value is consumed, and only when (a)‚Äì(c) all hold ‚Äî so the RNG-stream alignment is a fixed contract, not an accident of conjunct order. Stop at `target` accepted or land exhausted.

**Roles** (after placement):
3. For each `State`: in its `capital_province`, the settlement **nearest** the province `capital_cell` ‚Üí `Capital` ‚Äî "nearest" by `(Euclidean distance, cell_id)`, lowest wins (deterministic tie-break; design-r1 BLOCK-2). If that province has **zero** settlements, place one at `capital_cell` ‚Üí `Capital`; this is collision-free ‚Äî zero-settlements-in-province ‚áí `capital_cell` (which lies in its own province) is unoccupied, and `capital_cell` is always land (¬ß4 step 4). ‚áí every state has exactly one Capital, every `Settlement.cell` stays unique.
4. Non-Capital settlements ranked by `burg`: top 12 % ‚Üí `City`, next 22 % ‚Üí `Town`, next 33 % ‚Üí `Village`, rest ‚Üí `Hamlet`.
5. Override: any non-Capital settlement on a `Mountain`-biome cell ‚Üí `Fortress`.
6. `population_tier`: Capital 5 ¬∑ City 4 ¬∑ Town 3 ¬∑ Village 2 ¬∑ Fortress 2 ¬∑ Hamlet 1.

## 6. Stage 7 ‚Äî routes (routes.rs)

7a **Road** ‚Äî connects `population_tier ‚â• 2` settlements. For each such settlement run `single_source_dist` over `terrain_cost`; build the inter-settlement distance graph. **Roads = a Kruskal MST over that graph** ‚à™ each settlement's single nearest non-MST neighbour. On a multi-component world the graph is disconnected, so Kruskal yields a spanning **forest ‚Äî one MST per land component** (criterion #6 is per-component; design-r1 BLOCK-1). Determinism: Kruskal sorts edges by `(cost, lo_cell, hi_cell)`; the augmentation iterates settlements by ascending `cell_id`, picking the nearest non-MST neighbour by `(cost, neighbour_cell_id)`. If a settlement has no reachable non-MST neighbour (an isolated component, or all its neighbours already MST-linked) its augmentation step is a **no-op** (no panic on an empty min). One `Route` per unordered pair (dedup; emit with `from_cell < to_cell`). Unreachable pairs skipped.

7b **Trail** ‚Äî each `population_tier ‚àà {0,1}` settlement (iterated by ascending `cell_id`) ‚Üí a `Trail` to its nearest `tier ‚â• 2` settlement by terrain-cost distance, ties ‚Üí `(cost, cell_id)` lowest. Unreachable ‚Üí skipped.

7c **SeaLane** ‚Äî coastal `City` settlements sorted by `cell_id`; for each pair `i<j` within Euclidean `0.5`, BFS over ocean-water-cell adjacency between them; if connected emit one `SeaLane` (`from_cell < to_cell`).

7d **MountainPass** ‚Äî while reconstructing 7a Road paths, tally adjacency edges (keyed `(lo_cell, hi_cell)`) on Road paths with ‚â•1 `Mountain` endpoint; sort by `(tally desc, lo_cell, hi_cell)`; the top `min(8, count)` ‚Üí `MountainPass` routes.

7e **RiverNavigation** ‚Äî scanning each Road/Trail path's cell sequence (paths processed in their deterministic 7a/7b emission order): every run of ‚â• 3 consecutive cells with `river_flux > river_threshold` emits a parallel `RiverNavigation` route over that segment.

All pathfinding is integer-cost Dijkstra/BFS with `(cost, cell)` heap keys + Kruskal sorted by `(cost, lo_cell, hi_cell)`. **No `HashMap`/`HashSet` in any emission-order path** ‚Äî edge tallies use a sorted `Vec`/`BTreeMap` ‚Äî so the route Vec is byte-stable (design-r1 WARN-1).

## 7. Stage 8 ‚Äî culture (culture.rs)

RNG: `rng_for(seed, b"culture")`.
1. `k = culture_count.clamp(1, 16)`. Place `k` hearth cells, **distributed per land component** like province seeds (¬ß4 step 2): if `k ‚â• n_components` each component gets `max(1, round(k √ó component_land_fraction))` hearths; if `k < n_components` the `k` largest components get one hearth each (the rest fall back in step 2).
2. `culture_of = multi_source_assign(hearths, culture_barrier, neighbors)` ‚Äî every land cell ‚Üí nearest hearth's culture; water `u32::MAX`. A land cell in a component with no hearth (only possible when `k < n_components`) is unreachable ‚Üí culture `0` (fallback; `culture_regions[0]` always exists since `k ‚â• 1`).
3. `CultureRegion { id, hearth_cell }`.

## 9. Acceptance criteria (VERIFY gate)

1. `cargo build` + `cargo clippy --all-targets` clean.
2. **Determinism:** `generate` byte-identical across two runs (hash covers all 7 new fields) ‚Äî ‚â•4 fixture seeds.
3. **Provinces partition the land:** every land cell has `province_of != u32::MAX`; every water cell has `province_of == u32::MAX`; every `province_of` value indexes a real `Province`; `provinces.len()` equals the apportioned `n_prov`.
4. **States:** every `State` has exactly one `capital_province`; every `Province.state` indexes a real `State`; every `State.capital_province` is a province whose `state` points back (consistency).
5. **Settlements:** every `Settlement.cell` is a unique land cell; every `State` has exactly one `Capital`-role settlement; `population_tier` matches `role`.
6. **Routes:** at most one route per unordered cell-pair per kind; every `Route.from_cell`/`to_cell` is a valid cell. **Road connectivity** (machine check): build an undirected graph whose nodes are the `tier ‚â• 2` settlement cells, with one edge per emitted `Route` of `kind == Road` whose *both* endpoints are such cells; for every land component with ‚â•2 such settlements those settlements form a single connected union-find class (components with ‚â§1 are trivially satisfied).
7. **Culture:** every land cell has a `culture_of`; every value indexes a real `CultureRegion`; water cells `u32::MAX`; `culture_regions.len()` equals `culture_count.clamp(1,16)`.
8. CLI end-to-end ‚Üí JSON + political PNG; PNG flagged for human morning review.
