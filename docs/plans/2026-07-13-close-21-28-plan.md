### 10.1 The defect (verified against code, not the spec's word)

`plan_forge_service.propose(created_by, **book_id**, source_markdown, …)` — **`book_id` is already in
scope**, and the service already calls `_ensure_work(book_id)`. But it imports only `WorksRepo` /

a blank one.** This is *not* "blocked on a missing signal" — the signal and the plumbing are both
there. It is unbuilt work. (`blocked-on-sdk-field-often-self-owned`.)

### 10.2 What actually happens today, end-to-end — **and the safety net is in the wrong place**

The linker's upsert key is **`(book_id, plan_run_id, plan_arc_id)`** — `plan_run_id` is *in the key*.
A second proposal ⇒ a new `plan_run_id` ⇒ **`ON CONFLICT` can never match** ⇒ a **second, disjoint set
of arcs and chapters is inserted onto the same book.**

There *is* a guard — PF-10's `_FIND_DUPES` — and it is honest (*"we surface it; we never merge it"*).
**But it matches on `lower(btrim(title))`.** A proposer that never read the book invents *fresh*
titles ("Arc 1 — The Beginning") for a book already at chapter 40 — **so the titles don't collide and
the detector never fires.**

> **The safety net is downstream of the blindness, and it is keyed on the one signal the blindness
> destroys.** It can only catch a blind proposer that *happens to guess the same titles*.

### 10.5 Rules mode — **RESOLVED** (2026-07-13, by reading what it actually is)

**What rules mode is:** `plan_propose_spec` runs two modes over the same input — a markdown document
the **author wrote themselves**. `rules` **parses** it deterministically (**$0**, no model,
**synchronous**); `llm` has a model **write** the spec from it (billed, async, needs `model_ref`).

⇒ **Rules mode is a TRANSCRIBER, not a generator.** Its pitch is: *"I already wrote my plan. Structure
it for me — **don't let an AI reinterpret it.**"* And `propose.py`'s own header (written by 27-B5, the
fixture severing) states the law:

> *"parse what the document says; **emit nothing where it says nothing**. Absent ≠ invented — an honest
> empty tells the author exactly what their braindump is missing."*

*(Before B5, rules mode was a **fixture wearing a parser**: for ANY document it returned another
novel's variables, anchors, forbids and protagonist. Not a crash, not an empty — **a confident wrong
answer.** P-06.)*

**So the question answers itself.** **Grounding rules mode = inventing content the document does not
contain** — which is *precisely* the bug B5 just killed. Rules mode must **not** be grounded, not as a
compromise but because that is the only thing consistent with what it **is**.

**The resolution — pre-flight, not grounding:**

| Mode | Book-state treatment |
|---|---|
| **rules** | **PRE-FLIGHT.** Read the book's state; do **not** feed it to the parser. **Report** the collision — *"this book already has 3 arcs / 40 chapters; your document proposes 2 arcs matching none of them"* — and stop behind an explicit confirm. **Never invents** (rules mode's contract) and **never silently re-plans** (*absent ≠ zero*). |
| **llm** | **GROUND** (option B) — digest in, a spec that *continues* the book out. Fail-closed: a degraded read ⇒ refuse, never propose blind. |

**This also fixes §10.2's real defect**, which is not in the proposer at all: `_FIND_DUPES` keys on
**title**, so it can only catch a blind proposer that *happens to guess the same titles*. The
pre-flight compares **book state**, not strings — so it fires exactly when the blindness would have
been silent.
