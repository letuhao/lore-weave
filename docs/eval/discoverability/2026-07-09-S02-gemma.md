### Root cause — context ids are advertised as PROSE, not filled into tool args

Every failing call errored with:

```
VALIDATION — validating "arguments": required: missing properties: ["book_id"]
```

`glossary_book_ontology_read` / `glossary_search` **require `book_id`**, and gemma called them with `{}`.
The book_id *is* known to the turn (`book_context.book_id`), but it is surfaced to the model **only as a
prose system note** — `stream_service.py:2811` "You are working inside book_id=… Use these exact ids for
any tool that requires a book_id." — and **relies on the model to transcribe that UUID into every tool
call.** Gemma (mid-tier) didn't → `VALIDATION` → the agent **retried the same empty call up to 11×** →
gave up with a "technical error."

This is a textbook **mid-tier-model failure mode**: a strong model copies the UUID from the note and
succeeds (why this passes in a demo); a mid-tier model omits it and dies in a blind retry loop. The
black-box user can never supply the fix — the book_id is a UUID they must never see (§1).

## Gap → required capability (feeds the umbrella)

**Primary (blocks S02 *and* S06 and every book-scoped tool journey):**
**Deterministically fill context ids (`book_id` / `chapter_id` / `project_id`) into tool args
server-side** when (a) the tool's `inputSchema` marks them required and (b) the turn context carries them
— instead of relying on the model to transcribe a UUID from a prose note. This kills the whole
`VALIDATION: missing book_id` retry-loop class in one move. Pairs naturally with C2 `tool_load` (the
schema that says which args are required is exactly what the filler reads).

**Secondary:**
- **Retry cap on identical failing tool calls** — 11 identical `glossary_search({})` calls is pure waste;
  a "same call, same args, same error → don't repeat, surface the blocker" guard (F1/F2 cap) would cut it.
- **Jargon translation** — the failure narration leaks *kinds/attributes/ontology/propose* as required
  user input; a workflow's `notes_md` should own that vocabulary (F6).
- **Path B (paste notes) still needs the seed-doc→entities parser** (W2) — not reached here because the
  read failed first, but it remains an unbuilt capability.
