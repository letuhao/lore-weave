⚠ The problem we're correcting

Today the KG ships one hardcoded general ontology — entity/event/fact kinds are Python Literals, edge predicates are free strings with no vocab, baked into static prompts. Glossary already solved "customizable" for node identity (genre·kind·attribute, tiered). The KG needs the same for graph shape (edge vocab, fact/state types, controlled vocab) — but that's the KG's domain, not glossary's.

Ownership split

Two domains, one anchor. KG never re-defines node kinds.

GLOSSARYNode identity
"what a thing is" — kind, name, aliases, attributes. Already tiered (system/user/book). SSOT.
KNOWLEDGEGraph shape
"how things relate + change" — edge vocab, fact/state types, drive vocab, temporal. This epic.
❓ Gap (M1 / §10-G1): node kinds come from glossary at book tier (shared); graph schema is at project tier (per-user). They must agree on kind codes. Adopt is gated on the book glossary already having the kinds. → see Adopt.

A graph schema = the customizable unit

Tiered like glossary. Adopt (copy-down) + sync. Declares:

  • edge types code · direction · source/target node-kinds · temporal? · cardinality
  • fact types narrative state events (realm_change, allegiance_shift…)
  • vocab sets closed controlled vocab (e.g. drive ×16)
  • extraction hints what to pull / node budget

The 4-layer model

Layer 1Node kinds← from glossary (anchor, not redefined)
Layer 2Graph schema← KG-owned, tiered system/user/project, adopt + sync
system templates→user→project (active)
Layer 3View / lens← multi-graph v1: named {edge-types + node-kinds} over ONE project graph. READ-only.
Layer 4Graph partition← DEFERRED. graph_id on edge (nodes shared). View ≠ partition — not a continuum.
