Five-Stage Data Pipeline

Each stage is an independent consumer — add new stages without changing existing ones

Block Extractor
Tiptap JSON → denormalized text blocks. Runs as Postgres trigger (zero latency, ACID consistent).

Outbox Relay
Publish events from outbox to Redis Streams + permanent event log. LISTEN/NOTIFY driven.

PIPELINE 3

PIPELINE 4
Embedding Generator

Background Processing

Two-Worker Architecture

Split by language boundary and resource profile. Scale by adding containers with different WORKER_TASKS.

worker-infra Go · lightweight · always-on · 50MB RAM
LISTEN outbox-relay pg_notify + poll → Redis + event_log D1
CRON outbox-cleanup daily: delete published events > 7d D1
CRON event-archive weekly: partition old event_log future
CONSUME search-indexer Redis → Meilisearch future
CONSUME notifier Redis → push/email future
worker-ai Python · heavy · bursty · 2GB+ RAM / GPU
QUEUE translation-worker RabbitMQ → LLM → DB existing
CONSUME knowledge-extractor Redis → LLM NER → Neo4j D3
CONSUME embedding-generator Redis → embed → Neo4j vectors D4

Scaling Pattern

Dev:    worker-infra WORKER_TASKS=outbox-relay,outbox-cleanup
Prod:   worker-relay  WORKER_TASKS=outbox-relay          // high priority, dedicated
        worker-cron   WORKER_TASKS=outbox-cleanup,event-archive
        worker-ai-1  WORKER_TASKS=knowledge-extractor  // GPU instance
        worker-ai-2  WORKER_TASKS=embedding-generator  // GPU instance
        worker-ai-3  WORKER_TASKS=translation-worker   // scale horizontally

Technology Decisions

Infrastructure Stack

Latest versions researched 2026-03-31. Each technology chosen for specific capabilities.

PostgreSQL
v18 (released Sep 2025)
Source of truth for all application data.
  • JSON_TABLE — query JSONB as relational rows
  • Virtual generated columns — zero storage cost
  • uuidv7() — time-ordered UUIDs, native
  • Async I/O — 2-3x read performance
  • SIMD JSON processing
Neo4j
v2026.01 (latest)
Knowledge graph + vector search in one database.
  • Native filtered vector search (HNSW)
  • Cypher SEARCH clause — clean syntax
  • Multi-label vector indexes
  • Eliminates need for separate Qdrant
  • Community Edition (free, open source)
Redis
v7+ (Streams)
Event bus for pipeline communication.
  • Streams — persistent event log with replay
  • Consumer groups — parallel processing
  • MAXLEN auto-trim — bounded memory
  • Sub-millisecond pub/sub
  • Already in stack (lightweight addition)
