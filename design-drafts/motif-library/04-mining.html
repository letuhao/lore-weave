Abstract manually
Spec deltas surfaced here:
  • knowledge-service read — mining needs a new client to read (:Event)-[:CAUSES|:HAPPENS_BEFORE] chains per book; confirm the query + auth (internal token).
  • Dedup / merge-as-variant — a near-dup needs a cosine check vs existing motifs + a variant_of merge action. Add to the miner + API.
  • Abstraction confidence — the LLM lift must return a confidence; low-confidence candidates route to manual, not auto-add. New field on the mined draft.
  • Judge gate threshold — config motif_mine_min_judge; below-gate candidates are shown but not added (transparency, not silent drop).
  • Provenance back-links — a mined motif should link to the chapters/events it came from (source_ref richer than a run id) for "show me where this came from".
  • Cost guard — mining is a budgeted job (like the planner); needs the usage-billing pre-check + a $ estimate before run.
