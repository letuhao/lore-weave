3 · Apply to a new arc (your story)

arc span 12 ch→ reconcile → your target 8 ch (compress, surfaced)→ bind roster → your cast→ interleave threads → multi-chapter outline

= the decompose planner at arc scale (§12.5). Each scene stays traceable via motif_application. Drops/merges from the 12→8 compression are listed, never silent. → continues into Planner binding.

Confirms / spec deltas:
  • arc_template = JSONB threads + layout (placements) + arc_roster + pacing — the timeline is this data, visualized.
  • Placements need recurrence — same motif can appear twice (combat at ch1 & ch10) → a placement list, NOT a unique link. Confirms §12.2 JSONB layout over a unique-keyed link table.
  • Thread taxonomy — combat/cultivation/romance/politics: seed a small set + allow custom (tenancy like genres). New decision.
  • Copyright UX — raw-text panel is visibly private, the template carries roles/beats only (no names/prose). §12.6 lock made visible.
  • Reconciliation UX — span→target compression (12→8) must list dropped/merged motifs (no silent drop) — same as A3 B≠C.
  • Timeline editor = a NEW FE surface (drag placements, swap motif) distinct from the single-motif editor. Scope it as part of P4.
