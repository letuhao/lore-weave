⚠ The gap this fills (why the planner feels bland)

The A3 planner asks the LLM to invent scenes from a blank macro-beat ("Midpoint" + premise + cast). An uninstructed model under-weights plot 1.67× vs style (Style over Story, arXiv:2510.02025) — so a weak self-host produces plot-thin scenes. A motif supplies the missing plot structure as a reusable, role-bearing event chain — the planner shifts from "invent" to "select + bind", which a small model can do.

Where it sits — the altitude stack

The library is the meso layer. The planner consumes top-down; each layer constrains the next.

What a motif is (the schema, visualized)

Grounded in narratology — not invented. Roles = Greimas/Propp · ordered beats = Propp functions · conditions = MEXICA/plot-graph.

sequence Fortuitous Encounter → Hidden Legacy cultivation.fortuitous_encounter
tu-tiên xianxia

A weak/humiliated protagonist is isolated by disaster, encounters a dying master in a sealed place, and receives a legacy that inverts their power — at the cost of a debt or secret.

Ordered beats (the chain)
① Isolation by disaster T2→ ② Encounter the dying master T3→ ③ Trial of worthiness T4→ ④ Receipt of legacy T4→ ⑤ Secret debt incurred T2
Role slots → bind to cast
subject protagonist · weak/humiliated
sender dying master · mentor, near death
object the legacy · technique/bloodline
opponent (optional) the one who caused the fall
Preconditions
  • protagonist is weak / low-status
  • an isolated or sealed locale exists
  • protagonist recently suffered a loss
Effects (→ next motif's pre)
  • protagonist power ↑ (hidden)
  • a debt / secret to the sender
  • a long-term goal is seeded
Overall tension target ▲ 3/5 · Emotion target vindication / hope · Composes into → pattern: Rise-from-Trash Revenge Arc

Tiering & publish — same model as books

Three tiers resolve System → User → Book (higher shadows lower by code). Publish & adopt mirror the book flow.

Tier Scope key Who writes Use
Book book_id + code owner + manage-grant motifs mined/authored for this book; planner reads these first
User owner_user_id + code that user my reusable library across all my books
System code (both NULL) admin / seed only the seeded tu-tiên / báo-thù packs (read-only to users)
Public visibility='public' owner (instant, no review) discoverable in the catalog; others adopt (clone) into their tier
Book draft→ promote → My library→ publish → Public catalog→ adopt (clone) → Someone's book

Adopt = copy-on-adopt with provenance (source_ref); your edits never touch the shared original (glossary-tiering D8 precedent). Upstream edits pull on demand, never auto-push.

Self-enrichment flywheel — the library grows itself (mandatory)
