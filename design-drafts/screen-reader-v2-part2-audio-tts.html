Block-Level TTS Highlighting + Auto-Scroll
How it works:
1. Active block gets gold left border + subtle background highlight
2. Past blocks stay normal opacity, upcoming blocks slightly dimmed
3. Auto-scroll smoothly centers the active block in the viewport
4. Click any block to jump TTS to that position
5. Media blocks (image/video) are paused on — reader sees the visual while TTS waits

TTS Architecture
TTS COMPONENT TREE: <TTSProvider> ← React context wrapping ReaderPage state: { mode, status, activeBlockIdx, speed, voice, segments } │ ├── useBrowserTTS() ← Web Speech API hook │ speak(text) → SpeechSynthesisUtterance │ onBoundary → word-level progress │ onEnd → advance to next block │ ├── useAITTS() ← AI voice generation hook │ generateAudio(blocks) → POST /v1/tts/generate │ returns: { blockId, audioUrl }[] │ plays via <audio> element │ └── <TTSBar /> ← floating player UI BLOCK SLICING: 1. chapter.body.content → extract blocks 2. Filter: keep paragraph, heading, blockquote, callout, list 3. Skip: imageBlock, videoBlock, horizontalRule, codeBlock 4. For each text block: extractText(block) → plain string 5. Build ordered queue: [{ blockId, text, type }] SYNC FLOW: TTS reads block N → set ttsActiveBlock = blockId → ContentRenderer highlights block N → auto-scroll: block N → scrollIntoView({ behavior: 'smooth', block: 'center' }) → on complete: advance to block N+1 → if media block: pause for 3s (configurable), then skip AUDIO STORAGE (AI mode — persisted assets, NOT cache): Generated audio is a PERMANENT MEDIA ASSET — stored in DB + MinIO, just like images and videos. Never expires, never auto-deleted. DB table: chapter_audio_segments chapter_id UUID block_index INT ← position in chapter voice TEXT ← e.g. 'alloy', 'nova' provider TEXT ← e.g. 'openai', 'elevenlabs' language TEXT ← e.g. 'ja', 'en' media_url TEXT ← MinIO presigned URL media_key TEXT ← MinIO object key duration_ms INT content_hash TEXT ← hash of source text at generation time created_at TIMESTAMPTZ MinIO path: audio/{chapter_id}/{language}/{voice}/{block_index}.mp3 Content drift detection: - On load, compare current block text hash vs stored content_hash - If different → show "Content changed since audio was generated" with option to re-generate (old audio still playable) - User explicitly clicks "Re-generate" → creates new segment, old one is replaced (same DB row updated, old MinIO object deleted) Reuses existing media infrastructure: - MinIO upload/presigned URL from book-service - Provider credentials from provider-registry-service - Usage tracking via usage-billing-service BACKEND (AI TTS): POST /v1/tts/generate { chapter_id, language, voice, provider, blocks: [{ index, text }] } → Calls AI provider (OpenAI/ElevenLabs) per block → Stores each segment in MinIO + creates DB record → Returns: { segments: [{ block_index, media_url, duration_ms }] } GET /v1/tts/segments?chapter_id=X&language=Y&voice=Z → Returns existing audio segments for playback (no regeneration) → Includes content_hash for drift detection DELETE /v1/tts/segments/:segmentId → Removes audio asset from DB + MinIO KEYBOARD SHORTCUTS: Space → play/pause Left/Right → prev/next block [/] → decrease/increase speed M → toggle mute Escape → close TTS
