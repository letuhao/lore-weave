Reuses, doesn't fork

Title-mode = ChapterListBrowser (already server-paged + debounced search + multi-select) migrated in as-is, extended with sort/word-count/date-range. Content-mode = RawSearchPanel's hit list restyled to this panel's row density. Both existing hooks (useServerPagedList, useRawSearch) stay the data layer — no new fetch logic invented.

DOCK-1..11 shape

New catalog row chapter-browser (palette + agent openable — an agent plausibly wants "show me chapters missing a translation"). Row click opens the existing editor/book-reader panel via host.openPanel, never a route push. Bulk-translate opens the existing translate wizard as a sibling — not reimplemented here.

Scale contract

Title-mode: keyset cursor via listChaptersKeyset (already built for 10k+). Content-mode: top-K relevance hits, inherently bounded — but search.go's offset is currently ignored (known LOW-3 gap); paging past page 1 in Content-mode needs that fixed first.

Backend additions needed

  • sort param on the chapters list endpoints (fixed today)
  • word_count on the Chapter record (doesn't exist yet)
  • real offset/limit in search.go (LOW-3)
  • optional: updated_since date-range filter
