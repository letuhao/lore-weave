②b Materialize — the four refusals, and the one that is a silent no-op if you don't check it

POST /works/{pid}/arc/materialize has four distinct refusals and a replace negotiation. Each is a different sentence; none is a toast. Rendered here because a spec that only names error codes ships a panel that renders "something went wrong" for all four.

And the fifth failure has no error code at all — which is why it is the dangerous one. AT-6's stamp finishes with POST /books/{bid}/arcs/assign-chapters, which matches chapter rows by predicate and answers {assigned: <count>}. The panel MUST assert assigned === chapter_ids.length and raise an error on a mismatch, never a checkmark. Skip that assertion and the stamp no-ops, structure_node.arc_template_id stays NULL, and Drift renders 422 NO_TEMPLATE_PROVENANCE forever — the panel would be quietly broken while every test stayed green.
