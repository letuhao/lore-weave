1

Lanes = arcs, trunk = chapters, branches = scenes

The tech-tree feel comes from treating arcs as horizontal "eras" (like Civ) and chapters as the trunk progressing left→right within an era, with scenes as short branches — not a generic force-directed graph, which would look messy at book scale and carry no story-order meaning.

2

Collapsed runs, not 300 rendered nodes

A dense run of chapters (or a whole unvisited arc) collapses to one summary node with a count + "Expand" — the graph analogue of Manuscript Navigator's virtualization, so a 10k-chapter book never tries to lay out 10k boxes at once.

3

Badges are read from the SAME data as the sidebar

Cast stack / motif chip / canon ring are the identical pov_entity_id/present_entity_ids/motif_application/canon-check signals as Plan Navigator — one data layer, two renderers.

4

Action rail = PH7's fallback, made visible

Hovering a node shows icon buttons to jump into that node's deeper capabilities. Today most route to the legacy tab (labeled explicitly, e.g. "Cast Codex (legacy tab · not yet ported)") — never a silent dead click — and swap to the real Studio panel as Phase 2 ports land.

5

Detail drawer speaks plain language

Same lesson as the PlanForge Planner redesign (2026-07-06): no raw outline_node/motif_application vocabulary in the UI a writer reads — status pill, named cast, named motifs, a canon banner in a sentence, not a JSON dump.

6

The dashed gold node previews Phase 4 only

Chapter 0003's dashed border + sparkle badge illustrates what an in-flight AI proposal would look like on the graph — not built in v1. Shown here so the vision is legible now, per PH8's explicit sequencing (needs a locked resource_ref convention first).

Why React Flow over a plain SVG hand-roll: nodes here are meant to carry real interactive chrome (status pill, badges, hover action rail, click→drawer) — React Flow gives each node a real React component for free, plus pan/zoom/minimap/collapsible-group mechanics that would otherwise be hand-built. The lane/collapsed-run layout above is custom positioning logic on top of it, not a stock layout — React Flow's job is the canvas mechanics, not the story-specific arrangement.
