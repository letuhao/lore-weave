The codex triages, the arc reads one life. Same split as the Scene Browser/Inspector pair (#22): a grouped list cannot chart a timeline, and a timeline cannot scan 200 entities. Clicking a codex row's arc glyph focuses the arc on that entity; the arc's picker and the codex stay in sync on effectiveEntityId. Both read the same knowledge project (useKnowledgeProjectId, book-scoped).
The story-state badge (active | gone) is DERIVED, never a field you set. EntityStatusEntry is a spoiler-windowed projection of timeline events — from_order is the event_order of the gone-transition (here Ch. 40). Windowing to Ch. 52 dims event #6 and would show active if you read before Ch. 40. To change the band you author the exit event, not a dropdown. This mock renders it read-only on purpose (see the last callout).

Zero new write routes — this is exposure, not construction. Rename/aliases/kind → PATCH /entities/{id} (api.ts:1627). Create → POST /entities (1607). Archive → DELETE /me/entities/{id} (1426). Relation → POST /relations (1617). Every one exists and is OCC- or ownership-guarded; the only work is the FE surface + a catalog row. The reuse target is EntityEditDialog, which already edits name/kind/aliases in the KG feature — the codex mounts the same dialog.
Every write is OCC'd — a stale save must not clobber. updateEntity sends If-Match: W/"<version>"; the BE 428s a request with no If-Match and 412s a stale one. On 412 the row reloads from the refreshed detail (the dialog closes, the KG feature already implements this). The version rides the row (v7 in the inline rename above), so two editors never silently overwrite one identity.

④ The deep-link hand-off cast → character-arc — a cross-panel call, not a tab swap

Today the codex and arc are sub-tabs on one ChapterEditorPage; "view arc" flips onViewArc(entityId) to a lifted selectedEntityId and switches tabs. As dock panels they are separate windows — the same click becomes a typed host call. This is the exact hand-off that breaks silently if the panels are ported without giving openPanel a payload contract.

onViewArc(row.id) → parent lifts selectedEntityId → the arc tab reads it. One component tree, one page. Dies with ChapterEditorPage.

PORTED · dock

Typed openPanel

The arc glyph calls host.openPanel('character-arc', { entityId }) — focuses the panel if open, mounts it (payload = initial selectedEntityId) if not.

CONTRACT

Payload is closed-set-shaped

Panel id is an enum (registered, openable, contracted — WRITE_FRONTEND_CONTRACT=1 pytest). An unknown id must return an error, never a silent no-op (the panel_id bug).

What this mock does NOT propose. It does not add a way to set active/gone directly — that state is a derived projection of timeline events, and a dropdown over it would be the write-only-behavior lie (a control that stores nothing the reader can trust). It does not move prose or facts here — the codex edits entity identity (name/aliases/kind), not the extracted facts, which stay in their own review flow. And it does not invent a new relation model — "+ link entity" is exactly createRelation, ownership-gated (409 if an endpoint isn't yours, 422 on a self-loop). Stated here because the next agent will otherwise "finish" the codex by wiring a fake state toggle.
MCP parity is not optional (MCP-first invariant). Everything a human does here an agent can already do or must be able to: createEntity / updateEntity / createRelation back the authoring; entity kind ships as an enum on the closed set character|location|faction|concept, registered in CLOSED_SET_ARGS. The panel is the human mirror of tools that already exist — not a bespoke path around them.
