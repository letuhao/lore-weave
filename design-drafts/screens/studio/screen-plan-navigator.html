Same tree, different overlay lens

This renders outline_node the exact same way Manuscript Navigator does — reuse the fetch/virtualization/paging, only swap the row decoration. Cast/motif/beat/canon are never tree children (the DB doesn't model them that way); they're badges hanging off the node they attach to.

2

Cast chips (stacked avatars)

From pov_entity_id + present_entity_ids — resolved against glossary-service, batched per page load (not one call per row). Letter + color placeholder here; real build resolves to the entity's actual glossary avatar/initial.

3

Motif dot

From motif_application.outline_node_id — one dot per motif applied at this node, colored by motif. A tooltip names it; clicking jumps the Plan Hub to that motif's full thread.

4

Canon warning

Surfaces the same signal as the Quality tab's canon-issues/canon-flags feed (shipped 2026-07-06) — but IN-CONTEXT on the node it's about, instead of a separate report you have to cross-reference back to a chapter.

Why a separate navigator, not a toggle on Manuscript's: Manuscript Nav's row click contract is "open this in the Editor" — hard-locking that semantic into a second meaning ("focus this on the graph") would make one click do two different things depending on which activity tab you're in, which is confusing and couples two independent panels' state. A second Activity Bar tab keeps Manuscript's contract untouched and gives Plan its own click semantic, matching how VS Code's Explorer and Source Control are two navigators over related-but-different views of the same repo.
