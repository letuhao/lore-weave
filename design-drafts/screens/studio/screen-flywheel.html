Flywheel — the canon-growth home · studio dock panel, draft (S6, the 5th capability)

The companion draft screen-quality-completion.html covers four of S6's five capabilities (canon-rules · corrections · self-heal · progress). This is the fifth. Flywheel is the panel that makes the loop feel closed: you publish a chapter, an extraction pass grows your canon, and this panel says "+7 entities · +3 relations · +2 events" with a chip you can click to land on that new character in Cast. It is a pure PORT — FlywheelPanel, useFlywheel and GET /projects/{id}/flywheel all exist and are tested. Nothing here needs a route, a column, or a migration. What is new is registration, the deep-link retarget, one Lane-B handler (shared with the quality wave), and one contract-row fix.

② The deep-link retarget 3 dead chips if skipped

Lane-A wires the callbacks to CompositionPanel.selectTab(...) — an in-page tab switch. The dock has no tabs — it has panels. Port the render verbatim and the chips fire a selectTab that does nothing. The port replaces the three callbacks with host.openPanel(...), passing the entity name as the focus param the target panel already accepts.

The targets are S7's panels (cast · kg-timeline · kg-graph) — this port DEPENDS on S7 registering them.
loop-connected (bar §2 #6)
Cross-session dependency, stated up front. Flywheel's whole value is handing off into Cast/Timeline/Relations — which S7 owns. If S7 has not yet registered cast / kg-timeline / kg-graph, the port either (a) waits on S7, or (b) ships with the targets that DO exist and defers the rest with a tracked row. It must NOT ship three chips that open nothing — a dead deep-link is the exact "cho có" failure. A chip whose target panel is not yet registered is disabled with a reason, never silently inert.

④ Agent parity the only MUST-BUILD here

The delta is produced by the extraction that follows a publish. If the agent drives the publish, the open flywheel panel must refresh — otherwise it shows a stale "nothing grew". This is the same handlers/compositionEffects.ts the quality wave's X-4 introduces; flywheel adds one predicate to it. There is no MCP tool to build — flywheel is read-only; the agent never "writes a flywheel", it publishes, and this handler reacts.

⑤ Verdict — PORT, and the open questions to close in CLARIFY

PORT, zero backend build. The render, the hook, the read route and the write pipeline all exist and are tested. The work is: register (catalog+enum+contract+guide), retarget three deep-links to host.openPanel, add one predicate to the shared compositionEffects handler, fix the parity-contract row, distinguish error-from-empty (QC-F3), and drive a live-browser smoke. The genuine unknowns are product decisions, filed below — none is a code blocker.

Open questions — flywheel

OQ-F1Category. knowledge (sits with the kg-* panels it links into) vs editor (it's part of the write loop) vs quality? Draft proposes knowledge. PO to confirm.
OQ-F2The loop hand-off — when does the user SEE it? A post-publish reward nobody opens is worthless. Should publish-complete auto-open (or flash) the flywheel panel, or add a "canon grew (+12) →" line to the publish result / progress panel that opens it? Or is palette-only enough for v1? This is the difference between "reachable" and "felt".
OQ-F3Whose delta — last publish, or a range? The route returns the latest delta. Is "your last publish" the right scope, or does the user want "everything this week / this arc"? v1 proposes last-publish (matches the route); a range is a BE ask, deferred.
OQ-F4Relations target. onOpenRelations → kg-graph (the whole graph) or a dedicated relations view? Depends on what S7 registers. Resolve against S7's panel set, not in isolation.
OQ-F5Sequencing vs S7. The three deep-link targets are S7's. Does flywheel port after S7 registers cast/kg-timeline/kg-graph, or port now with disabled-with-reason chips for any not-yet-registered target? Draft leans: port now, degrade gracefully, track the tail.
What this draft deliberately does NOT propose. (a) No MCP flywheel tool — it is a read-only reward view; the agent grows canon by publishing, not by "writing a flywheel". (b) No new backend — every claim above carries a teal tick except the one shared Lane-B predicate. (c) No merge with quality-corrections — the parity contract's conflation is a bug this port fixes, not a design to honor. (d) No book-scale list — the delta is one publish's growth; the scale surface lives in the deep-link targets, which own it.
