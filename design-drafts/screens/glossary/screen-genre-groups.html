Design Notes — P3-08a/b/c
Genre Create/Edit Modal: Launched from Genre Group Editor. Color picker with 10 presets + custom hex. Edit mode shows stats (attrs/kinds/books count). Delete with confirmation.
Book SettingsTab Genre Selector: Multi-select dropdown with checkboxes. Shows selected genres as colored pills with remove button. Impact preview shows which glossary attributes each genre activates. Links to Glossary genre management for creation.
Browse Genre Filter: Replace disabled dashed chips with enabled solid chips. Each chip gets its genre color dot. Active state uses genre color instead of primary. Filter by genre_tags array intersection. Can multi-select genres (OR logic).
Book Cards: Show multiple genre tags (small pills) on cover image bottom-left. Max 2-3 visible, overflow as "+N more".
Data Model (tag-based, no matrix): genre_groups table (glossary-service) defines available genres (name, color, description). Kinds have genre_tags[] — kind appears in entity forms when book has matching genre. Attributes have genre_tags[] — attribute only shows when book has matching genre (empty = always show). Books have genre_tags[] (book-service) — string array of genre names selected by user. No activation matrix table — genre scoping is purely tag-based.
Data Flow: genre_groups (glossary-service) → kind.genre_tags + attr.genre_tags (glossary-service) → book.genre_tags (book-service) → catalog-service filter → browse page chips. Entity form filters kinds/attrs by intersection of book.genre_tags with kind/attr genre_tags.
