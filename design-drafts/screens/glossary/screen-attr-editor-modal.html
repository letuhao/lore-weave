Design Notes

Modal layout: 640px max-width, 4 collapsible sections. Scrollable body, sticky header/footer.

Sections:

  • Core — always visible: name, type, code (readonly), required toggle, active toggle, description
  • Options — only when field_type = "select": drag-reorder list with add/remove
  • Genre Scoping — genre tag pills with colored dots (from genre_groups)
  • AI Assistance — auto-fill prompt + translation hint. Shows "Not configured" / "Configured" badge. Variable chips are clickable to insert into textarea.

Footer left: Kind name context + "Revert to default" (system attrs) or "Delete" (user attrs).

Footer right: Cancel + Save.

AI section notes:

  • Auto-fill prompt: template used when user clicks "AI Fill" on this attribute in the entity editor
  • Translation hint: injected into the translation system prompt when translating this attribute's value
  • Variables: resolved from entity context at runtime (entity_name, role, gender, etc.)
  • Prompt fields are stored as columns on attribute_definitions (future BE migration)
  • AI section is fully functional from day one — just no AI backend to consume the prompts yet
