Section D — Unified TTSBar + Playback Engine
Source priority per block:
1. Attached audio (recorded/uploaded) → plays audio file
2. AI TTS (generated, persisted) → plays audio file
3. Browser TTS (Web Speech API) → speaks text live
4. Standalone audioBlock → played inline, TTSBar skips or pauses

TTSBar shows source badge so reader knows what they're hearing.

Section F — Architecture
TIPTAP SCHEMA: paragraph / heading / blockquote / callout (text blocks): Existing attrs + NEW optional audio attrs: audio_url: string | null ← MinIO presigned URL audio_key: string | null ← MinIO object key audio_subtitle: string | null ← what the audio says (may differ from text) audio_duration_ms: number | null audio_source: 'recorded' | 'uploaded' | 'ai' | null audioBlock (NEW node type): attrs: src: string | null ← MinIO presigned URL media_key: string | null ← MinIO object key subtitle: string | null title: string | null ← filename or description duration_ms: number | null size_bytes: number | null DB TABLE (persisted AI audio — extends previous design): chapter_audio_segments segment_id UUID PK chapter_id UUID NOT NULL block_index INT NOT NULL source_text TEXT NOT NULL ← subtitle / text that was spoken source_text_hash VARCHAR(64) NOT NULL ← for drift detection voice TEXT NOT NULL provider TEXT NOT NULL language TEXT NOT NULL media_key TEXT NOT NULL ← MinIO key duration_ms INT NOT NULL created_at TIMESTAMPTZ DEFAULT now() INDEX: (chapter_id, language, voice, block_index) Note: This table stores AI-generated audio only. Recorded/uploaded audio is stored as block attrs (audio_url, audio_key) directly in the Tiptap JSON — persisted when chapter is saved. MISMATCH DETECTION: For text blocks with audio attachment: blockText = extractText(block) subtitle = block.attrs.audio_subtitle normalize = (s) => s.toLowerCase().replace(/[\s\p{P}]/gu, '') isMismatch = subtitle && normalize(blockText) !== normalize(subtitle) PLAYBACK ENGINE (source priority per block): for each block in chapter: if block.type === 'audioBlock': → play inline (not in TTS bar) else if block.attrs.audio_url: → play attached audio via <audio> element → source badge: 'recorded' or 'uploaded' or 'ai' else if aiSegments[block_index]: → play AI-generated segment via <audio> element → source badge: 'ai' else: → speak via Web Speech API (browser TTS) → source badge: 'browser' COMPONENT TREE: <TTSProvider> state: { status, activeBlockId, source, speed, voice } engines: BrowserTTSEngine ← Web Speech API AudioFileEngine ← plays attached/AI audio via <audio> │ ├── <TTSBar /> ← floating player (reads from context) ├── <TTSSettings /> ← voice, speed, mode toggles └── <AudioOverview /> ← per-block status panel FILES TO CREATE: Editor: frontend/src/components/editor/AudioBlockNode.tsx ← new Tiptap node frontend/src/components/editor/AudioAttachBar.tsx ← mini player on text blocks frontend/src/components/editor/AudioAttachActions.tsx ← upload/record/generate buttons Reader: frontend/src/components/reader/blocks/AudioBlock.tsx ← display component frontend/src/components/reader/TTSBar.tsx frontend/src/components/reader/TTSSettings.tsx frontend/src/components/reader/AudioOverview.tsx Hooks: frontend/src/hooks/useTTS.ts ← unified playback engine frontend/src/hooks/useBlockScroll.ts ← auto-scroll on block change frontend/src/hooks/useAudioUpload.ts ← upload/record audio to MinIO Backend: services/book-service migrate.go ← chapter_audio_segments table services/book-service server.go ← audio CRUD + generate endpoints
