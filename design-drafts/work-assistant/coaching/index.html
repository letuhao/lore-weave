Coaching & self-reflection

Detailed screens for the self-improvement leg. Each realizes a specific design guarantee from the spec — these are not decoration, they are the guarantees made visible.

patterns · evidence · OIS
1 · Weekly reflection NEW
≤2 patterns, each in Observation→Impact→Suggestion form with clickable evidence refs. Dismiss = tombstone.
roleplay · observed behavior
2 · Practice session REUSES roleplay
The only place we actually observe how the user communicates — hence the only place we may score it.
rubric · spans · trend
3 · Scorecard NEW
Rubric-anchored scores; every number cites the exact turn. Trend is comparable because the rubric is versioned.

The guarantees these screens encode

  • Reflection ≠ coaching. In reflection the user does the thinking and we scaffold; in coaching we judge — and only then do we need a standard.
  • The honesty constraint. We never saw the real meeting, so we never grade it. We surface patterns in what the user reported ("you mentioned…"), and we score only what we observed — the practice session. Both screens say this out loud.
  • Cite or drop. Every pattern carries evidence refs into the user's own entries/facts. A pattern with no evidence is never shown.
  • The standard is data, not model memory. Scores come from a versioned, published rubric (Advocacy–Inquiry), shown by name and version — so a trend line means something.
  • Advice cites its source, or is explicitly marked as the assistant's opinion.
  • Precision over volume. ≤2 patterns/week; the dismiss rate is the quality metric.

Open questions for these screens

Does showing the rubric name/version build trust or feel clinical? · Should the growth-edge trend be visible from day one, or only after enough sessions to be meaningful? · Is "Dismiss" enough, or does the user need "this is wrong, and here's why" (which would feed detector tuning)?
