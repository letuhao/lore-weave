Privacy — "Trust Me" Edition

Honest about what this project is (a hobby) and what it isn't (an audited platform).

🔒 The Commitment

LoreWeave is a personal hobby project, not an audited platform. Your content stays on the server you or your host controls. We don't train AI models on your writing — we can't afford the infrastructure to train models even if we wanted to. LLM extraction uses your own BYOK API keys, so AI providers see your content under your own account and billing, not ours. You can export or delete your data any time. No SLA, no support contracts, no enterprise compliance. Use at your own risk.

📊 What Actually Happens to Your Content

1. Raw content (chapters, chat, glossary)
Stored in Postgres on the server you run. Encrypted at rest if you configure disk encryption. Never copied anywhere except your backups. Never sent to any AI provider except via your own BYOK API key.
2. Knowledge graph (Track 2, if enabled)
Stored in Neo4j on the server you run. Derived from your raw content via LLM extraction. Extraction calls go to your chosen BYOK provider under your account.
3. Embeddings
Default: bge-m3 runs locally on your server (fully offline). Your content becomes vectors stored in Neo4j. No external provider involved for the default embedding model.

✅ Your Rights

❌ What We Don't Do

  • No SOC 2 audit — we can't afford $15-50k for annual audits
  • No HIPAA BAA — we cannot be your Business Associate
  • No GDPR DPA — no legal entity to sign one
  • No 24/7 incident response — it's a hobby
  • No uptime guarantee — the server runs when it runs

✅ What We Do Instead

  • Open source — read the code, verify the implementation
  • Self-hostable — run on your own hardware for maximum control
  • BYOK-first — you own the relationship with AI providers
  • Local-first defaults — bge-m3 runs locally, no cloud unless you opt in
  • Honest docs — this page, not a marketing PDF

Memory State Legend

All 13 project memory states from the state machine (KSA §8.4). Each triggers a different card layout.

📖 Static memory
disabled — default for new projects. Glossary fallback active. Zero AI cost.
⚙️ Checking cost...
estimating — awaiting cost estimate from backend.
💡 Ready to build
ready_to_build — cost shown, awaiting user confirm.
⚙️ Building...
building_running — extraction actively running.
⏸ Paused (user)
building_paused_user — user hit pause.
💰 Budget reached
building_paused_budget — auto-paused at spend cap.
⚠️ Paused (error)
building_paused_error — retryable error, user can resume.
🧠 Full memory
complete — full knowledge graph ready.
🆕 Updates pending
stale — new chapters since last extraction.
✗ Failed
failed — unrecoverable error, needs user action.
⚠️ Model change
model_change_pending — user picked new model, needs rebuild.
⚙️ Cancelling...
cancelling — backend cleaning up.
🗑 Deleting graph...
deleting — removing Neo4j data (raw content preserved).

Color semantics

Gray — inactive / disabled
Yellow — caution / paused / static
Blue — in progress / building
Green — ready / active / complete
Red — error / failed / destructive
