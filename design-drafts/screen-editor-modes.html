Editor Modes — Classic & AI Assistant

Design spec for the dual-mode editor system with block guards and media version tracking

1. Mode Toggle

Single toggle in the editor toolbar. Persisted per-user in localStorage. Default: AI Assistant for new users.

Toggle Component

Mode toggle — lives in the toolbar next to the grammar toggle
Classic Mode

Pure writing. No AI chrome. Text blocks only visible. Media blocks shown as compact locked placeholders. Grammar check available but off by default.

AI Assistant Mode

Full features. Grammar inline, AI prompts on media, TTS/audio controls, image/video generation, format bar with insert tools. All block types editable.

Switching modes does NOT delete content. A chapter with images/video/audio viewed in Classic mode simply shows those blocks as compact locked placeholders. All data is preserved. The writer can switch to AI Assistant mode anytime to edit media.

4. Block Guards — Classic Mode Protection

When a Classic mode user tries to delete or modify a media block (via keyboard backspace at block boundary, drag, or any edit action):

Guard overlay — appears when Classic user tries to delete a media block

Guard overlay on attempted media deletion Classic
throne-room-concept.png
This is a media block. Switch to AI Assistant mode to edit or delete it.
Media blocks are protected in Classic mode to prevent accidental loss.

Guard behaviors

User action in Classic modeBehavior
Backspace at start of text block that follows a media blockCursor stops. Does NOT merge into or delete the media block.
Delete at end of text block before a media blockCursor stops. Media block protected.
Drag a media blockDrag handle hidden on media blocks. Not draggable.
Select-all + delete (Ctrl+A, Backspace)Only text blocks are cleared. Media blocks survive.
Paste over a selection that includes media blocksPaste replaces text blocks only. Media blocks stay in place.
Click on a media blockShows compact preview. No edit controls.
Right-click a media blockContext menu: "View in AI mode" / "Copy filename"
Implementation note (Tiptap): Use NodeView with editable: false for media blocks in Classic mode. The Tiptap schema stays the same — only the NodeView rendering and key bindings change per mode. The handleKeyDown extension intercepts backspace/delete at media boundaries.

Storage note: Version media files (old PNGs, MP3s) are kept in MinIO with path pattern chapters/{chapterId}/blocks/{blockId}/v{N}.{ext}. Old versions can be garbage-collected by retention policy (e.g. keep last 10 versions, or last 30 days). The prompt snapshots are cheap text — always kept.

7. Mode Switching Flow

TransitionWhat happens
Classic → AI AssistantAll blocks become fully editable. Media blocks expand from compact placeholders to full views. Audio controls appear on text blocks. Grammar auto-enables. Format bar expands. No data loss.
AI Assistant → ClassicMedia blocks collapse to compact locked placeholders. Audio controls hide. Format bar minimizes. AI prompts, grammar panel, insert tools hide. No data is deleted. Writer can switch back anytime.
AI → Classic with unsaved media editsStandard unsaved-changes dialog: "Save changes before switching?" Save / Discard / Stay.
