# Foundation tiles — LICENSES

**Status:** V1 foundation tiles for the tilemap viewer.
**License:** OWN CODE — no third-party model output baked in.

## Source

10 PNGs algorithmically generated by
[`frontend-game/scripts/gen-foundation-tiles.py`](../../../../scripts/gen-foundation-tiles.py).

The script generates seamless tileable terrain foundations using:
- FFT-band-passed white noise (guaranteed periodic in the DFT, so the
  IFFT output tiles seamlessly without UV seam artifacts)
- Per-terrain RGB palette + ±low/mid-freq luminance modulation
- Per-terrain "specials" — Voronoi cracks for mountain, sinusoidal
  ripple for water, directional wind for sand, mineral-vein gradient
  for subterranean, etc.

Filename = TerrainKind tag from `services/tilemap-service/src/types/tile.rs`:
`grass`, `forest`, `mountain`, `water`, `sand`, `snow`, `swamp`, `road`,
`rough`, `subterranean`.

256×256 each, ~50 KB per tile, ~540 KB total.

Deterministic per `SEED_BASE` constant in the script (default `2026`).
Re-run with a different `SEED_BASE` to refresh the look.

## Why algorithm, not AI-generated

Foundation tiles serve as the BASE LAYER under which props (trees,
structures, decoration) are placed. They must be:
- **Flat** — no objects, no scene composition (else props overlay
  looks cluttered)
- **Seamless** — tileable without visible seams when the same tile
  repeats across a 64×64 grid
- **Cheap-per-book** — multi-book platform constraint: each new book
  picks a palette and re-runs the script in seconds, no GPU needed

AI image-gen models (Flux1-dev, even SDXL with texture LoRA) tend to
produce "scene compositions" with vegetation framing — that's what
they were trained on. Procedural noise + per-terrain palette gives
the exact "flat foundation under props" result we want for V1+.
